## Fix 1 — Near Transit: Remove It (It's Actively Broken)

Near Transit doesn't just do nothing — it **resets the listing count to 19**, meaning it's clearing other active filters or applying no filter clause at all. This is worse than useless.

**Find and delete every reference to `near_transit` in ExploreScreen:**

```ts
// DELETE from the filter type union:
type QuickFilter = 'best_match' | 'under_2k' | 'pet_friendly' | 'available_now';
// Remove: | 'near_transit' | 'private_room'  ← private_room moves to listingTypeFilter

// DELETE the chip JSX:
// <Pressable style={...} onPress={() => setActiveFilter('near_transit')}>
//   <Text>Near Transit</Text>
// </Pressable>

// DELETE from filter logic (this is what's causing the 19-listing reset):
// case 'near_transit':
//   return listings;   ← this returns ALL listings, overriding any previous filter
// OR it may look like:
// if (activeFilter === 'near_transit') setFilteredListings(allListings);
```

> **Root cause of the 19-listing bug:** The `near_transit` case in the filter switch/if-else is almost certainly returning `allListings` unfiltered (since there's no transit data to filter against). This overwrites whatever other filter was active, which is why the count jumps UP when you tap it.

**Future:** When you add Walk Score API data or a `transit_score` field to your listings table, re-add transit filtering inside the **Filters modal** (not the quick chips) with a "Within X min of transit" slider.

## Fix 3 — Best Match: Verify Sort Field

Best Match sorts listings but with only 8 listings it's hard to see the effect. Make sure the sort is pointing at the correct field name from your Supabase response.

```ts
// In your filteredListings memo — find the Best Match sort and verify the field:

if (quickFilter === 'best_match') {
  results = [...results].sort((a, b) => {
    // CHECK: what field holds the match score on your listing objects?
    // Common names: matchScore, match_score, compatibilityScore, score
    const scoreA = a.matchScore ?? a.match_score ?? a.score ?? 0;
    const scoreB = b.matchScore ?? b.match_score ?? b.score ?? 0;
    return scoreB - scoreA; // descending: highest match first
  });
}
```

> **Also check:** If match scores are computed per-user (comparing the renter's profile to each listing), make sure you're passing `userId` into the score lookup. If `matchScore` is null or undefined for most listings, they'll all sort to 0 and appear randomly ordered.

## Fix 7 — Private Room: Move to listingTypeFilter State

Private Room is a **type** filter, not a quick filter. It should live in `listingTypeFilter` state (mutually exclusive with Entire Unit), not `quickFilter`. This prevents the edge case where someone has "Best Match" active AND "Private Room" active at the same time — both should work simultaneously.

```ts
// BEFORE — private_room is mixed into the quickFilter state:
const [quickFilter, setQuickFilter] = useState<'best_match' | 'under_2k' | 'private_room' | 'pet_friendly' | 'available_now' | null>(null);

// AFTER — private_room moves to its own state, separate from quickFilter:
const [quickFilter, setQuickFilter] = useState<'best_match' | 'under_2k' | 'pet_friendly' | 'available_now' | null>(null);
const [listingTypeFilter, setListingTypeFilter] = useState<'room' | 'entire' | null>(null);
```

Update the Private Room chip to use `listingTypeFilter`:

## Complete Corrected Filter Logic

```ts
const filteredListings = useMemo(() => {
  let results = [...allListings];

  // ── 1. City ──────────────────────────────────────────
  results = results.filter(l => l.city === selectedCity);

  // ── 2. Neighborhood ──────────────────────────────────
  if (selectedNeighborhood) {
    results = results.filter(l =>
      l.neighborhood?.toLowerCase().includes(selectedNeighborhood.toLowerCase())
    );
  }

  // ── 3. Listing type (Private Room / Entire Unit) ──────
  // These are MUTUALLY EXCLUSIVE — selecting one clears the other
  if (listingTypeFilter === 'room') {
    results = results.filter(l =>
      ['room', 'private_room', 'Private Room'].some(v =>
        l.type?.toLowerCase() === v.toLowerCase() ||
        l.listing_type?.toLowerCase() === v.toLowerCase()
      )
    );
  } else if (listingTypeFilter === 'entire') {
    results = results.filter(l =>
      ['entire', 'entire_unit', 'Entire Unit'].some(v =>
        l.type?.toLowerCase() === v.toLowerCase() ||
        l.listing_type?.toLowerCase() === v.toLowerCase()
      )
    );
  }

  // ── 4. Quick filters (can stack with type filter) ─────

  // Pet Friendly
  if (quickFilter === 'pet_friendly') {
    results = results.filter(l =>
      l.petFriendly === true || l.pet_friendly === true || l.pets_allowed === true
    );
  }

  // Available Now
  if (quickFilter === 'available_now') {
    const today = new Date().toISOString().split('T')[0];
    results = results.filter(l =>
      l.available_now === true ||
      l.status === 'available' ||
      l.status === 'active' ||
      (l.move_in_date && l.move_in_date <= today)
    );
  }

  // Under $2k
  if (quickFilter === 'under_2k') {
    results = results.filter(l => (l.price ?? l.rent ?? 0) <= 2000);
  }

  // ── 5. Full Filters modal (budget, bedrooms, amenities) ─
  results = results.filter(l => {
    const price = l.price ?? l.rent ?? 0;
    return price >= budgetMin && price <= budgetMax;
  });
  if (bedroomsFilter > 0) {
    results = results.filter(l => (l.bedrooms ?? 0) >= bedroomsFilter);
  }
  if (bathroomsFilter > 0) {
    results = results.filter(l => (l.bathrooms ?? 0) >= bathroomsFilter);
  }
  if (amenitiesFilter.length > 0) {
    results = results.filter(l =>
      amenitiesFilter.every(a => l.amenities?.includes(a))
    );
  }

  // ── 6. Best Match sort (applied last, after all filters) ─
  if (quickFilter === 'best_match') {
    results = results.sort((a, b) => {
      const scoreA = a.matchScore ?? a.match_score ?? a.score ?? 0;
      const scoreB = b.matchScore ?? b.match_score ?? b.score ?? 0;
      return scoreB - scoreA;
    });
  }

  return results;
}, [
  allListings,
  selectedCity,
  selectedNeighborhood,
  listingTypeFilter,
  quickFilter,
  budgetMin,
  budgetMax,
  bedroomsFilter,
  bathroomsFilter,
  amenitiesFilter,
]);
```
