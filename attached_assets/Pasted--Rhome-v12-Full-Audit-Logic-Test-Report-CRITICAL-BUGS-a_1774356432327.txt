# Rhome v12 — Full Audit & Logic Test Report

## CRITICAL BUGS (app crashes or payments completely broken)

### 1. All Stripe Boost Price IDs Are Fake — PAYMENT BLOCKER
- **File:** `constants/stripePrices.ts` lines 19–26
- All 6 boost price IDs are set to `'price_REPLACE_WITH_REAL_ID'`
- Affected: renter_boost_6h, renter_boost_12h, renter_boost_24h, listing_boost_6h, listing_boost_12h, listing_boost_24h
- **Impact:** Every boost purchase fails. Stripe rejects non-existent price IDs.

### 2. All Subscription Price IDs Are Also Fake
- **File:** `constants/stripePrices.ts`
- Renter plans: `price_plus_monthly`, `price_plus_3month`, `price_plus_annual`, `price_elite_monthly` etc.
- Host plans: `price_host_starter_monthly`, `price_host_pro_monthly`, `price_host_business_monthly` etc.
- **Impact:** Zero subscriptions can be processed. Every plan upgrade fails silently.
- **Total fake price IDs: 27**

### 3. Stripe Publishable Key Not Configured — Silent Payment Failure
- **File:** `components/StripeWrapper.native.tsx`
- If `EXPO_PUBLIC_STRIPE_PUBLISHABLE_KEY` is empty, the wrapper skips Stripe entirely with `return <>{children}<>`
- **Impact:** The app looks normal but no payment UI ever loads. Users can't pay for anything.

### 4. useStripePayment Hook Missing Required Parameters
- **File:** `hooks/useStripePayment.native.ts` lines 23–24
- Calls `create-subscription` Edge Function but does NOT send `plan` or `billingCycle` in the body
- The Edge Function expects these fields
- **Impact:** Subscription creation fails silently every time

### 5. Demo Account Security Bypass — REMOVE BEFORE PRODUCTION
- **File:** `contexts/AuthContext.tsx` line 412 and `screens/auth/LoginScreen.tsx` line 31
- Email `demo@rhome.com` / password `demo123` bypasses real auth and loads mock data
- **Impact:** Anyone can log into a demo account with full app access. Critical security risk in production.

### 6. PaymentScreen Uses Fake Card Storage — PCI Violation
- **File:** `screens/shared/PaymentScreen.tsx` lines 69–76
- Generates fake payment method IDs (`pm_dev_` or `pm_`) instead of using real Stripe tokens
- Stores card data locally in app state
- **Impact:** No real payments processed. PCI non-compliant. Must use Stripe Payment Sheet.

## HIGH PRIORITY (features broken or incorrect)

### 7. Stripe Webhook Looks Up Subscription in Wrong Table
- **File:** `supabase/functions/stripe-webhook/index.ts` lines 41–45 vs 149–153
- Sometimes queries `profiles` table for stripe_customer_id, sometimes queries `subscriptions` table
- **Impact:** Webhook events may not update the correct record → subscriptions don't activate after payment

### 8. AI Assistant Daily Usage Count Doesn't Handle Timezones
- **File:** `supabase/functions/ai-assistant/index.ts` lines 43–52
- Uses `new Date().toISOString().split('T')[0]` for date but doesn't account for user timezone
- **Impact:** Free tier AI limits reset at wrong time for users in non-UTC timezones

### 9. Group Listings Match Without Checking Listing is Still Active
- **File:** `services/groupService.ts`
- No validation that a listing is still active before matching
- **Impact:** Renter groups get matched to deleted/archived listings

### 10. Notification Badge Shows on Wrong Tabs
- **File:** `navigation/RenterTabNavigator.tsx` lines 50 and 186
- Unread message badge appears on both Profile AND Messages tabs simultaneously
- **Impact:** Confusing UX — users don't know which tab has new content

## MEDIUM PRIORITY (UX issues, missing states)

### 11. Missing Empty States on Key Screens
- RoommatesScreen: No state when user has swiped everyone
- BrowseRentersScreen: No message when no renters match filters
- GroupsScreen: No first-time user empty state
- MessagesScreen: No empty state

### 12. Prices Hardcoded in UI Instead of Coming From Backend
- **Files:** `screens/shared/PlansScreen.tsx` and `screens/host/HostPricingScreen.tsx`
- All prices ($14.99, $29.99, $19.99 etc.) are hardcoded in the component
- If you change pricing in Stripe, UI won't update automatically

### 13. Navigation Type Safety Uses `as any` Casts
- Multiple screens use `as any` in navigation calls
- Hides TypeScript errors that could cause runtime crashes

### 14. No Error Boundaries
- If any screen crashes, the entire app crashes
- No user-facing error messages for most failures

### 15. References Section Marked TODO — Never Implemented
- **File:** `screens/shared/ProfileScreen.tsx`
- Comment: "TODO: References section — build invite-based reference system post-launch"
- Section is documented but not built
