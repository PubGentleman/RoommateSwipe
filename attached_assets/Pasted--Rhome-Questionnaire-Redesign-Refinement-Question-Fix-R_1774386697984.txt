# Rhome — Questionnaire Redesign + Refinement Question Fix
## Replit Agent Prompt

---

## Overview

This prompt fixes two deeply connected problems:

1. **Questionnaire order and length** — Budget and location are collected too late (steps 4 and 12). The flow is 15 steps long which kills completion rates. The expenses step is low value. Personality step has duplicate questions.

2. **Refinement questions** — All 5 current refinement questions are duplicates of what the questionnaire already collects. They must be completely replaced with questions that capture genuinely new signals.

## Part 2 — Fix the Matching Algorithm to Use Dealbreakers

### File: `utils/matchingAlgorithm.ts`

Add dealbreaker filtering at the very top of `calculateDetailedCompatibility`, before any scoring. If a user's dealbreakers conflict with a profile, return a score of 0 immediately:

```typescript
export const calculateDetailedCompatibility = (
  currentUser: User,
  roommateProfile: RoommateProfile
): MatchScore => {

  // ── HARD DEALBREAKER CHECK — runs before any scoring ──
  const userDealbreakers: string[] = currentUser.profileData?.dealbreakers || []

  if (userDealbreakers.length > 0) {
    const profileSmoking = roommateProfile.lifestyle?.smoking || roommateProfile.profileData?.preferences?.smoking
    const profilePets = roommateProfile.lifestyle?.pets || roommateProfile.profileData?.preferences?.pets
    const profilePetType = roommateProfile.profileData?.petType  // 'cat' | 'dog' | 'other'

    for (const dealbreaker of userDealbreakers) {
      switch (dealbreaker) {
        case 'no_smokers':
          if (profileSmoking === 'yes' || profileSmoking === 'only_outside') return ZERO_SCORE
          break
        case 'no_cats':
          if (profilePets === 'have_pets' && profilePetType === 'cat') return ZERO_SCORE
          break
        case 'no_dogs':
          if (profilePets === 'have_pets' && profilePetType === 'dog') return ZERO_SCORE
          break
        case 'no_pets':
          if (profilePets === 'have_pets') return ZERO_SCORE
          break
        case 'no_overnight_guests':
          if (roommateProfile.profileData?.preferences?.guestPolicy === 'frequently') return ZERO_SCORE
          break
      }
    }
  }

  // ... rest of existing scoring code unchanged ...
```

export const shouldShowRefinementQuestion = async (
  rightSwipes: number,

  allQuestionsAnswered: boolean

  if (allQuestionsAnswered) return false

  const matchRate = totalSwipes > 0 ? rightSwipes / totalSwipes : 1

  if (questionsAsked.length === 0) {
    return (
      swipeCount >= MIN_SWIPES_BEFORE_FIRST_QUESTION &&

## Part 5 — Save Refinement Answers to the Right Place

### File: `components/RhomeAISheet.tsx`

Find where refinement question answers are currently saved. They should be saved to the same `personalityAnswers` field in the profile as the main questionnaire, NOT as separate data:

```typescript
// When a refinement answer is submitted, save it to both:
// 1. The local personalityAnswers state (for immediate matching)
// 2. The profiles table in Supabase (for persistence)

const handleRefinementAnswer = async (questionId: string, value: string) => {
  const { data: { user: authUser } } = await supabase.auth.getUser()
  if (!authUser) return

  // Get current profile
  const { data: profile } = await supabase
    .from('profiles')
    .select('personality_answers')
    .eq('id', authUser.id)
    .single()

  const currentAnswers = profile?.personality_answers || {}
  const updatedAnswers = { ...currentAnswers, [questionId]: value }

  // Save to Supabase
  await supabase
    .from('profiles')
    .update({ personality_answers: updatedAnswers })
    .eq('id', authUser.id)

  // Also update AI memory if the field maps to a memory field
  await supabase
    .from('user_ai_memory')
    .upsert({
      user_id: authUser.id,
      ...(questionId === 'commuteTime' && { commute_preference: value }),
      ...(questionId === 'roommateRelationship' && { social_preference: value }),
      ...(questionId === 'priorityFactor' && { priority_factor: value }),
      last_updated: new Date().toISOString(),
    }, { onConflict: 'user_id' })
}
```
