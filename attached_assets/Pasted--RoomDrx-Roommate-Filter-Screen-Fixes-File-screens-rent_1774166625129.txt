# RoomDrx — Roommate Filter Screen Fixes

## File: `screens/renter/RoommateFiltersScreen.tsx`
(may also be `MatchFiltersScreen.tsx` or a modal inside `RoommatesScreen.tsx`)

---

## Issues Being Fixed

1. **Chip contrast is too low** — dark grey pills on near-black background, unselected state is nearly invisible
2. **Room Type inconsistency** — this screen has Private Room / Shared Room / Entire Place / Studio; Explore screen has Private Room / Entire Unit. Needs to match
3. **Search Radius distances are too large** — 25mi and 50mi extend far outside NYC; replace with neighborhood-scale options
4. **Minimum Compatibility has 6 chips** — too many for one row, overflows on smaller screens; simplify to 4

---

## Fix 1 — Chip Contrast: Active vs Unselected States

All filter chips (Move-in Date, Room Type, Lifestyle, Search Radius, Compatibility) currently use near-invisible unselected styles. Apply this consistent chip pattern across all chip groups:

```ts
// REPLACE all existing chip style pairs with these:

// Unselected chip — visible border so it reads as a tappable element
filterChip: {
  paddingHorizontal: 16,
  paddingVertical: 9,
  borderRadius: 20,
  backgroundColor: 'rgba(255,255,255,0.06)',
  borderWidth: 1.5,
  borderColor: 'rgba(255,255,255,0.15)',   // ← visible border
},
filterChipText: {
  fontSize: 13,
  fontWeight: '600',
  color: 'rgba(255,255,255,0.55)',          // ← readable unselected text
},

// Selected chip — coral fill, no border needed
filterChipActive: {
  backgroundColor: '#ff6b5b',
  borderColor: 'transparent',
},
filterChipTextActive: {
  color: '#fff',
  fontWeight: '700',
},
```

Apply these styles to **every** chip group on the screen:
- Budget presets ($500-$1,000, $1,000-$2,000, etc.)
- Move-in Date (ASAP, Within 30 days, Within 3 months)
- Room Type (all options)
- Lifestyle (all options)
- Search Radius (all options)
- Minimum Compatibility (all options)

## Fix 2 — Room Type: Align with Explore Screen

The Explore screen uses: **Private Room | Entire Unit**
This filter screen uses: **Private Room | Shared Room | Entire Place | Studio**

Since this is the roommate matching filter (not listing filter), it makes sense to have more options. But "Entire Place" should be renamed "Entire Unit" to match the badge text shown on listing cards, and keep "Shared Room" and "Studio" as they're valid room types.

```tsx
// Updated Room Type chips — 4 options, consistent naming:
{[
  { key: 'private_room', label: 'Private Room', icon: 'user'   },
  { key: 'shared_room',  label: 'Shared Room',  icon: 'users'  },
  { key: 'entire_unit',  label: 'Entire Unit',  icon: 'home'   },  // ← renamed from "Entire Place"
  { key: 'studio',       label: 'Studio',       icon: 'square' },
].map((opt) => (
  <Pressable
    key={opt.key}
    style={[styles.filterChip, roomTypes.includes(opt.key) && styles.filterChipActive]}
    onPress={() => toggleRoomType(opt.key)}
  >
    <Feather
      name={opt.icon}
      size={12}
      color={roomTypes.includes(opt.key) ? '#fff' : 'rgba(255,255,255,0.45)'}
    />
    <Text style={[styles.filterChipText, roomTypes.includes(opt.key) && styles.filterChipTextActive]}>
      {opt.label}
    </Text>
  </Pressable>
))}
```

> Also update the Explore screen's listing filter to use `entire_unit` (not `entire`) as the key so both screens share the same value strings.

## Fix 3 — Search Radius: City-Scale Distances

Replace the current 5mi / 10mi / 25mi / 50mi options with neighborhood-scale options appropriate for dense city living:

```tsx
// BEFORE:
const RADIUS_OPTIONS = ['5mi', '10mi', '25mi', '50mi'];

// AFTER:
const RADIUS_OPTIONS = [
  { label: 'Nearby',  value: 2  },   // ~walking / same neighborhood
  { label: '5 mi',   value: 5  },   // same borough
  { label: '10 mi',  value: 10 },   // adjacent boroughs
  { label: '25 mi',  value: 25 },   // outer metro
];

// Render:
{RADIUS_OPTIONS.map((opt) => (
  <Pressable
    key={opt.value}
    style={[styles.filterChip, searchRadius === opt.value && styles.filterChipActive]}
    onPress={() => setSearchRadius(opt.value)}
  >
    <Text style={[styles.filterChipText, searchRadius === opt.value && styles.filterChipTextActive]}>
      {opt.label}
    </Text>
  </Pressable>
))}
```

> **Default:** Set `searchRadius` default to `5` (5mi) — specific enough to be useful, not so tight it filters out most results.

## Fix 4 — Minimum Compatibility: Reduce to 4 Options

6 chips in one row overflows on smaller screens. Simplify:

```tsx
// BEFORE — 6 options:
const COMPAT_OPTIONS = ['Any', '50%+', '60%+', '70%+', '80%+', '90%+'];

// AFTER — 4 options that cover the meaningful thresholds:
const COMPAT_OPTIONS = [
  { label: 'Any',  value: 0  },
  { label: '60%+', value: 60 },
  { label: '75%+', value: 75 },
  { label: '90%+', value: 90 },
];

// Render:
<View style={styles.compatRow}>
  {COMPAT_OPTIONS.map((opt) => (
    <Pressable
      key={opt.value}
      style={[
        styles.compatChip,
        minCompatibility === opt.value && styles.filterChipActive,
      ]}
      onPress={() => setMinCompatibility(opt.value)}
    >
      <Text style={[
        styles.filterChipText,
        minCompatibility === opt.value && styles.filterChipTextActive,
      ]}>
        {opt.label}
      </Text>
    </Pressable>
  ))}
</View>
```

**Styles — give each chip equal flex so they fill the row cleanly:**

```ts
compatRow: {
  flexDirection: 'row',
  gap: 10,
},
compatChip: {
  flex: 1,                              // ← equal width, no overflow
  alignItems: 'center',
  paddingVertical: 11,
  borderRadius: 20,
  backgroundColor: 'rgba(255,255,255,0.06)',
  borderWidth: 1.5,
  borderColor: 'rgba(255,255,255,0.15)',
},
```
