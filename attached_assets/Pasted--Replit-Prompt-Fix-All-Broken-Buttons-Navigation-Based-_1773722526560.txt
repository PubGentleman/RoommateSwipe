## BUG #1 — CRASH: `ListingBoostScreen.tsx` — "See Plans" crashes from Listings tab

**File:** `screens/host/ListingBoostScreen.tsx`

**Problem:** When a free-plan host opens `ListingBoostScreen` from the **Listings tab** (via `MyListings → ListingBoost`), the "See Plans" upgrade button calls `navigation.navigate('HostSubscription')`. But `HostSubscription` is NOT registered in the `ListingsStack` — only in `DashboardStack`. This crashes the app.

**Find this code (around line 96):**
```tsx
onPress={() => navigation.navigate('HostSubscription')}
```

**Replace with:**
```tsx
onPress={() => {
  const parent = navigation.getParent();
  if (parent) {
    parent.navigate('Dashboard', { screen: 'HostSubscription' });
  }
}}
```

## BUG #3 — WRONG DESTINATION: `HostDashboardScreen.tsx` — Bell icon goes to Inquiries, not Notifications

**File:** `screens/host/HostDashboardScreen.tsx`

**Problem:** The bell/notification icon in the header navigates to `'Inquiries'` (a stack screen). A bell icon should navigate to a notifications screen. Users tapping the bell expect to see notifications, not inquiry cards.

**Find this code (around line 342):**
```tsx
<Pressable style={styles.iconBtn} onPress={() => navigation.navigate('Inquiries')}>
  <Feather name="bell" size={16} color="rgba(255,255,255,0.6)" />
```

**Replace with:**
```tsx
<Pressable style={styles.iconBtn} onPress={() => navigateToTab('Profile', { screen: 'Notifications' })}>
  <Feather name="bell" size={16} color="rgba(255,255,255,0.6)" />
```

If `navigateToTab` doesn't support params, use:
```tsx
<Pressable style={styles.iconBtn} onPress={() => {
  const parent = navigation.getParent();
  if (parent) parent.navigate('Profile', { screen: 'Notifications' });
}}>
  <Feather name="bell" size={16} color="rgba(255,255,255,0.6)" />
```

## BUG #4 — WRONG ACTION: `CreateEditListingScreen.tsx` — "Upgrade Plan" in cap-limit alert calls `goBack()`

**File:** `screens/host/CreateEditListingScreen.tsx`

**Problem:** When a host tries to add a listing but is over their plan's listing cap, an Alert fires with an "Upgrade Plan" button. That button currently calls `navigation.goBack()` — it should send the user to the subscription/upgrade screen instead.

**Find this code (around line 173):**
```tsx
{ text: 'Upgrade Plan', onPress: () => navigation.goBack() },
```

**Replace with:**
```tsx
{
  text: 'Upgrade Plan',
  onPress: () => {
    const parent = navigation.getParent();
    if (parent) {
      parent.navigate('Dashboard', { screen: 'HostSubscription' });
    } else {
      navigation.navigate('HostSubscription' as any);
    }
  },
},
```

## BUG #8 — CONFIRM: All `onPress={() => {}}` patterns are intentional

The following `onPress={() => {}}` patterns found in the codebase are **intentional stop-propagation** patterns on modal sheet inner containers. They prevent taps inside the sheet from dismissing it by bubbling to the overlay. Do NOT change these:

- `screens/shared/ProfileScreen.tsx:583` — boost modal inner sheet
- `screens/shared/ProfileScreen.tsx:715` — reference request sheet
- `screens/shared/ManageSubscriptionScreen.tsx:154` — cancel confirmation sheet
- `screens/renter/RoommatesScreen.tsx:1353` — profile boost modal sheet
- `screens/renter/ExploreScreen.tsx:1159` — inquiry modal sheet
- `components/DatePickerModal.tsx:277` — date picker sheet
- `components/PaywallSheet.tsx:124` — paywall sheet

These are all correct and should remain as `onPress={() => {}}`.
