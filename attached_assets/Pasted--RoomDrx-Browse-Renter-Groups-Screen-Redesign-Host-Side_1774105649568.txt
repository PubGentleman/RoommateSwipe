# RoomDrx — Browse Renter Groups Screen Redesign (Host Side)

## File: `screens/host/BrowseRenterGroupsScreen.tsx`

---

## Issues Being Fixed

1. **"Message This Group" CTA uses wrong brand color** — `LinearGradient colors={[ROOMDR_PURPLE, '#6a4d96']}` doesn't match the coral `#ff6b5b` accent used everywhere else in the app
2. **Dashed "+" avatar slot reads as broken UI** — same `avatarEmpty` pattern fixed in Groups Discover; needs same treatment here
3. **Quota bar too narrow** — `width: 70` hardcoded track makes the occupancy bar feel like a thin decorative line rather than a meaningful indicator
4. **Compose sheet send button is also purple** — `sendBtnGradient` uses purple gradient, inconsistent with app-wide coral CTA pattern
5. **Card lacks visual depth** — `#1a1a1a` on `#111` background with no shadow; cards feel flat

---

## Change 1 — CTA Button: Replace Purple Gradient with Coral

The "Message This Group" button uses `ROOMDR_PURPLE` which conflicts with the coral accent used in every other primary action in the app.

```tsx
// BEFORE:
<Pressable style={styles.ctaButton} onPress={() => handleMessageGroup(item)}>
  <LinearGradient
    colors={[ROOMDR_PURPLE, '#6a4d96']}
    start={{ x: 0, y: 0 }}
    end={{ x: 1, y: 0 }}
    style={styles.ctaGradient}
  >
    <Feather name="message-circle" size={16} color="#fff" />
    <Text style={styles.ctaText}>Message This Group</Text>
  </LinearGradient>
</Pressable>

// AFTER — drop the LinearGradient wrapper, use solid coral:
<Pressable
  style={[styles.ctaButton, { backgroundColor: '#ff6b5b' }]}
  onPress={() => handleMessageGroup(item)}
>
  <Feather name="message-circle" size={16} color="#fff" />
  <Text style={styles.ctaText}>Message This Group</Text>
</Pressable>
```

**Update styles:**

```ts
ctaButton: {
  flexDirection: 'row',
  alignItems: 'center',
  justifyContent: 'center',
  gap: 8,
  backgroundColor: '#ff6b5b',
  borderRadius: 14,
  paddingVertical: 14,
  marginTop: 16,
},
ctaText: {
  fontSize: 15,
  fontWeight: '700',
  color: '#fff',
  letterSpacing: 0.3,
},
// Remove ctaGradient from StyleSheet entirely
```

## Change 3 — Quota Bar: Full-Width Track

The quota bar track is hardcoded at `width: 70`, making it feel like a thin decorative element. Replace with a flex-based layout so it fills the available card width.

```tsx
// BEFORE:
<View style={styles.quotaTrack}>
  <View style={[styles.quotaFill, { width: quotaPercent * 70 }]} />
</View>

// AFTER — flex-based width:
<View style={styles.quotaTrack}>
  <View style={[styles.quotaFill, { flex: quotaPercent }]} />
  <View style={{ flex: 1 - quotaPercent }} />
</View>
```

**Update styles:**

```ts
quotaBarRow: {
  flexDirection: 'row',
  alignItems: 'center',
  gap: 10,
  marginTop: 8,
  paddingHorizontal: 2,
},
quotaLabel: {
  fontSize: 11,
  color: 'rgba(255,255,255,0.4)',
  fontWeight: '500',
  minWidth: 64,
},
quotaTrack: {
  flex: 1,
  height: 6,
  backgroundColor: 'rgba(255,255,255,0.08)',
  borderRadius: 3,
  flexDirection: 'row',
  overflow: 'hidden',
},
quotaFill: {
  height: 6,
  backgroundColor: '#ff6b5b',
  borderRadius: 3,
},
quotaCount: {
  fontSize: 11,
  color: 'rgba(255,255,255,0.4)',
  fontWeight: '500',
  minWidth: 32,
  textAlign: 'right',
},
```

> If `quotaPercent` is computed as `item.interestedCount / item.maxMembers`, ensure it's clamped: `const quotaPercent = Math.min(item.interestedCount / item.maxMembers, 1)`.
