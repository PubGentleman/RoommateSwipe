# Rhome — Fix Swipe Card Flash-Back Bug
## Replit Agent Prompt

---

## The Bug

When a user swipes a card, the swiped card briefly reappears at the center of the deck for about half a second before disappearing. This is caused by a race condition between the exit animation and the React state update that removes the card from the array. When the state updates, React re-renders the deck — and for one frame the card snaps back to its default position (center/0) before the component unmounts.

---

## Fix Overview

Four changes are required:

1. Track the swiped card with a **ref** (not state) so removal doesn't trigger a re-render mid-animation
2. Keep the swiping card **rendered but invisible** using opacity 0 until after the exit animation fully completes
3. Wrap the card-removal state update in **`InteractionManager.runAfterInteractions`** so it waits until all animations finish
4. Ensure **`useNativeDriver: true`** is set on every `Animated.Value` used for position and opacity

---

## Part 1 — Find the Swipe Card Component

Locate the main swipe deck component. It will be one of these files:
- `components/SwipeCard.tsx`
- `components/SwipeDeck.tsx`
- `components/CardStack.tsx`
- `screens/DiscoverScreen.tsx` (if the deck logic lives in the screen)

Read the file and identify:
- Where `Animated.Value` is created for position (x/y)
- Where `onSwipeComplete` or equivalent fires and removes the card from the array
- Whether `useNativeDriver: true` is present on all animated transforms

---

## Part 2 — Add a Swiping-Out Ref

At the top of the swipe deck component, add a ref that tracks which card ID is currently mid-swipe:

```typescript
import { useRef, useState } from 'react';
import { InteractionManager } from 'react-native';

// Track which card is animating out — using a ref avoids triggering re-renders
const swipingOutId = useRef<string | null>(null);
```

---

## Part 3 — Mark Card as Swiping Out Before Animation Starts

In the swipe handler (wherever `Animated.spring` or `Animated.timing` fires to throw the card off screen), set the ref **before** the animation starts:

```typescript
const handleSwipe = (direction: 'left' | 'right', cardId: string) => {
  // Mark this card as swiping out BEFORE the animation runs
  swipingOutId.current = cardId;

  const toValue = direction === 'right' ? 500 : -500;

  Animated.timing(position, {
    toValue: { x: toValue, y: 0 },
    duration: 250,
    useNativeDriver: true, // ← MUST be true
  }).start(() => {
    // Animation is fully complete — now safe to remove the card
    InteractionManager.runAfterInteractions(() => {
      setCards(prev => prev.filter(card => card.id !== cardId));
      swipingOutId.current = null;
    });
  });
};
```

**Important:** If the current code uses `Animated.decay` or `Animated.spring` for the throw animation, keep the same easing but add `useNativeDriver: true` and move the card-removal logic into the `.start()` callback as shown above.

---

## Part 4 — Hide the Card During Exit Instead of Removing It Immediately

In the card render function, if a card's ID matches `swipingOutId.current`, render it with `opacity: 0` and `pointerEvents: 'none'` so it's invisible but still mounted during the animation:

```typescript
const renderCard = (card: CardType, index: number) => {
  const isSwiping = swipingOutId.current === card.id;

  return (
    <Animated.View
      key={card.id}
      style={[
        styles.cardContainer,
        { zIndex: cards.length - index },
        isSwiping && {
          opacity: 0,
          pointerEvents: 'none',
        },
      ]}
    >
      {/* card content */}
    </Animated.View>
  );
};
```

This prevents the flash — the card is invisible as soon as the swipe gesture is committed, and React never gets a chance to show it at position 0.

---

## Part 5 — Ensure useNativeDriver: true on All Animated Values

Search the swipe component for every `Animated.timing`, `Animated.spring`, and `Animated.decay` call. Every single one must have `useNativeDriver: true`. Missing this is a common cause of animation jank on the JS thread.

```typescript
// Every animation call should look like this:
Animated.spring(position, {
  toValue: { x: 0, y: 0 },
  useNativeDriver: true, // ← required on every call
  friction: 5,
}).start();
```

If you find any animation using a property that doesn't support the native driver (like `width`, `height`, or `backgroundColor`), split it into a separate `Animated.Value` that runs on the JS thread, keeping position/opacity/transform on the native driver.

---

## Part 6 — Fix the Interpolation Clamp

If the card's rotation or opacity is interpolated from the position value, add `extrapolate: 'clamp'` to prevent the values from snapping outside their range when the animation resets:

```typescript
const rotate = position.x.interpolate({
  inputRange: [-200, 0, 200],
  outputRange: ['-30deg', '0deg', '30deg'],
  extrapolate: 'clamp', // ← add this
});

const cardOpacity = position.x.interpolate({
  inputRange: [-200, -100, 0, 100, 200],
  outputRange: [0, 1, 1, 1, 0],
  extrapolate: 'clamp', // ← add this
});
```

---

## Part 7 — Reset the Animated Value After Card Removal

After the card is removed from the array, reset the `Animated.Value` back to `{ x: 0, y: 0 }` **without animation** so the next card starts from a clean position:

```typescript
InteractionManager.runAfterInteractions(() => {
  setCards(prev => prev.filter(card => card.id !== cardId));
  swipingOutId.current = null;

  // Reset position for the next card immediately (no animation)
  position.setValue({ x: 0, y: 0 });
});
```

---

## Part 8 — If Using a Third-Party Swipe Library

If the app uses `react-native-deck-swiper`, `react-native-swipe-cards`, or similar:

### For `react-native-deck-swiper`:
```typescript
<Swiper
  ref={swiperRef}
  cards={cards}
  renderCard={renderCard}
  onSwipedLeft={(index) => handleSwiped(index, 'left')}
  onSwipedRight={(index) => handleSwiped(index, 'right')}
  cardIndex={0}
  backgroundColor="transparent"
  stackSize={3}
  stackSeparation={12}
  animateCardOpacity        // ← enables opacity fade on swipe
  swipeAnimationDuration={200}  // ← shorter = snappier, less time for flash
  useViewOverflow={false}   // ← prevents overflow rendering artifacts
  overlayOpacityHorizontalThreshold={10}
  overlayOpacityVerticalThreshold={10}
/>
```

The key settings that fix the flash: `animateCardOpacity`, shorter `swipeAnimationDuration`, and `useViewOverflow={false}`.

### For custom gesture handler implementations:
Make sure `GestureHandlerRootView` wraps the entire app (in `App.tsx`) and that `react-native-reanimated` is used instead of the built-in `Animated` API if the project already has it installed — Reanimated runs entirely on the UI thread and eliminates this class of bug entirely.

---

## Summary

| Root Cause | Fix |
|---|---|
| State update mid-animation triggers re-render | Move card removal into `.start()` callback |
| Card briefly visible at position 0 on re-render | Set `opacity: 0` via `swipingOutId` ref before animation |
| JS thread delays | `useNativeDriver: true` on every Animated call |
| Values snap outside range | `extrapolate: 'clamp'` on all interpolations |
| Position not reset cleanly | `position.setValue({ x: 0, y: 0 })` after removal |

After these fixes, the swipe should feel instant — the card flies off screen in one clean motion with no flash, no snap-back, no hesitation.
