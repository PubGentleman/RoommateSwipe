# Rhome — Host Plans Include Renter Access (Bundled Pricing)
## Replit Agent Prompt

---

## Overview

Users should never need two active subscriptions at the same time. Host plans already cost significantly more than renter plans, so renter features are bundled into every host subscription automatically. This prompt updates the entitlement checks, paywall logic, and pricing screens to reflect this.

**The rule:**
- Paying for any **host plan** → renter features included at the appropriate tier (no second subscription)
- Paying for a **renter plan only** → renter features only (must upgrade to host plan to get listings)
- Switching from renter to host → renter subscription is cancelled, host plan takes over
- Agents and companies → host-only accounts, no renter subscription ever shown to them

**Renter tier bundled by host plan level:**

| Host Plan | Renter Features Included |
|---|---|
| Host Starter ($29/mo) | Renter Plus |
| Host Pro ($59/mo) | Renter Elite |
| Host Business ($99/mo) | Renter Elite |
| Agent Starter ($79/mo) | N/A — agents don't use renter mode |
| Agent Pro ($149/mo) | N/A |
| Agent Business ($249/mo) | N/A |
| Company any tier | N/A |

## Part 1 — Entitlement Logic

### Update `utils/revenueCatService.ts`

Replace the current entitlement check logic with a function that returns both the host entitlement AND the derived renter entitlement:

```typescript
export type RenterTier = 'free' | 'plus' | 'elite';
export type HostTier = 'none' | 'starter' | 'pro' | 'business' | 'agent_starter' | 'agent_pro' | 'agent_business' | 'company_starter' | 'company_pro';

export interface UserEntitlements {
  hostTier: HostTier;
  renterTier: RenterTier;
  hasActiveSubscription: boolean;
  subscriptionSource: 'renter' | 'host' | 'none';
}

export async function getUserEntitlements(): Promise<UserEntitlements> {
  try {
    const customerInfo = await Purchases.getCustomerInfo();
    const active = customerInfo.entitlements.active;

    // Check host entitlements first
    if (active['host_business']) {
      return { hostTier: 'business', renterTier: 'elite', hasActiveSubscription: true, subscriptionSource: 'host' };
    }
    if (active['host_pro']) {
      return { hostTier: 'pro', renterTier: 'elite', hasActiveSubscription: true, subscriptionSource: 'host' };
    }
    if (active['host_starter']) {
      return { hostTier: 'starter', renterTier: 'plus', hasActiveSubscription: true, subscriptionSource: 'host' };
    }

    // Agent tiers — no renter access
    if (active['host_agent_business']) {
      return { hostTier: 'agent_business', renterTier: 'free', hasActiveSubscription: true, subscriptionSource: 'host' };
    }
    if (active['host_agent_pro']) {
      return { hostTier: 'agent_pro', renterTier: 'free', hasActiveSubscription: true, subscriptionSource: 'host' };
    }
    if (active['host_agent_starter']) {
      return { hostTier: 'agent_starter', renterTier: 'free', hasActiveSubscription: true, subscriptionSource: 'host' };
    }

    // Company tiers — no renter access
    if (active['host_company_pro']) {
      return { hostTier: 'company_pro', renterTier: 'free', hasActiveSubscription: true, subscriptionSource: 'host' };
    }
    if (active['host_company_starter']) {
      return { hostTier: 'company_starter', renterTier: 'free', hasActiveSubscription: true, subscriptionSource: 'host' };
    }

    // Renter-only entitlements
    if (active['renter_elite']) {
      return { hostTier: 'none', renterTier: 'elite', hasActiveSubscription: true, subscriptionSource: 'renter' };
    }
    if (active['renter_plus']) {
      return { hostTier: 'none', renterTier: 'plus', hasActiveSubscription: true, subscriptionSource: 'renter' };
    }

    return { hostTier: 'none', renterTier: 'free', hasActiveSubscription: false, subscriptionSource: 'none' };
  } catch (e) {
    return { hostTier: 'none', renterTier: 'free', hasActiveSubscription: false, subscriptionSource: 'none' };
  }
}
```

## Part 2 — useEntitlements Hook

Create `hooks/useEntitlements.ts`:

```typescript
import { useState, useEffect } from 'react';
import { getUserEntitlements, UserEntitlements } from '../utils/revenueCatService';

const defaultEntitlements: UserEntitlements = {
  hostTier: 'none',
  renterTier: 'free',
  hasActiveSubscription: false,
  subscriptionSource: 'none',
};

export function useEntitlements() {
  const [entitlements, setEntitlements] = useState<UserEntitlements>(defaultEntitlements);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    getUserEntitlements().then((e) => {
      setEntitlements(e);
      setLoading(false);
    });
  }, []);

  const isRenterPlus = entitlements.renterTier === 'plus' || entitlements.renterTier === 'elite';
  const isRenterElite = entitlements.renterTier === 'elite';
  const isAnyHost = entitlements.hostTier !== 'none';

  return { ...entitlements, isRenterPlus, isRenterElite, isAnyHost, loading };
}
```

Replace all existing `useSubscription`, `usePlan`, or direct entitlement checks throughout the app with `useEntitlements()`.

## Part 5 — Cancel Renter Subscription on Host Upgrade

When an individual renter upgrades to a host plan, their renter subscription should be cancelled to avoid duplicate charges. RevenueCat handles this automatically if the products are in separate subscription groups (which they are — Rhome Renter group vs Rhome Host group). Apple and Google will cancel the old subscription when a new one in a different group starts.

However, add a UI message in the host upgrade confirmation flow:

```typescript
Alert.alert(
  'Upgrade to Host Plan',
  'Your host plan includes renter features — your current renter subscription will be cancelled automatically when your host plan activates. You won\'t be charged twice.',
  [
    { text: 'Cancel', style: 'cancel' },
    { text: 'Upgrade', onPress: () => purchaseHostPlan(plan) },
  ]
);
```

## Summary

No user ever pays two subscriptions at once. Host plans bundle renter access. Agents and companies don't see renter pricing at all. The entitlement system flows from one source of truth — RevenueCat — and propagates to Supabase for server-side checks. Every paywall and pricing screen reads from `useEntitlements()` so the logic is centralized and consistent.
