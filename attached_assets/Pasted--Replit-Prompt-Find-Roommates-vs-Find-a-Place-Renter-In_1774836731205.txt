# Replit Prompt: Find Roommates vs. Find a Place — Renter Intent & Listing Types

## GOAL

Add a **renter intent system** that asks the right question upfront: "What do you need?" Instead of forcing everyone through a roommate-centric flow, the first screen branches based on what the person actually needs. Someone who just wants a 1-bedroom for themselves gets a fast, clean path. Someone who needs roommates gets the full matching experience. Someone who already has their group just needs to find a place. Hosts tag their listings as Room or Entire Apartment. The Explore screen auto-filters based on the renter's intent.

**After tapping:**
- If they picked **"Find Roommates"** → 150ms delay → transition to **Find Roommates follow-up** (see below)
- If they picked **"Find a Place"** → 150ms delay → transition to **Find a Place follow-up** (see below)

## DATABASE CHANGES

### Profile Table Updates

```sql
-- What the renter is looking for: room, entire apartment, or has a group
ALTER TABLE public.profiles
  ADD COLUMN IF NOT EXISTS listing_type_preference TEXT DEFAULT 'room';
  -- Values: 'room', 'entire_apartment', 'have_group'
  -- Default: 'room' (most common use case for a roommate app)

-- Who are they searching with/for?
ALTER TABLE public.profiles
  ADD COLUMN IF NOT EXISTS apartment_search_type TEXT;
  -- Values: 'with_roommates', 'solo', 'partner', NULL
  -- NULL when listing_type_preference = 'room' (not applicable — they're finding roommates for a room)
  -- 'with_roommates' = Find Roommates → Entire Apartment (need to match with people first)
  -- 'solo' = Find a Place → Just Me (apartment finder, no roommates)
  -- 'partner' = Find a Place → Me & Partner (apartment finder for 2, no roommates)

-- Is this person a Group Lead? (the one doing the searching for the group)
ALTER TABLE public.profiles
  ADD COLUMN IF NOT EXISTS is_group_lead BOOLEAN DEFAULT false;
```

### New Tables — Preformed Groups

```sql
-- Groups of friends who already know each other (pre-app)
CREATE TABLE IF NOT EXISTS public.preformed_groups (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT, -- optional group name (e.g., "The Brooklyn Crew")
  group_lead_id UUID REFERENCES public.users(id) ON DELETE CASCADE,
  group_size INTEGER NOT NULL DEFAULT 2, -- total people including lead
  status TEXT NOT NULL DEFAULT 'forming',
    -- 'forming' = lead setting up, inviting friends
    -- 'ready' = all members joined (or lead decided to start searching without everyone)
    -- 'searching' = actively browsing listings
    -- 'applied' = applied to a listing
    -- 'placed' = found a place
  invite_code TEXT UNIQUE NOT NULL, -- 6-8 char alphanumeric code for sharing
  city TEXT,
  preferred_neighborhoods JSONB,
  combined_budget_min NUMERIC,
  combined_budget_max NUMERIC,
  desired_bedroom_count INTEGER,
  move_in_date DATE,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  converted_group_id UUID REFERENCES public.groups(id) -- links to real group once formed
);

CREATE INDEX idx_preformed_groups_lead ON public.preformed_groups(group_lead_id);
CREATE INDEX idx_preformed_groups_code ON public.preformed_groups(invite_code);

-- Members of a preformed group (includes lead + invited friends)
CREATE TABLE IF NOT EXISTS public.preformed_group_members (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  preformed_group_id UUID REFERENCES public.preformed_groups(id) ON DELETE CASCADE,
  user_id UUID REFERENCES public.users(id), -- NULL if friend hasn't signed up yet
  name TEXT NOT NULL, -- entered by Group Lead before friend joins
  status TEXT NOT NULL DEFAULT 'invited',
    -- 'invited' = invite sent, friend hasn't joined yet
    -- 'joined' = friend signed up and joined the group
    -- 'declined' = friend declined the invite
  invited_at TIMESTAMPTZ DEFAULT NOW(),
  joined_at TIMESTAMPTZ,
  invite_link TEXT, -- deep link for this specific member
  UNIQUE(preformed_group_id, user_id)
);

CREATE INDEX idx_preformed_members_group ON public.preformed_group_members(preformed_group_id);
CREATE INDEX idx_preformed_members_user ON public.preformed_group_members(user_id);

-- Shortlisted listings (Group Lead saves listings for the group)
CREATE TABLE IF NOT EXISTS public.group_shortlist (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  preformed_group_id UUID REFERENCES public.preformed_groups(id) ON DELETE CASCADE,
  listing_id UUID REFERENCES public.listings(id) ON DELETE CASCADE,
  added_by UUID REFERENCES public.users(id), -- who shortlisted it (usually the lead)
  notes TEXT, -- optional note ("This one has the big kitchen!")
  vote_count INTEGER DEFAULT 0, -- members can upvote shortlisted listings
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(preformed_group_id, listing_id)
);

CREATE INDEX idx_shortlist_group ON public.group_shortlist(preformed_group_id);
```

## "I ALREADY HAVE A GROUP" — FULL FLOW

This is for renters who already have friends they want to live with. One person (the **Group Lead**) sets up the group and does the apartment searching. Friends join when they're ready — they don't need to be on the app for the Lead to start browsing.

### Step 1: Group Setup Screen (During Onboarding)

**Screen: `GroupSetupScreen`** (`screens/renter/GroupSetupScreen.tsx`)

Shown after the user picks "I Have a Group" on the WhatAreYouLookingFor screen. Same dark theme as previous screens.

**Layout:**
- Headline: **"Set up your group"**
- Subheadline: **"You'll be the Group Lead — you can start searching right away"**

**Fields:**
1. **Group name** (optional) — text input, placeholder: "e.g., The Brooklyn Crew"
2. **How many people total?** — number selector: 2, 3, 4, 5+ (including the lead)
3. **Add your roommates** — for each additional person:
   - First name (required) — just so listings/agents can see group composition
   - That's it. No email required yet. They'll be invited later.
4. **"Continue"** button

**What happens on Continue:**
- Creates a `preformed_groups` record with the lead as `group_lead_id`
- Creates `preformed_group_members` records for each friend (with name, `user_id = NULL`, `status = 'invited'`)
- Creates a member record for the lead themselves (`user_id` set, `status = 'joined'`)
- Generates a unique `invite_code` (6-8 alphanumeric characters)
- Sets `is_group_lead = true` on the lead's profile
- Continues to **lite onboarding**: Photos → Basic Info → Budget & Location (skips dealbreakers, roommate preferences, Tell Pi — they already have their roommates)

### Step 2: Group Lead Browses Listings

After onboarding, the Group Lead lands in the app with their Explore screen **auto-filtered to "Entire Apartments"** (since a group needs a full unit, not a single room).

**The Group Lead experience:**
- Explore shows entire apartment listings sized for their group (e.g., 3-person group → shows 2BR and 3BR listings)
- Listings are filtered by the Lead's budget & location preferences
- The Lead can **save/shortlist** listings for the group (heart icon or "Save for Group" button)
- Saved listings appear in a **"Group Shortlist"** tab (new tab in the group screen)
- The Lead can add notes to shortlisted listings ("This one has the big kitchen!")
- **No swipe deck** — the Lead doesn't need roommate matching since they already have their people

**The Lead sees a persistent banner** at the top of their home/explore screen:
```
"👥 Your group: [Name1], [Name2] — Invite them to join!"
[Send Invites] button
```

### Step 3: Invite Friends to Join

The Group Lead can invite friends at any time (during onboarding or later from the group screen).

**Invite methods:**
1. **Share invite link** — a universal deep link that works on iOS (TestFlight/App Store) and eventually Android
   - Format: `https://rhome.app/join/[invite_code]` or deep link: `rhome://join/[invite_code]`
   - Share via: native share sheet (iMessage, WhatsApp, etc.)
   - One link per group (not per member) — anyone with the link can join

2. **Copy invite code** — the 6-8 character code they can text to friends
   - Friends enter the code during signup or in the app

**Screen: `InviteFriendsScreen`** (`screens/renter/InviteFriendsScreen.tsx`)

- Headline: **"Invite your roommates"**
- Group name + member list showing who's joined and who hasn't:
  - ✅ "You (Group Lead)" — joined
  - ⏳ "Sarah" — invited, hasn't joined yet
  - ⏳ "Mike" — invited, hasn't joined yet
- **Big "Share Invite Link" button** → opens native share sheet
- **"Copy Code: ABCD1234"** — tap to copy
- **"Skip for now"** link — they can invite later from the group screen
- Helper text: "Your friends will sign up through the link and automatically join your group. You can start searching for apartments right away!"

### Step 4: Friend Joins Through Invite Link

When a friend taps the invite link (or enters the code):

**If they DON'T have the app:**
- Link goes to App Store / TestFlight download page
- After install + signup, the invite code is preserved (via deferred deep link or onboarding prompt: "Have an invite code?")
- They're auto-added to the group after creating their account

**If they already have the app but no account:**
- Opens the app → signup flow
- After account creation: "You've been invited to join [Group Name]!"
- Shows the group details: Lead's name, other members, what they're looking for
- **Accept / Decline** buttons
- Accept → runs lite onboarding (photos, basicInfo, budget & location only — no roommate matching stuff)
- They land in the app as a group member

**If they already have an account:**
- Opens the app → "You've been invited to join [Group Name]!"
- Same Accept / Decline flow
- On accept: their `listing_type_preference` switches to `'have_group'`, they're added to the group
- If they were previously in the matching pool, they're removed from swipe decks and Pi auto-matching

**On join:**
- Update `preformed_group_members` record: set `user_id`, `status = 'joined'`, `joined_at = NOW()`
- Send push notification to Group Lead: "[Name] joined your group!"
- If all members have joined → update group status to `'ready'`
- Group Lead does NOT have to wait for all members to start searching — they can browse immediately

### Step 5: Group Experience (After Friends Join)

Once friends join, they can all see the group and participate:

**Screen: `MyGroupScreen`** (`screens/renter/MyGroupScreen.tsx`)

- Header: group name (or "Your Group")
- Member list with photos + names + "Lead" badge on the Group Lead
- **Group Shortlist tab** — listings the Lead (or any member) saved
  - Each listing card shows: photo, price, bedrooms, neighborhood, notes
  - Members can **upvote** listings they like (👍 count shown)
  - Members can add their own notes
  - Any member can add listings to the shortlist (not just the Lead)
- **Group Chat tab** — group messaging (uses existing chat system)
- **Group Settings:**
  - Edit group name
  - Group Lead can remove members
  - Any member can leave the group
  - Group Lead can invite more people
  - Desired bedrooms, budget range, neighborhoods (editable by Lead)

**All group members:**
- See Explore auto-filtered to "Entire Apartments"
- Can save listings to the group shortlist
- Can browse independently but the shortlist is shared

**Group Lead specific:**
- Sees "Lead" badge on their profile within the group
- Can edit group preferences (budget, bedrooms, neighborhoods)
- Can contact agents/hosts on behalf of the group
- When applying to a listing: application shows it's from a group of X people

### How Preformed Groups Appear to Hosts/Agents

Preformed groups look the same as Pi auto-matched groups from the host/agent perspective:

- In the agent marketplace (PiMatchedGroupsScreen), preformed groups also appear
- Badge: "Pre-formed Group" (vs. "Pi Matched" for auto-assembled groups)
- Package includes: group size, combined budget, neighborhoods, bedrooms, move-in date
- Agent claims the group the same way
- Difference: no compatibility score shown (they chose each other, so it's implicit)

**Database distinction:**
- `preformed_groups.converted_group_id` links to the same `groups` table that Pi auto-groups convert into
- Once a preformed group has all members joined, create a corresponding `groups` record so the rest of the app (chat, agent marketplace) works uniformly

## EXPLORE SCREEN — AUTO-FILTERING

### How It Works

When a renter opens the Explore/Discover screen, the app automatically filters listings based on their `listing_type_preference`.

**Default behavior based on renter intent:**
- **Find Roommates → A Room** → Explore shows only `listing_type = 'room'` listings
- **Find Roommates → Entire Apartment** → Explore shows only `listing_type = 'entire_apartment'` listings
- **Find a Place → Just Me** → Explore shows only `listing_type = 'entire_apartment'` listings
- **Find a Place → Me & Partner** → Explore shows only `listing_type = 'entire_apartment'` listings (1BR+)
- **Find a Place → My Group** → Explore shows only `listing_type = 'entire_apartment'` listings (sized for group)

**Filter bar update:**
Add a **toggle or segmented control** at the top of the Explore screen:

```
[ Rooms ] [ Entire Apartments ] [ All ]
```

- The renter's preference is pre-selected (highlighted) when they open Explore
- They can tap the other option to switch views at any time
- "All" shows both room and entire apartment listings
- This filter persists during the session but resets to their preference when they reopen the app
- This filter works alongside existing filters (city, neighborhood, budget, bedrooms, etc.)

**Implementation in the Explore/Discover screen:**

```typescript
// In the Explore screen component:
const [listingTypeFilter, setListingTypeFilter] = useState<'room' | 'entire_apartment' | 'all'>(
  currentUser.listing_type_preference || 'room'
);

// When querying listings, add to the Supabase filter:
let query = supabase.from('listings').select('*');
if (listingTypeFilter !== 'all') {
  query = query.eq('listing_type', listingTypeFilter);
}
```

**Visual differentiation:**
- Room listings: show a "Room" badge/tag on the card
- Entire apartment listings: show an "Entire Apt" badge/tag on the card
- Different card accent color or icon so renters can instantly tell the type

## IMPACT ON MATCHING & PI

### Roommate Matching (Swipe Deck)

- **Find Roommates → A Room:** Normal roommate matching (find compatible people to live with in a shared place)
- **Find Roommates → Entire Apartment:** Roommate matching (find a group first, then find an apartment together)
- **Find a Place → Just Me:** **NO roommate matching.** Skip swipe deck. Explore shows entire apartment listings only. Pure apartment finder.
- **Find a Place → Me & Partner:** **NO roommate matching.** Skip swipe deck. Explore shows entire apartment listings (1BR+). Profile indicates 2 occupants for hosts/agents.
- **Find a Place → My Group:** **NO roommate matching.** They already have their people. Skip swipe deck. Explore shows entire apartments.

**For Pi Auto-Matching:**
- Pi only assembles groups from renters on the **"Find Roommates"** path (either Room or Entire Apartment)
- "Just Me," "Me & Partner," and "My Group" renters are ALL excluded from Pi auto-matching
- They do NOT appear in other renters' swipe decks

### Host Matching

- Individual hosts listing "A Room" → matched with renters looking for rooms
- Individual hosts listing "Entire Apartment" → matched with confirmed roommate groups (from Pi auto-matching or manual groups) OR solo renters looking for entire apartments
- Agents/companies can list both types and get matched accordingly

## SETTINGS — CHANGE PREFERENCE

### In EditProfileScreen or Settings

Add a section where renters can change their intent after signup:

- Section: "What I Need"
- Current selection shown (e.g., "Finding roommates — looking for a room" or "Finding a place — just me")
- Tap to change → walks through the same 2-step selection as signup
- **Switching FROM "Find Roommates" TO "Find a Place":**
  - Remove them from any pending Pi auto-groups
  - Remove them from other renters' swipe decks
  - Show warning: "This will remove you from roommate matching. Continue?"
- **Switching FROM "Find a Place" TO "Find Roommates":**
  - Add them back to the matching pool
  - Show: "You're now in the roommate matching pool. Pi will start looking for your roommates!"
- **Switching FROM "My Group" to anything else:**
  - If they're the Group Lead: show warning "Leaving your group will dissolve it for everyone. Are you sure?"
  - If they're a member (not lead): remove them from the preformed group, notify the Lead
- **Switching FROM anything TO "My Group":**
  - Remove from matching pool / Pi auto-groups if applicable
  - Redirect to Group Setup Screen
- **Switching between "Just Me" and "Me & Partner":**
  - No warning needed — just updates the profile. Both are on the "Find a Place" fast lane.

## MIGRATION FOR EXISTING USERS

```sql
-- Default all existing renters to 'room' (the app's original purpose)
UPDATE public.profiles
SET listing_type_preference = 'room'
WHERE listing_type_preference IS NULL
  AND user_type = 'renter';

-- Default all existing listings to 'room'
UPDATE public.listings
SET listing_type = 'room'
WHERE listing_type IS NULL;
```

For existing renters who haven't set this preference: show the `WhatAreYouLookingForScreen` the next time they open the app (one-time prompt, like a migration screen).
