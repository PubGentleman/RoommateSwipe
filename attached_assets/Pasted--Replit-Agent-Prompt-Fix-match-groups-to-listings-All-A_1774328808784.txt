# Replit Agent Prompt — Fix match-groups-to-listings + All AI Functions for Rooms Available

## Context
Rhome is a React Native / Expo (TypeScript) roommate matching app with a Supabase backend. The current system calculates available rooms as `bedrooms - (host_lives_in ? 1 : 0)` but does not account for existing roommates already living in the unit who are not in the system. This prompt fixes the `listings` table, the listing creation form, and every AI Edge Function that uses bedroom/rooms data to ensure they all use the correct `rooms_available` value.

---

## What To Build

### 1. Database Migration — `supabase/migrations/023_rooms_available.sql`

```sql
-- Add existing_roommates_count to listings
ALTER TABLE listings
  ADD COLUMN IF NOT EXISTS existing_roommates_count INTEGER NOT NULL DEFAULT 0,
  ADD COLUMN IF NOT EXISTS rooms_available INTEGER GENERATED ALWAYS AS (
    bedrooms - CASE WHEN host_lives_in THEN 1 ELSE 0 END - existing_roommates_count
  ) STORED;

-- Ensure rooms_available never goes below 1
ALTER TABLE listings ADD CONSTRAINT rooms_available_positive CHECK (
  (bedrooms - CASE WHEN host_lives_in THEN 1 ELSE 0 END - existing_roommates_count) >= 1
);

-- Add index for fast group size filtering
CREATE INDEX IF NOT EXISTS idx_listings_rooms_available ON listings(rooms_available);
```

Run this migration in the Supabase SQL Editor.

### 3. Fix `match-groups-to-listings` Edge Function

In `supabase/functions/match-groups-to-listings/index.ts`, update the group filtering to use `rooms_available` instead of calculating from bedrooms:

Find the section where listings are fetched and groups are matched. Replace any bedroom calculation with:

```typescript
// Fetch listings with rooms_available (computed column)
const { data: listings } = await supabase
  .from('listings')
  .select(`
    id, address, neighborhood, price, bedrooms, rooms_available,
    existing_roommates_count, host_lives_in, available_date,
    amenities, preferred_trains, host_id
  `)
  .eq('is_active', true)
  .gt('rooms_available', 0) // Only listings with open rooms

// When matching groups to listings, filter by group size = rooms_available
const compatibleListings = listings.filter(listing =>
  group.member_count === listing.rooms_available
)

// If no exact match, allow ±1 flexibility (e.g. 2-person group for 3-room unit
// where one room can stay empty — host's discretion)
const flexibleMatches = listings.filter(listing =>
  group.member_count >= listing.rooms_available - 1 &&
  group.member_count <= listing.rooms_available
)
```

Replace anywhere in the file that calculates rooms as `listing.bedrooms - 1` or `listing.bedrooms - (host_lives_in ? 1 : 0)` with `listing.rooms_available`.

### 9. Add Validation to Listing Save Logic

When the host saves a listing (in the API call or Edge Function that handles listing creation), add a server-side validation:

```typescript
// Validate that rooms_available will be >= 1
const { bedrooms, host_lives_in, existing_roommates_count } = listingData
const roomsAvailable = bedrooms - (host_lives_in ? 1 : 0) - (existing_roommates_count || 0)

if (roomsAvailable < 1) {
  return new Response(
    JSON.stringify({ error: 'Listing must have at least 1 room available for renters.' }),
    { status: 400 }
  )
}
```
