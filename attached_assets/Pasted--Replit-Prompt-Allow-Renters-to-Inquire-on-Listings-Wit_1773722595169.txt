### Update `handleSendInquiry` to support solo inquiries

Find the `handleSendInquiry` function in `ExploreScreen.tsx`.

Update it so that when `roommateGroups.length === 0` (no group), it sends the inquiry using the renter's individual profile instead of a group ID.

The function should:
1. If `selectedGroupId` is set — use it as the group for the inquiry (existing behavior)
2. If `selectedGroupId` is null AND `roommateGroups.length === 0` — proceed with a solo inquiry using the current user's ID as the "group" identifier

```tsx
const handleSendInquiry = async () => {
  if (!inquireProperty || !user) return;

  // Solo inquiry: no group exists, use individual renter
  const isSoloInquiry = roommateGroups.length === 0 && !selectedGroupId;

  if (!isSoloInquiry && !selectedGroupId) return; // has groups but none selected

  setCreatingInquiry(true);
  try {
    const groupIdToUse = isSoloInquiry ? `solo-${user.id}` : selectedGroupId;

    // Create the inquiry record
    await StorageService.createInquiry({
      propertyId: inquireProperty.id,
      groupId: groupIdToUse,
      renterId: user.id,
      renterName: user.name || 'Renter',
      renterPhoto: user.profilePhoto || '',
      personalNote: '',
      isSoloInquiry,
      status: 'pending',
      createdAt: new Date().toISOString(),
    });

    setShowInquireModal(false);
    setSelectedGroupId(null);

    Alert.alert(
      'Inquiry Sent!',
      isSoloInquiry
        ? 'The host has been notified. You\'ll hear back in your Messages tab.'
        : 'Your group\'s inquiry has been sent. The host will respond in your Messages tab.',
      [{ text: 'OK' }]
    );
  } catch (error) {
    Alert.alert('Error', 'Failed to send inquiry. Please try again.');
  } finally {
    setCreatingInquiry(false);
  }
};
```

## WHAT THIS DOES

- Renters with **no group** see a "Inquiring as an Individual" info card and can send immediately — no friction
- Renters **with a group** still pick which group to use (existing behavior, unchanged)
- The inquiry is created either way — host sees it in their Inquiries screen
- Solo inquiries are flagged with `isSoloInquiry: true` so the host can see the context
- Success alert message is personalized based on solo vs group inquiry
