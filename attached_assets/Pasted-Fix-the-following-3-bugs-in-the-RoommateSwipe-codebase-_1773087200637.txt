BUG 2 — Work Style Tag Reads from Wrong Field
In utils/matchingAlgorithm.ts, the function getWorkStyleTag() reads from lifestyle.workSchedule which is a free-text field with inconsistent values. The correct structured field is profileData.preferences.workLocation which uses the typed enum 'wfh_fulltime' | 'hybrid' | 'office_fulltime' | 'irregular'.
Update getWorkStyleTag() to accept the structured enum and map it correctly. Replace the existing function with:
tsexport function getWorkStyleTag(workLocation?: string): string | null {
  if (!workLocation) return null;
  const map: Record<string, string> = {
    wfh_fulltime: 'Remote',
    hybrid: 'Hybrid',
    office_fulltime: 'Office',
    irregular: 'Flexible',
  };
  return map[workLocation] ?? null;
}
