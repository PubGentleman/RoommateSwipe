Add Recurring Subscriptions, 3-Month Billing Option & Cancellation Flow
Implement fully recurring subscriptions with auto-renewal, add a 3-month billing option to both the renter and host subscription screens, and build a clear cancellation flow.
PART 1 — RECURRING BILLING WITH STRIPE
All paid plans must auto-renew using Stripe recurring subscriptions. Never use one-time payments for subscription plans. Set up the following Stripe price objects for each plan:
Renter Plus: price_plus_monthly ($14.99/mo), price_plus_3month ($40.47 every 3 months), price_plus_annual ($179.88/yr)
Renter Elite: price_elite_monthly ($29.99/mo), price_elite_3month ($80.97 every 3 months), price_elite_annual ($359.88/yr)
Host Pro: price_pro_monthly ($29.99/mo), price_pro_3month ($80.97 every 3 months), price_pro_annual ($359.88/yr)
Host Business: price_biz_monthly ($79.99/mo), price_biz_3month ($215.97 every 3 months), price_biz_annual ($959.88/yr)
For 3-month prices use Stripe's recurring.interval: month with recurring.interval_count: 3. For annual use recurring.interval: year. All plans set recurring.usage_type: licensed.
When the user taps the upgrade button, pass the correct Stripe price ID based on which plan and billing cycle they selected. Attach the Stripe customer_id to the user profile on first purchase so future billing and cancellations are tied to their account.
PART 2 — 3-MONTH BILLING TOGGLE
On both the renter and host subscription screens, expand the billing toggle from 2 options to 3: Monthly · 3 Months · Annual. All three buttons sit inside the existing toggle container with equal widths. Active option gets the coral gradient (#ff6b5b → #e83a2a). Inactive options are transparent with muted text.
When 3 Months is selected, update all plan card prices to show the discounted per-month rate with a note beneath:
Renter Plus: "$13.49/mo" — note: "Billed $40.47 every 3 months · Save 10%"
Renter Elite: "$26.99/mo" — note: "Billed $80.97 every 3 months · Save 10%"
Host Pro: "$26.99/mo" — note: "Billed $80.97 every 3 months · Save 10%"
Host Business: "$71.99/mo" — note: "Billed $215.97 every 3 months · Save 10%"
The tier comparison strip at the top must also update its displayed price when 3-month is selected. Show a coral badge on each paid tile that reads "SAVE 10%" when 3-month is active.
The Plus plan CTA button reads "Start 7-Day Free Trial →" on all billing cycles. Elite and host plan CTAs read "Upgrade — Billed Every 3 Months →" when 3-month is selected, "Upgrade — Billed Monthly →" on monthly, and "Upgrade — Billed Annually →" on annual, so users always know exactly what they are committing to before tapping.
PART 3 — FREE TRIAL
The 7-day free trial on Renter Plus requires a valid payment method upfront. Use Stripe's trial_period_days: 7 on the Plus subscription. The user enters their card but is not charged until day 8. Send a push notification and email 24 hours before the trial ends: "Your free trial ends tomorrow — you'll be charged $[price] unless you cancel." If no card is added, do not allow the trial to start.
PART 4 — CANCELLATION FLOW
On the Profile screen, inside Account Settings, add a Manage Subscription row. Tapping it opens a screen showing:

Current plan name and billing cycle
Next renewal date and amount (e.g., "Renews Dec 9, 2026 · $14.99")
Billing history (last 3 charges with date and amount)
A Cancel Plan button at the bottom in muted red

When the user taps Cancel Plan, show a confirmation bottom sheet — do not cancel immediately. The sheet must show: "Are you sure? You'll keep access to [Plan Name] until [end of current period date]." Two buttons: Keep My Plan (coral, primary) and Cancel Anyway (muted outline). This order is intentional — make it slightly harder to cancel by putting the confirm action first.
If they confirm cancellation, call Stripe's subscription.cancel_at_period_end: true — do NOT cancel immediately. The user keeps full access until the period ends. Update the subscription status in the database to cancelling and show a banner on their profile: "Your plan ends on [date]. Resubscribe anytime."
PART 5 — WEBHOOK HANDLING
Set up Stripe webhooks to handle these events: invoice.payment_succeeded (extend subscription period in DB), invoice.payment_failed (send payment failure push notification and email, show in-app banner prompting card update), customer.subscription.deleted (downgrade user to free tier in DB), customer.subscription.trial_will_end (send 24-hour trial ending reminder).
PART 6 — RENEWAL REMINDER
3 days before any subscription renews, send a push notification: "Your [Plan] renews in 3 days for $[amount]. Manage in Settings." This applies to all billing cycles — monthly, 3-month, and annual.
