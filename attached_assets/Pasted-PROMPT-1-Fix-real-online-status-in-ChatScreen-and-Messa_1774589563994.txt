Fix the fake online status in the chat screens. Currently Math.random() is used in three places to fake online/offline status — this needs to use real data from the `last_active_at` field in the Supabase `users` table.

   TO:   const [isOnline, setIsOnline] = useState(false);

2. In the loadMessages function, after setting `otherUser`, add a Supabase lookup to get real online status:
   After the block that sets `otherUser` (around line 518-530), add:

   // Fetch real online status for other user
   if (otherUser || conversation?.participant?.id) {
     const participantId = otherUser?.id || conversation?.participant?.id;
     if (participantId) {
       try {
         const { data: participantData } = await supabase
           .from('users')
           .select('last_active_at')
           .eq('id', participantId)
           .maybeSingle();
         if (participantData?.last_active_at) {
           const lastActive = new Date(participantData.last_active_at);
           const thirtyMinsAgo = new Date(Date.now() - 30 * 60 * 1000);
           setIsOnline(lastActive > thirtyMinsAgo);
         } else {
           setIsOnline(false);
         }
       } catch (_e) {
         setIsOnline(false);
       }
     }
   }
