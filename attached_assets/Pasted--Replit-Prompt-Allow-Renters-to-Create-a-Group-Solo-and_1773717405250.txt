# Replit Prompt: Allow Renters to Create a Group Solo and Look for Roommates to Join

---

## OVERVIEW

Renters should be able to create a group with just themselves in it. They can then browse listings, send inquiries, and look for other renters to join their group — all without needing to have roommates lined up first. This is a key use case for renters who are actively apartment hunting while also searching for roommates at the same time.

The `handleInviteToGroup` function should:
1. Send a group join request notification to the matched renter
2. Show a confirmation: "Invite sent to [Name] to join your group!"
3. The invited renter sees the request in their Groups tab under "Pending Invites"

```tsx
const handleInviteToGroup = async (profile: RoommateProfile) => {
  if (!userGroup || !user) return;
  try {
    await StorageService.sendGroupInvite({
      groupId: userGroup.id,
      groupName: userGroup.name,
      invitedUserId: profile.id,
      invitedByUserId: user.id,
      invitedByName: user.name || 'Someone',
      createdAt: new Date().toISOString(),
    });
    Alert.alert(
      'Invite Sent!',
      `${profile.name} has been invited to join your group.`,
      [{ text: 'OK' }]
    );
  } catch {
    Alert.alert('Error', 'Could not send invite. Try again.');
  }
};
```

## SUMMARY OF FULL FLOW

1. Renter opens Groups tab → taps "Create Group"
2. Fills in name, budget, move-in date (no roommates required)
3. Group is created with just them — shows "Looking for Roommates" banner
4. They can browse listings and inquire immediately as a 1-person group
5. Other renters find them on the Match/Roommates swipe screen
6. After matching, either renter can tap "Invite to My Group"
7. Invited renter sees the invite in their Groups tab and accepts/declines
8. Group grows organically — host sees updated member count on the inquiry
