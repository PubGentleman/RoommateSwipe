# Prompt: Fix Icons Not Showing on Android

## Bug

All Feather icons are blank/invisible on Android. iOS works fine. The app uses `Feather` from `@expo/vector-icons` throughout every screen and component, but `App.tsx` never loads the icon font files at startup. iOS bundles these fonts automatically, but Android requires them to be explicitly loaded via `useFonts` from `expo-font` before they can render. Since `expo-font` is already installed (`~14.0.9`), this is a one-file fix.

---

## The Fix — `App.tsx` only

### Step 1 — Add these two imports at the top of `App.tsx`

```tsx
import { useFonts } from 'expo-font';
import { Feather } from '@expo/vector-icons';
import * as SplashScreen from 'expo-splash-screen';
```

`expo-splash-screen` is already a dependency (used in `app.json`) so no install needed.

---

### Step 2 — Prevent the splash screen from auto-hiding before fonts load

Add this line BEFORE the `App` component function (at module level, outside the component):

```tsx
SplashScreen.preventAutoHideAsync();
```

---

### Step 3 — Load the font inside the `App` component

Inside the `App` function, add the `useFonts` call right at the top, before the `useEffect`:

```tsx
const [fontsLoaded, fontError] = useFonts({
  ...Feather.font,
});
```

---

### Step 4 — Hide the splash screen once fonts are ready

Inside the existing `useEffect` in `App.tsx`, add a call to hide the splash screen after fonts load. Add a new `useEffect` after the font declaration:

```tsx
useEffect(() => {
  if (fontsLoaded || fontError) {
    SplashScreen.hideAsync();
  }
}, [fontsLoaded, fontError]);
```

---

### Step 5 — Block rendering until fonts are loaded

Before the `return` statement in `App`, add an early return:

```tsx
if (!fontsLoaded && !fontError) {
  return null;
}
```

---

## Full Updated `App.tsx`

Here is the complete corrected file:

```tsx
import React, { useEffect } from "react";
import { StyleSheet } from "react-native";
import { NavigationContainer } from "@react-navigation/native";
import { GestureHandlerRootView } from "react-native-gesture-handler";
import { KeyboardProvider } from "react-native-keyboard-controller";
import { SafeAreaProvider } from "react-native-safe-area-context";
import { StatusBar } from "expo-status-bar";
import { useFonts } from 'expo-font';
import { Feather } from '@expo/vector-icons';
import * as SplashScreen from 'expo-splash-screen';

import { RootNavigator } from "./navigation/RootNavigator";
import { ErrorBoundary } from "./components/ErrorBoundary";
import { AuthProvider } from "./contexts/AuthContext";
import { NotificationProvider } from "./contexts/NotificationContext";
import { CityProvider } from "./contexts/CityContext";
import { ProfileReminderProvider } from "./contexts/ProfileReminderContext";
import { StripeWrapper } from "./components/StripeWrapper";
import { StorageService } from "./utils/storage";
import { isDev } from "./utils/dataUtils";
import { checkDailyTrigger } from "./utils/insightRefresh";

SplashScreen.preventAutoHideAsync();

export default function App() {
  const [fontsLoaded, fontError] = useFonts({
    ...Feather.font,
  });

  useEffect(() => {
    if (fontsLoaded || fontError) {
      SplashScreen.hideAsync();
    }
  }, [fontsLoaded, fontError]);

  useEffect(() => {
    if (typeof window !== 'undefined') {
      if (isDev) {
        (window as any).reloadMockData = async () => {
          await StorageService.forceReloadMockData();
          alert('Mock data reloaded! Please refresh the page to see changes.');
        };
        console.log('[App] Dev mode — mock data enabled. Run: window.reloadMockData()');
      } else {
        console.log('[App] Production mode — mock data disabled');
      }
      (window as any).resetApp = async () => {
        await StorageService.clearAllData();
        alert('All app data cleared! Refreshing...');
        window.location.reload();
      };
    }
    checkDailyTrigger();
  }, []);

  if (!fontsLoaded && !fontError) {
    return null;
  }

  return (
    <ErrorBoundary>
      <StripeWrapper>
        <SafeAreaProvider>
          <GestureHandlerRootView style={styles.root}>
            <KeyboardProvider>
              <AuthProvider>
                <CityProvider>
                  <ProfileReminderProvider>
                    <NotificationProvider>
                      <NavigationContainer>
                        <RootNavigator />
                      </NavigationContainer>
                      <StatusBar style="auto" />
                    </NotificationProvider>
                  </ProfileReminderProvider>
                </CityProvider>
              </AuthProvider>
            </KeyboardProvider>
          </GestureHandlerRootView>
        </SafeAreaProvider>
      </StripeWrapper>
    </ErrorBoundary>
  );
}

const styles = StyleSheet.create({
  root: {
    flex: 1,
  },
});
```

---

## Why This Works

- `Feather.font` is an object that maps the font family name to its `.ttf` file path — spreading it into `useFonts` tells Expo to load that font file before anything renders
- `SplashScreen.preventAutoHideAsync()` keeps the splash screen up while the font loads (typically under 100ms) so the user never sees a blank screen
- `SplashScreen.hideAsync()` is called as soon as fonts are ready (or if there's an error, it hides anyway so the app doesn't get stuck)
- The `if (!fontsLoaded && !fontError) return null` guard ensures no screen tries to render icon components before the font is available on Android

## What NOT to Change

- No other files need to be touched
- Do not change any `Feather` usage across screens — the icon names and sizes are all correct
- Do not add `useFonts` to individual screens — loading it once in `App.tsx` makes it available everywhere
- `expo-font` is already in `package.json` — no `npm install` needed
