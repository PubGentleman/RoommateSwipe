# Replit Prompt: Show Sent Inquiries as Chat Threads in Renter's Messages Tab

---

## OVERVIEW

When a renter sends an inquiry on a listing, it should immediately appear as a chat thread in their Messages tab. Right now the inquiry disappears after sending and the renter has no way to track it, follow up, or know when the host responds. This needs to be fixed so every inquiry creates a visible, trackable conversation thread from the moment it's sent.

---

## HOW IT SHOULD WORK

1. Renter sends inquiry on a listing (solo or as a group)
2. A chat thread is created immediately in their Messages tab with status "Pending"
3. The thread shows the listing photo, title, price, and when the inquiry was sent
4. If the host **accepts** → thread becomes a live two-way conversation, status changes to "Accepted"
5. If the host **declines** → thread is archived with "Not available" note, status changes to "Declined"
6. If it's a group inquiry → all group members see the same thread

### 1. `screens/renter/ExploreScreen.tsx` — Create chat thread when inquiry is sent

In the `handleSendInquiry` function, after the inquiry is successfully created, immediately create a conversation/chat thread so it shows up in the renter's Messages tab.

**Find the `handleSendInquiry` function** and add thread creation after the inquiry is saved:

```tsx
const handleSendInquiry = async () => {
  if (!inquireProperty || !user) return;

  const isSoloInquiry = roommateGroups.length === 0 && !selectedGroupId;
  if (!isSoloInquiry && !selectedGroupId) return;

  setCreatingInquiry(true);
  try {
    const groupIdToUse = isSoloInquiry ? `solo-${user.id}` : selectedGroupId;
    const inquiryId = `inquiry-${Date.now()}`;

    // Save the inquiry record
    await StorageService.createInquiry({
      id: inquiryId,
      propertyId: inquireProperty.id,
      groupId: groupIdToUse,
      renterId: user.id,
      renterName: user.name || 'Renter',
      renterPhoto: user.profilePhoto || '',
      personalNote: '',
      isSoloInquiry,
      status: 'pending',
      createdAt: new Date().toISOString(),
    });

    // CREATE CHAT THREAD — so it shows in Messages tab immediately
    const conversationId = `inquiry-conv-${inquiryId}`;
    const systemMessage = {
      id: `msg-${Date.now()}`,
      senderId: 'system',
      text: `You inquired about this listing. Waiting for the host to respond.`,
      content: `You inquired about this listing. Waiting for the host to respond.`,
      timestamp: new Date(),
      read: true,
    };

    const newConversation = {
      id: conversationId,
      inquiryId,
      propertyId: inquireProperty.id,
      listingTitle: inquireProperty.title,
      listingPhoto: inquireProperty.photos?.[0] || '',
      listingPrice: inquireProperty.price,
      hostId: inquireProperty.hostId,
      hostName: inquireProperty.hostName || 'Host',
      hostPhoto: inquireProperty.hostProfilePhoto || '',
      participant: {
        id: inquireProperty.hostId,
        name: inquireProperty.hostName || 'Host',
        photo: inquireProperty.hostProfilePhoto || '',
        online: false,
      },
      lastMessage: 'Inquiry sent — awaiting response',
      lastMessageTime: new Date(),
      unreadCount: 0,
      inquiryStatus: 'pending' as const,  // 'pending' | 'accepted' | 'declined'
      isInquiryThread: true,
      groupId: groupIdToUse,
      isSoloInquiry,
      messages: [systemMessage],
      createdAt: new Date().toISOString(),
    };

    await StorageService.addOrUpdateConversation(newConversation);

    setShowInquireModal(false);
    setSelectedGroupId(null);

    Alert.alert(
      'Inquiry Sent!',
      'Track your inquiry in the Messages tab.',
      [
        {
          text: 'View in Messages',
          onPress: () => (navigation as any).navigate('Messages'),
        },
        { text: 'OK' },
      ]
    );
  } catch (error) {
    Alert.alert('Error', 'Failed to send inquiry. Please try again.');
  } finally {
    setCreatingInquiry(false);
  }
};
```
