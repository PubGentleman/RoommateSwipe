Prompt 1 — Fix Fake Online Status
In screens/renter/RoommatesScreen.tsx, find this line:
javascriptconst isProfileOnline = currentProfile ? Math.random() > 0.5 : false;
Replace it with:
javascriptconst isProfileOnline = currentProfile
  ? (() => {
      const lastActive = currentProfileUser?.last_active_at || currentProfile?.lastActiveAt;
      if (!lastActive) return false;
      const minutesAgo = (Date.now() - new Date(lastActive).getTime()) / 60000;
      return minutesAgo < 30;
    })()
  : false;
This shows the green dot only if the user was active in the last 30 minutes based on real last_active_at data.

Prompt 2 — Apply All Filters When Using Supabase Deck
In screens/renter/RoommatesScreen.tsx, find this line:
javascriptconst filteredProfiles = usedSupabase ? profilesWithCompatibility : applyFiltersToProfiles(profilesWithCompatibility, matchFilters);

javascriptconst filteredProfiles = applyFiltersToProfiles(profilesWithCompatibility, matchFilters);
Always run applyFiltersToProfiles regardless of whether profiles came from Supabase or local storage. This ensures lifestyle, move-in date, search radius, and compatibility filters are always respected.
