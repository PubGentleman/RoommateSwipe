# Rhome — Swipe Screen Performance Fix
## Replit Agent Prompt

---

## Problem Summary

The swipe screen (`screens/renter/RoommatesScreen.tsx`) and `services/discoverService.ts` have 5 performance issues causing slow load times and delayed swipe response:

1. `loadProfiles` fires 3 times on startup and re-fires on every tab focus
2. `getSwipeDeck` makes 3 sequential DB queries that could run in parallel
3. Every swipe awaits 3 DB calls before the animation fires
4. No image prefetching — each new card shows blank while image downloads
5. Compatibility score calculated inline on every render

Fix all 5 issues as described below. Do not change any UI, styles, or logic unrelated to these fixes.

---

## Fix 1 — Stop `loadProfiles` From Firing 3 Times

### File: `screens/renter/RoommatesScreen.tsx`

**Problem:** `loadProfiles` is triggered by:
- A `useEffect` on mount (line ~152)
- A `useEffect` watching `[activeCity, activeSubArea, matchFilters]` (line ~140) — this also fires on mount
- A `useFocusEffect` that fires every time the screen is focused (line ~229) — including when returning from other tabs

This means every tab switch causes a full Supabase deck reload.

**Fix — remove the standalone mount useEffect and add a ref guard to useFocusEffect:**

Find and DELETE this useEffect (it's redundant — the filter watcher already covers it):
```typescript
useEffect(() => {
  loadProfiles();
}, []);
```

Find the `useFocusEffect` block and add a `hasLoadedOnce` ref guard so it only reloads when the user has been away for more than 5 minutes, not on every tab switch:

```typescript
// Add this ref near the top of the component with other refs
const lastLoadTime = useRef<number>(0);
const RELOAD_THRESHOLD_MS = 5 * 60 * 1000; // 5 minutes

// Replace the existing useFocusEffect with this:
useFocusEffect(
  React.useCallback(() => {
    const now = Date.now();
    const shouldReload = now - lastLoadTime.current > RELOAD_THRESHOLD_MS;

    if (shouldReload) {
      console.log('[RoommatesScreen] Screen focused, reloading profiles (threshold met)');
      loadProfiles();
      lastLoadTime.current = now;
    }

    if (user) {
      (async () => {
        try {
          const supabaseGroups = await getMyGroupsFromSupabase('roommate');
          if (supabaseGroups && supabaseGroups.length > 0) {
            const mapped = supabaseGroups.map((g: any) => ({
              id: g.id,
              name: g.name,
              members: (g.group_members || []).map((m: any) => m.user_id),
              maxMembers: g.max_members || 10,
              createdBy: g.created_by,
            }));
            const openGroup = mapped.find((g: any) =>
              g.createdBy === user.id && g.members.length < g.maxMembers
            );
            setUserOpenGroup(openGroup || null);
            return;
          }
        } catch {}
        try {
          const groups = await StorageService.getGroups();
          const openGroup = groups.find(g =>
            g.createdBy === user.id &&
            g.members.includes(user.id) &&
            g.members.length < g.maxMembers
          );
          setUserOpenGroup(openGroup || null);
        } catch {}
      })();
    }

    return () => {
      dispatchInsightTrigger('swipe_session_end');
    };
  }, [activeCity, user])
);
```

Also update the `loadProfiles` function to record when it runs. Find the line `setIsLoading(true)` at the top of `loadProfiles` and add:

## Fix 2 — Parallelize the 3 Sequential DB Calls in `getSwipeDeck`

### File: `services/discoverService.ts`

**Problem:** `getSwipeDeck` runs 3 queries in sequence:
1. `blocked_users` query
2. `interest_cards` query
3. Main `users + profiles + boosts` query

Queries 1 and 2 could run in parallel, cutting the wait time roughly in half.

**Fix — wrap the first two queries in `Promise.all`:**

Find the `getSwipeDeck` function and replace the sequential blocked/swiped lookups:

```typescript
// BEFORE (sequential):
const { data: blockedIds } = await supabase
  .from('blocked_users')
  .select('blocked_id')
  .eq('blocker_id', user.id);
const blocked = (blockedIds || []).map(b => b.blocked_id);

const { data: alreadySwiped } = await supabase
  .from('interest_cards')
  .select('recipient_id')
  .eq('sender_id', user.id);
const swiped = (alreadySwiped || []).map(s => s.recipient_id);

// AFTER (parallel):
const [blockedResult, swipedResult] = await Promise.all([
  supabase.from('blocked_users').select('blocked_id').eq('blocker_id', user.id),
  supabase.from('interest_cards').select('recipient_id').eq('sender_id', user.id),
]);
const blocked = (blockedResult.data || []).map((b: any) => b.blocked_id);
const swiped = (swipedResult.data || []).map((s: any) => s.recipient_id);
```

## Fix 3 — Fire Swipe Animation Instantly, Do DB Writes in Background

### File: `services/discoverService.ts`

**Problem:** `sendLike` awaits `incrementUsage` (2 DB calls) and a match check query before returning. The UI waits for all of this before the swipe animation completes, making swipes feel laggy.

**Fix — make `incrementUsage` and the match check fire-and-forget:**

Replace the `sendLike` function with:

```typescript
export async function sendLike(recipientId: string) {
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) throw new Error('Not authenticated');

  // Insert the like — this is the only thing we MUST await
  const { data, error } = await supabase
    .from('interest_cards')
    .insert({ sender_id: user.id, recipient_id: recipientId, action: 'like' })
    .select()
    .single();

  if (error) throw error;

  // Fire these in the background — do NOT await them
  // They do not affect the swipe animation or immediate UI
  incrementUsage('interest_cards_today').catch(() => {});

  supabase.functions.invoke('calculate-match-scores', {
    body: { userId: user.id },
  }).catch(() => {});

  // Check for match in background and return a promise the caller can optionally await
  const matchPromise = supabase
    .from('matches')
    .select('*')
    .or(`and(user_id_1.eq.${user.id},user_id_2.eq.${recipientId}),and(user_id_1.eq.${recipientId},user_id_2.eq.${user.id})`)
    .eq('status', 'matched')
    .single()
    .then(({ data: match }) => match)
    .catch(() => null);

  return { interestCard: data, matchPromise };
}
```

### File: `screens/renter/RoommatesScreen.tsx`

Find where `sendLike` result is used (around line 529) and update the match check to use the promise:

## Fix 4 — Prefetch Next Card Images

### File: `screens/renter/RoommatesScreen.tsx`

**Problem:** Profile card images are loaded only when the card becomes visible. Users see a blank card briefly on each new swipe.

**Fix — prefetch the next 3 cards' images after the deck loads:**

Add this import at the top of the file if not already present:
```typescript
import { Image } from 'react-native';
```

Add this helper function inside the component, after the `loadProfiles` function:

```typescript
const prefetchNextImages = useCallback((profileList: RoommateProfile[], fromIndex: number) => {
  // Prefetch photos for the next 3 profiles
  const nextProfiles = profileList.slice(fromIndex, fromIndex + 3);
  nextProfiles.forEach(profile => {
    const photos = Array.isArray(profile.photos) ? profile.photos : profile.photos ? [profile.photos] : [];
    photos.slice(0, 2).forEach(uri => {
      if (uri && uri.startsWith('http')) {
        Image.prefetch(uri).catch(() => {});
      }
    });
  });
}, []);
```

Call `prefetchNextImages` in two places:

1. At the end of `loadProfiles`, just before `setIsLoading(false)`:
```typescript
prefetchNextImages(finalProfiles, 0);
setIsLoading(false);
```

2. After each swipe advances `currentIndex` — find where `setCurrentIndex` is called after a swipe and add:
```typescript
setCurrentIndex(prev => {
  const next = prev + 1;
  prefetchNextImages(profiles, next + 1); // prefetch starting 2 ahead
  return next;
});
```

## Fix 5 — Memoize Compatibility Scores

### File: `screens/renter/RoommatesScreen.tsx`

**Problem:** `calculateCompatibility(user, currentProfile)` is called inline during render at lines ~1106 and ~1518. This runs the full algorithm on every re-render (every state change, animation tick, etc.).

**Fix — use `useMemo` to cache compatibility scores:**

Find the two inline `calculateCompatibility` calls and replace them with a memoized map:

Add this near the top of the component, after `profiles` state is declared:

```typescript
const compatibilityMap = useMemo(() => {
  if (!user) return new Map<string, number>();
  const map = new Map<string, number>();
  profiles.forEach(profile => {
    const score = profile.compatibility ?? calculateCompatibility(user, profile);
    map.set(profile.id, score);
  });
  return map;
}, [profiles, user?.id]);
```

Then replace every inline `calculateCompatibility(user, currentProfile)` and `calculateCompatibility(user, profile)` call with:

```typescript
compatibilityMap.get(profile.id) ?? 50
```
