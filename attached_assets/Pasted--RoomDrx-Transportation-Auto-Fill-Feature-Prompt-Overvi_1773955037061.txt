# RoomDrx – Transportation Auto-Fill Feature Prompt

## Overview
When a host saves a listing, automatically fetch nearby transit stops using the Google Places API and save them to the listing. The listing card and detail view will then display transit info without any extra input from the host.

This feature works globally — but coverage varies by city. To handle all cases gracefully, three layers are implemented:
1. **Auto-fill** — Google Places API fetches stops automatically from the address (works in most major cities worldwide)
2. **"No transit nearby" label** — if auto-fill finds nothing, display a clear indicator instead of hiding the section
3. **Manual override** — host can always type in their own transit description if auto-fill is wrong or missing

### 2. Create `utils/transitService.ts` — New utility file

This file handles geocoding the address and fetching nearby transit stops.

```ts
// utils/transitService.ts

const GOOGLE_API_KEY = process.env.EXPO_PUBLIC_GOOGLE_MAPS_API_KEY || '';

export interface TransitStop {
  name: string;
  type: 'subway' | 'bus' | 'train' | 'tram' | 'ferry' | 'other';
  distanceMiles: number;
}

// Step 1: Geocode address to lat/lng
export async function geocodeAddress(address: string, city: string, state: string): Promise<{ lat: number; lng: number } | null> {
  const fullAddress = encodeURIComponent(`${address}, ${city}, ${state}`);
  const url = `https://maps.googleapis.com/maps/api/geocode/json?address=${fullAddress}&key=${GOOGLE_API_KEY}`;

  const res = await fetch(url);
  const data = await res.json();

  if (data.status === 'OK' && data.results.length > 0) {
    const loc = data.results[0].geometry.location;
    return { lat: loc.lat, lng: loc.lng };
  }
  return null;
}

// Step 2: Find nearby transit stops using Places Nearby Search
export async function fetchNearbyTransit(lat: number, lng: number): Promise<TransitStop[]> {
  const radius = 800; // 800 meters (~0.5 miles)
  const types = ['subway_station', 'bus_station', 'train_station', 'transit_station'];
  const stops: TransitStop[] = [];
  const seen = new Set<string>();

  for (const type of types) {
    const url = `https://maps.googleapis.com/maps/api/place/nearbysearch/json?location=${lat},${lng}&radius=${radius}&type=${type}&key=${GOOGLE_API_KEY}`;
    const res = await fetch(url);
    const data = await res.json();

    if (data.status === 'OK') {
      for (const place of data.results.slice(0, 3)) {
        if (seen.has(place.place_id)) continue;
        seen.add(place.place_id);

        const placeLat = place.geometry.location.lat;
        const placeLng = place.geometry.location.lng;
        const distanceMiles = haversineDistance(lat, lng, placeLat, placeLng);

        stops.push({
          name: place.name,
          type: mapPlaceTypeToTransitType(type),
          distanceMiles: Math.round(distanceMiles * 10) / 10,
        });
      }
    }
  }

  // Sort by distance, return closest 5
  return stops.sort((a, b) => a.distanceMiles - b.distanceMiles).slice(0, 5);
}

// Map Google place type to our internal type
function mapPlaceTypeToTransitType(type: string): TransitStop['type'] {
  switch (type) {
    case 'subway_station': return 'subway';
    case 'bus_station': return 'bus';
    case 'train_station': return 'train';
    case 'transit_station': return 'other';
    default: return 'other';
  }
}

// Haversine formula — distance in miles between two lat/lng points
function haversineDistance(lat1: number, lng1: number, lat2: number, lng2: number): number {
  const R = 3958.8; // Earth radius in miles
  const dLat = ((lat2 - lat1) * Math.PI) / 180;
  const dLng = ((lng2 - lng1) * Math.PI) / 180;
  const a =
    Math.sin(dLat / 2) ** 2 +
    Math.cos((lat1 * Math.PI) / 180) *
      Math.cos((lat2 * Math.PI) / 180) *
      Math.sin(dLng / 2) ** 2;
  return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
}
```

#### C. Update `handleSave` to fetch transit and respect manual override

In `handleSave`, after building the `property` object and before calling `StorageService.addOrUpdateProperty()`, add:

```ts
// --- TRANSPORTATION AUTO-FILL ---
try {
  // Always save the manual override if the host typed one
  const override = transitOverride.trim() || undefined;

  const coords = await geocodeAddress(address.trim(), city.trim(), state.trim());
  if (coords) {
    property.coordinates = coords;
    const stops = await fetchNearbyTransit(coords.lat, coords.lng);

    property.transitInfo = {
      stops,
      noTransitNearby: stops.length === 0,
      manualOverride: override,
      fetchedAt: new Date().toISOString(),
    };
  } else {
    // Could not geocode — save override only
    property.transitInfo = {
      stops: [],
      noTransitNearby: true,
      manualOverride: override,
      fetchedAt: new Date().toISOString(),
    };
  }
} catch (transitError) {
  // Non-blocking — listing still saves without transit info
  console.warn('Transit fetch failed:', transitError);
}
// --- END TRANSPORTATION AUTO-FILL ---
```

Add imports at the top of `CreateEditListingScreen.tsx`:

## Notes

- **Non-blocking**: If the Google API call fails for any reason, the listing still saves. Transit info just won't appear until next save.
- **Re-fetches on edit**: Every time the host saves, transit info is re-fetched. This keeps it accurate if the address changes.
- **Cost**: Each listing save = 1 Geocoding call + up to 4 Places calls = ~$0.13. Very low cost.
- **No host action needed**: Fully automatic from the address field — manual override is optional.
- **Manual override wins**: If a host types anything in the override field, it displays instead of the auto-fetched stops. This lets hosts in areas with poor Google coverage still show accurate transit info.
- **No transit label**: If auto-fetch returns zero results and no manual override is set, the listing card shows "No public transit nearby" — useful info for renters rather than a blank section.
- **Works globally**: Auto-fill works in any city where Google Maps has transit data. For cities with partial or no coverage, the manual override ensures hosts can always provide accurate info.

## Display Priority (on listing card)

```
1. Manual override (if set by host)         → shown as italic description
        ↓ if empty
2. Auto-detected stops (if found)           → shown as list with icons + distance
        ↓ if none found
3. "No public transit nearby"               → shown as greyed-out note
```
