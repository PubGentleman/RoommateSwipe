# Replit Prompt — Implement Stripe Identity Verification + Checkr Background Checks

## Overview

The backend infrastructure for Stripe Identity is already built:
- ✅ `supabase/functions/create-verification-session/index.ts` — creates Stripe Identity session
- ✅ `supabase/functions/stripe-webhook/index.ts` — handles `identity.verification_session.verified` and updates `identity_verified` in Supabase
- ✅ `services/paymentService.ts` — has `createVerificationSession()` and `initiateBackgroundCheck()`

**What's missing:**
- ❌ The app calls a fake `setTimeout` instead of real Stripe Identity SDK
- ❌ No Stripe SDK initialization for Identity
- ❌ No status sync between Supabase `identity_verified` and local user state
- ❌ Background check has no real provider — just sets `pending` status

**Background checks:** Stripe Identity only verifies WHO someone is (ID + selfie). It **cannot** do background checks. Use **Checkr** for criminal history, eviction records, and employment verification.

### 3a. Replace fake `handleVerifyId` with real Stripe Identity

Add the import at the top of the file:
```tsx

```

Remove the old `handleVerifyId` function and replace with:

```tsx
const { present: presentIdentity, status: identityStatus } = useStripeIdentity(
  async () => {
    if (!user) return { clientSecret: '', ephemeralKeySecret: '' };
    try {
      const { clientSecret } = await createVerificationSession(user.id);
      return { clientSecret };
    } catch (e) {
      Alert.alert('Error', 'Failed to start verification. Please try again.');
      return { clientSecret: '', ephemeralKeySecret: '' };
    }
  }
);

const handleVerifyId = async () => {
  if (!user) return;

  // In dev mode, simulate verification
  if (isDev) {
    setIdUploading(true);
    setTimeout(async () => {
      const newVerification = {
        ...verification,
        government_id: { verified: true, verifiedAt: new Date() },
      };
      await updateUser({ verification: newVerification });
      await syncVerificationToSupabase(newVerification);
      setIdUploading(false);
      Alert.alert('ID Verified (Dev)', 'Simulated verification complete.');
    }, 1500);
    return;
  }

  // Production: launch Stripe Identity sheet
  setIdUploading(true);
  try {
    await presentIdentity();
    // After the sheet closes, check status
    if (identityStatus === 'FlowCompleted') {
      // Poll Supabase for the verified status (webhook updates it asynchronously)
      await pollForVerificationStatus();
    } else if (identityStatus === 'FlowCanceled') {
      Alert.alert('Cancelled', 'ID verification was cancelled.');
    } else if (identityStatus === 'FlowFailed') {
      Alert.alert('Failed', 'ID verification failed. Please try again with a clear photo.');
    }
  } catch (e: any) {
    Alert.alert('Error', e.message || 'Something went wrong. Please try again.');
  } finally {
    setIdUploading(false);
  }
};

const pollForVerificationStatus = async () => {
  if (!user) return;
  // Poll Supabase up to 10 times, every 2 seconds, for identity_verified = true
  // (Stripe webhook updates this async)
  let attempts = 0;
  const maxAttempts = 10;
  const interval = 2000;

  const check = async (): Promise<void> => {
    attempts++;
    try {
      const { data } = await supabase
        .from('users')
        .select('identity_verified, identity_verified_at')
        .eq('id', user.id)
        .single();

      if (data?.identity_verified) {
        // Stripe confirmed — update local user state
        const newVerification = {
          ...verification,
          government_id: {
            verified: true,
            verifiedAt: data.identity_verified_at
              ? new Date(data.identity_verified_at)
              : new Date(),
          },
        };
        await updateUser({ verification: newVerification });
        await syncVerificationToSupabase(newVerification);
        Alert.alert(
          '✅ ID Verified',
          'Your government ID has been verified successfully by Stripe Identity.'
        );
        return;
      }
    } catch (_e) {}

    if (attempts < maxAttempts) {
      await new Promise(res => setTimeout(res, interval));
      return check();
    } else {
      // Timeout — show pending message
      Alert.alert(
        'Verification Pending',
        'Your ID is being reviewed. This usually takes a few minutes. We\'ll notify you when it\'s complete.'
      );
    }
  };

  await check();
};
```

Make sure `supabase` is imported from `../../lib/supabase` in VerificationScreen.

## Step 4 — Update `supabase/functions/initiate-background-check/index.ts`

Replace the placeholder with a real Checkr API integration:

```ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
);

const CHECKR_API_KEY = Deno.env.get('CHECKR_API_KEY')!;
const CHECKR_PACKAGE = 'checkr_package_roommate_basic'; // your Checkr package slug

Deno.serve(async (req) => {
  const { userId, email } = await req.json();

  try {
    // Step 1: Create a Checkr candidate
    const candidateRes = await fetch('https://api.checkr.com/v1/candidates', {
      method: 'POST',
      headers: {
        'Authorization': `Basic ${btoa(CHECKR_API_KEY + ':')}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({ email }),
    });
    const candidate = await candidateRes.json();
    if (!candidateRes.ok) throw new Error(candidate.message || 'Failed to create candidate');

    // Step 2: Create an invitation (sends email to candidate with Checkr link)
    const invitationRes = await fetch('https://api.checkr.com/v1/invitations', {
      method: 'POST',
      headers: {
        'Authorization': `Basic ${btoa(CHECKR_API_KEY + ':')}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        candidate_id: candidate.id,
        package: CHECKR_PACKAGE,
      }),
    });
    const invitation = await invitationRes.json();
    if (!invitationRes.ok) throw new Error(invitation.message || 'Failed to create invitation');

    // Step 3: Store Checkr IDs in Supabase
    await supabase
      .from('users')
      .update({
        background_check_status: 'pending',
        checkr_candidate_id: candidate.id,
        background_check_initiated_at: new Date().toISOString(),
      })
      .eq('id', userId);

    return new Response(
      JSON.stringify({
        status: 'pending',
        invitationUrl: invitation.invitation_url,
        message: 'Background check initiated. A link has been sent to your email.',
      }),
      { headers: { 'Content-Type': 'application/json' } }
    );
  } catch (error: any) {
    return new Response(
      JSON.stringify({ error: error.message }),
      { status: 500, headers: { 'Content-Type': 'application/json' } }
    );
  }
});
```

## Step 5 — Handle Checkr webhook in `supabase/functions/stripe-webhook/index.ts`

Create a **separate** Supabase edge function for Checkr webhooks:
`supabase/functions/checkr-webhook/index.ts`

```ts
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
);

Deno.serve(async (req) => {
  const body = await req.json();
  const { type, data } = body;

  // Checkr sends 'report.completed' when background check is done
  if (type === 'report.completed') {
    const report = data.object;
    const candidateId = report.candidate_id;
    const status = report.adjudication === 'clear' ? 'clear' : 'consider';

    await supabase
      .from('users')
      .update({
        background_check_status: status,
        background_check_completed_at: new Date().toISOString(),
      })
      .eq('checkr_candidate_id', candidateId);

    // Send in-app notification if status is clear
    if (status === 'clear') {
      const { data: userData } = await supabase
        .from('users')
        .select('id')
        .eq('checkr_candidate_id', candidateId)
        .single();

      if (userData) {
        await supabase.from('notifications').insert({
          user_id: userData.id,
          type: 'background_check_complete',
          title: '✅ Background Check Complete',
          body: 'Your background check came back clear. Your profile now shows a verified badge.',
          is_read: false,
          created_at: new Date().toISOString(),
        });
      }
    }
  }

  return new Response(JSON.stringify({ received: true }), { status: 200 });
});
```

Register this endpoint in Checkr's dashboard as your webhook URL:
`https://<your-project>.supabase.co/functions/v1/checkr-webhook`

| Feature | Before | After |
|---------|--------|-------|
| ID Verification | Fake setTimeout | Real Stripe Identity SDK (document + selfie) |
