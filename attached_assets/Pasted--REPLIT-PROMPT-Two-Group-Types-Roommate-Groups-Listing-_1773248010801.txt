**REPLIT PROMPT — Two Group Types: Roommate Groups + Listing Inquiry Groups**

Build two distinct group types in Roomdr. Roommate groups are renter-only. Listing inquiry groups allow a host to participate in a limited, clearly labeled role. Complete all parts fully.

---

**PART 1 — UPDATE GROUP DATA MODEL**

In `types/models.ts`, update the Group type to support two group types:

```ts
export type GroupType = 'roommate' | 'listing_inquiry';

export interface Group {
  id: string;
  type: GroupType;
  name: string;
  members: GroupMember[];
  createdBy: string;
  createdAt: string;
  updatedAt: string;
  listingId?: string;          // listing_inquiry only
  hostId?: string;             // listing_inquiry only
  listingAddress?: string;     // listing_inquiry only
  isArchived: boolean;
}

export interface GroupMember {
  userId: string;
  role: 'renter' | 'host';
  joinedAt: string;
  isHost: boolean;             // true only for host in listing_inquiry
}
```

Update the `groups` table in Supabase:

```sql
alter table public.groups add column if not exists type text 
  check (type in ('roommate', 'listing_inquiry')) default 'roommate';
alter table public.groups add column if not exists listing_id uuid 
  references public.listings(id) on delete set null;
alter table public.groups add column if not exists host_id uuid 
  references public.users(id) on delete set null;
alter table public.groups add column if not exists listing_address text;
alter table public.groups add column if not exists is_archived boolean default false;

alter table public.group_members add column if not exists is_host boolean default false;
```

**"Inquire Together" Modal:**
- Header: **"Start a Group Inquiry"**

- Shows a list of the renter's existing roommate groups — each as a selectable row with group name and member count

  3. Add the listing's host as a member with `isHost: true, role: 'host'`
  4. Set `listingId`, `hostId`, and `listingAddress` on the group
  5. Send a notification to the host: **"[Group name] wants to inquire about your listing at [address]"**
  6. Navigate directly into the new inquiry group chat

**PART 7 — ARCHIVE FLOW**

When a listing inquiry is resolved (rented or no longer interested):

Renters can archive the inquiry group by long pressing the group card and tapping **"Archive Inquiry"**. This:
- Sets `is_archived: true` on the group
- Moves it to a collapsed **"Past Inquiries"** section at the bottom of the Listing Inquiries list
- Sends a notification to the host: **"[Group] has closed their inquiry for [address]"**

Hosts can archive from their side in the same way.

Archived groups are read-only — messages are visible but no new messages can be sent. Show a banner at the top of archived chats: **"This inquiry has been archived — no new messages can be sent"**

**PART 8 — RENTER PROTECTIONS**

Add these safeguards so renters always feel safe:

- Hosts cannot initiate contact with a renter group — the inquiry must always be started by renters
- In the inquiry chat, renters can vote to remove themselves from the inquiry with a **"Leave Inquiry"** option — if all renters leave, the group auto-archives
- Renters can report the host directly from the inquiry chat via a **"Report Host"** option in the chat menu
- The host's profile in the inquiry group links to their public host profile and listings — not to a match card
