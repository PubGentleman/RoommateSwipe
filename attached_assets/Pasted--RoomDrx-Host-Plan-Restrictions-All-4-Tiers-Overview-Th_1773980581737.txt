# RoomDrx — Host Plan Restrictions (All 4 Tiers)

## Overview

There are 4 host tiers: **Free**, **Starter ($19.99/mo)**, **Pro ($49.99/mo)**, and **Business ($99.99/mo)**. This prompt defines the complete restriction logic for every feature across all 4 tiers, a central `PLAN_LIMITS` constants file, enforcement helpers, and updates to the purchase config and all affected screens.

### Two distinct outreach flows — do not confuse them

**Proactive outreach** (pre-rental) — The host browses the Renter Groups / Discover page, finds groups actively looking for a place, and messages them to introduce a listing. The host initiates this at any time. This is gated by plan tier with a daily cap and hourly rate limit.

**Post-rental outreach** (pay-per-use) — After a host marks a listing as rented and waits 48 hours, they unlock the ability to message groups about their *next* available listing. This is a one-time payment ($2.99 / $6.99 / $9.99) and applies equally to all 4 tiers. It is never free and has no plan-based daily limit — the payment itself is the gate.

## Step 2 — Update Proactive Outreach Service

Update `OUTREACH_PLAN_LIMITS` in `hostOutreachService.ts` to use the central constants:

```ts
// services/hostOutreachService.ts

import { getPlanLimits, canUseProactiveOutreach, type HostPlan } from '../constants/planLimits';

// Replace old hardcoded OUTREACH_PLAN_LIMITS object entirely — now derived from PLAN_LIMITS:
function getOutreachLimits(plan: HostPlan) {
  const limits = getPlanLimits(plan);
  return {
    dailyCap:    limits.proactiveOutreachPerDay,
    hourlyCap:   limits.proactiveOutreachPerHour,
    cooldownDays: limits.groupCooldownDays,
    canUnlock:   limits.canPayToUnlock,
  };
}

export async function sendProactiveOutreach(
  hostId: string,
  groupId: string,
  message: string,
  plan: HostPlan
): Promise<{ success: boolean; error?: string }> {

  // 0. Free tier — feature not available at all
  if (!canUseProactiveOutreach(plan)) {
    return { success: false, error: 'upgrade_required' };
  }

  const { dailyCap, hourlyCap, cooldownDays, canUnlock } = getOutreachLimits(plan);

  // 1. Check suspension
  const suspended = await checkSuspension(hostId);
  if (suspended) return { success: false, error: 'suspended' };

  // 2. Check daily cap (including paid_extra credits)
  const quota = await getOutreachQuotaStatus(hostId, dailyCap);
  if (quota.hitDailyLimit) {
    if (canUnlock) return { success: false, error: 'daily_limit_hit' }; // show unlock modal
    return { success: false, error: 'daily_limit_hit_no_unlock' };
  }

  // 3. Check hourly rate limit
  const hitHourly = await checkHourlyLimit(hostId, hourlyCap);
  if (hitHourly) return { success: false, error: 'hourly_limit' };

  // 4. Check per-group cooldown
  const onCooldown = await checkGroupCooldown(hostId, groupId, cooldownDays);
  if (onCooldown) return { success: false, error: 'group_cooldown' };

  // 5. Message minimum length
  if (message.trim().length < 50) return { success: false, error: 'message_too_short' };

  // 6. Insert outreach + increment quota
  const { error } = await supabase
    .from('host_group_outreach')
    .insert({ host_id: hostId, group_id: groupId, message });

  if (error) return { success: false, error: 'db_error' };

  await supabase.rpc('increment_daily_outreach', { p_host_id: hostId });
  return { success: true };
}

export async function getOutreachQuotaStatus(hostId: string, dailyCap: number) {
  const { data } = await supabase
    .from('host_outreach_daily_quota')
    .select('used, paid_extra')
    .eq('host_id', hostId)
    .eq('date', new Date().toISOString().split('T')[0])
    .single();

  const used      = data?.used ?? 0;
  const paidExtra = data?.paid_extra ?? 0;
  const total     = dailyCap + paidExtra;
  const remaining = Math.max(0, total - used);

  return {
    used,
    dailyCap,
    paidExtra,
    total,
    remaining,
    hitDailyLimit: used >= total,
  };
}
```

## Step 3 — Listing Creation Guard

Enforce `maxListings` before allowing a new listing to be posted:

```ts
// services/listingService.ts

import { canAddListing, getPlanLimits } from '../constants/planLimits';
import type { HostPlan } from '../constants/planLimits';

export async function createListing(hostId: string, plan: HostPlan, data: any) {
  // Count active listings for this host
  const { count } = await supabase
    .from('listings')
    .select('id', { count: 'exact', head: true })
    .eq('host_id', hostId)
    .neq('status', 'archived');

  if (!canAddListing(plan, count ?? 0)) {
    const limits = getPlanLimits(plan);
    return {
      success: false,
      error: 'listing_limit_reached',
      limit: limits.maxListings,
      upgradeRequired: true,
    };
  }

  const { data: listing, error } = await supabase
    .from('listings')
    .insert({ ...data, host_id: hostId })
    .select()
    .single();

  if (error) return { success: false, error: 'db_error' };
  return { success: true, listing };
}
```

In `CreateListingScreen.tsx`, check the result and show `PurchaseConfirmModal` with the next tier's config if `upgradeRequired: true`:

```tsx
const result = await createListing(user.id, hostPlan, formData);

if (!result.success && result.upgradeRequired) {
  // Show upgrade modal for the next plan tier
  const nextPlan = hostPlan === 'free' ? 'starter' : hostPlan === 'starter' ? 'pro' : 'business';
  setUpgradePlan(nextPlan);   // triggers PurchaseConfirmModal
  return;
}
```

## Step 8 — Listing Placement Badge

```ts
const { data: listings } = await supabase

  .order('placement_rank', { ascending: false })  // see below
  .order('created_at', { ascending: false });
```

Add `placement_rank` as a computed column or DB trigger based on `host_plan`:

```sql
-- Add computed placement rank to listings view or trigger
-- featured = 4, top = 3, priority = 2, standard = 1

ALTER TABLE listings ADD COLUMN IF NOT EXISTS placement_rank INT GENERATED ALWAYS AS (
  CASE
    WHEN (SELECT host_plan FROM profiles WHERE id = host_id) = 'business' THEN 4
    WHEN (SELECT host_plan FROM profiles WHERE id = host_id) = 'pro'      THEN 3
    WHEN (SELECT host_plan FROM profiles WHERE id = host_id) = 'starter'  THEN 2
    ELSE 1
  END
) STORED;
```

## Step 9 — DB: Store Host Plan on Profiles

Ensure `host_plan` is stored on the profile and kept in sync with Stripe:

```sql
ALTER TABLE profiles
  ADD COLUMN IF NOT EXISTS host_plan TEXT DEFAULT 'free'
    CHECK (host_plan IN ('free', 'starter', 'pro', 'business')),
  ADD COLUMN IF NOT EXISTS host_plan_expires_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS stripe_subscription_id TEXT;
```

The Stripe webhook (`stripe-webhook` edge function) updates `host_plan` on `customer.subscription.updated` and resets to `'free'` on `customer.subscription.deleted`.

## Complete Enforcement Map

| Restriction | Where enforced | What happens when limit hit |
|---|---|---|
| Max listings (1/5/∞) | `listingService.createListing()` | Error returned → `PurchaseConfirmModal` for next tier |
| Proactive outreach daily cap | `hostOutreachService.sendProactiveOutreach()` | `daily_limit_hit` → show unlock modal (Starter+ only) |
| Proactive outreach hourly cap | `hostOutreachService.checkHourlyLimit()` | `hourly_limit` → toast "Slow down, try again soon" |
| Per-group 30-day cooldown | `hostOutreachService.checkGroupCooldown()` | `group_cooldown` → group grayed out with "X days left" |
| Free tier proactive outreach | `canUseProactiveOutreach()` check | `upgrade_required` → `LockedFeatureWall` + `PurchaseConfirmModal` Starter |
| Group profile full access | `canViewFullGroupProfile()` check in card tap | `PurchaseConfirmModal` → Starter |
| Analytics access | `canAccessAnalytics()` check on screen load | `LockedFeatureWall` → Pro |
| Company/Agent branding | `canUseCompanyBranding()` check | `LockedFeatureWall` → Business |
| Pay-to-unlock after cap | `limits.canPayToUnlock` (false for Free) | No unlock option shown for Free hosts |
| Listing placement sort order | DB `placement_rank` column + query order | Automatic — higher plan = higher in feed |
