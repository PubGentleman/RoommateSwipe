# Prompt: Real Group Messaging — Host Messages All Members + New Member Auto-Sync

## Overview

Two problems to fix:

1. **"Message This Group" does nothing real** — it just marks a local state flag. No actual conversation is created. The host messages nobody.
2. **New members who join after a chat starts are invisible to it** — if a group of 2 starts a chat with a host and a 3rd person joins the roommate group, they're never added to the inquiry chat.

The fix uses the existing `createListingInquiryGroup` infrastructure which already handles multi-party group chats correctly.

## Part 3 — Auto-add new members to existing inquiry chats

### 3a. Update `joinGroup()` in `services/groupService.ts`

After a member successfully joins a roommate group, find all active `listing_inquiry` groups that were created from that roommate group and add the new member to each one, plus insert a system message.

Replace the current `joinGroup` function with:

```ts
export async function joinGroup(groupId: string) {
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) throw new Error('Not authenticated');

  // Add member to the roommate group
  const { data, error } = await supabase
    .from('group_members')
    .insert({ group_id: groupId, user_id: user.id, role: 'member', status: 'active' })
    .select()
    .single();

  if (error) throw error;

  // Find all active listing_inquiry groups that originated from this roommate group
  const { data: inquiryGroups } = await supabase
    .from('groups')
    .select('id, inquiry_status, name')
    .eq('source_group_id', groupId)
    .eq('type', 'listing_inquiry')
    .eq('is_archived', false);

  if (inquiryGroups && inquiryGroups.length > 0) {
    // Get the new member's name for the system message
    const { data: profile } = await supabase
      .from('users')
      .select('full_name')
      .eq('id', user.id)
      .single();

    const memberName = profile?.full_name || 'A new member';

    for (const inquiryGroup of inquiryGroups) {
      // Add new member to the inquiry group
      await supabase
        .from('group_members')
        .insert({
          group_id: inquiryGroup.id,
          user_id: user.id,
          role: 'member',
          is_host: false,
          status: 'active',
          joined_at: new Date().toISOString(),
        })
        .select()
        .single();

      // Insert a system message marking when they joined
      const systemMessage =
        inquiryGroup.inquiry_status === 'accepted'
          ? `${memberName} joined after this inquiry was accepted.`
          : inquiryGroup.inquiry_status === 'declined'
          ? `${memberName} joined after this inquiry was declined.`
          : `${memberName} joined the group.`;

      await supabase
        .from('messages')
        .insert({
          group_id: inquiryGroup.id,
          sender_id: null,           // null sender = system message
          content: systemMessage,
          is_system_message: true,
          created_at: new Date().toISOString(),
        });
    }
  }

  return data;
}
```

## Part 5 — Supabase `groups` table migration note

The `groups` table needs a `source_group_id` column if it doesn't already exist. Run this migration in Supabase SQL editor:

```sql
ALTER TABLE groups
ADD COLUMN IF NOT EXISTS source_group_id UUID REFERENCES groups(id) ON DELETE SET NULL;

-- Index for fast lookup of inquiry groups by source
CREATE INDEX IF NOT EXISTS idx_groups_source_group_id ON groups(source_group_id);
```

Also ensure the `messages` table has `is_system_message`:
```sql
ALTER TABLE messages
ADD COLUMN IF NOT EXISTS is_system_message BOOLEAN DEFAULT false;
```

## Summary of the full flow

### Host messages a renter group:
1. Host taps "Message This Group"
2. Confirmation modal appears: "A group chat will be created with all members"
3. Host confirms → `createListingInquiryGroup` called with host's listing + renter group ID
4. Inquiry group created with host + ALL current renter group members
5. Host is navigated into the new group chat
6. All renter group members get a notification and see the chat in their Messages tab

### New member joins the roommate group later:
1. Renter swipes right or accepts invite to join roommate group
2. `joinGroup()` adds them to the roommate group
3. System automatically finds all active inquiry chats linked to that roommate group
4. New member is silently added to each one — they can now see and send messages
5. A system message appears in each chat: **"Jordan Kim joined the group."**
6. If the inquiry was already accepted/declined, the message notes that context

### What new members see:
Full message history from before they joined — they need context about what listing was discussed, what the host said, what the group agreed to. There is no privacy concern between group members about housing discussions.
