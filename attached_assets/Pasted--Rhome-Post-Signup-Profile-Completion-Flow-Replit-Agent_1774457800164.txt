# Rhome — Post-Signup Profile Completion Flow
## Replit Agent Prompt

---

## Overview

Move all detailed profile questions out of the sign-up flow and into a post-signup **profile completion** experience. Sign-up should only collect the minimum needed to create an account (account type, name, email, password, city, photo). Everything else — occupation, lifestyle, sleep schedule, roommate preferences, budget — is collected after the user is already in the app.

This reduces sign-up drop-off and lets users see the app before committing to a long questionnaire.

---

## Part 1 — Remove These Questions From Sign-Up

Remove the following screens from the current onboarding flow entirely. They will be replaced by the profile completion system described below:

- "What do you do?" (occupation picker)
- "Describe yourself" / lifestyle tags
- Sleep schedule
- Cleanliness level
- Noise tolerance
- Guest preferences
- Budget range (can be collected later)
- Any other preference or personality questions

Keep only:
- Account type selection
- Name, email, password
- City / location
- Profile photo

## Part 2 — Profile Completion Percentage

Add a `profile_completion` integer (0–100) to the `profiles` table, or compute it dynamically from which fields are filled in. Track which sections are complete:

```typescript
function getProfileCompletion(user: User): number {
  const fields = [
    !!user.profilePicture,        // 15 points
    !!user.occupation,            // 15 points
    !!user.bio,                   // 15 points
    !!user.budget,                // 10 points
    !!user.sleepSchedule,         // 10 points
    !!user.cleanlinessLevel,      // 10 points
    !!user.noiseTolerance,        // 10 points
    !!user.guestPreference,       // 10 points
    (user.tags?.length ?? 0) >= 3, // 5 points
  ];

  const weights = [15, 15, 15, 10, 10, 10, 10, 10, 5];
  return fields.reduce((total, done, i) => total + (done ? weights[i] : 0), 0);
}
```

## Part 3 — Profile Completion Banner

Show a dismissible banner on the home/discover screen for users whose profile is less than 80% complete.

```typescript
{profileCompletion < 80 && !bannerDismissed && (
  <TouchableOpacity
    style={styles.completionBanner}
    onPress={() => navigation.navigate('ProfileCompletion')}
    activeOpacity={0.85}
  >
    <View style={styles.completionLeft}>
      <Text style={styles.completionTitle}>Complete your profile</Text>
      <Text style={styles.completionSub}>
        {profileCompletion}% done — better profiles get more matches
      </Text>
      <View style={styles.progressBarTrack}>
        <View style={[styles.progressBarFill, { width: `${profileCompletion}%` }]} />
      </View>
    </View>
    <Ionicons name="chevron-forward" size={18} color="#888" />
  </TouchableOpacity>
)}
```

Dismiss button (X) saves dismissal to AsyncStorage so it doesn't reappear in the same session. Reappears next session if still under 80%.

## Summary

| Before | After |
|---|---|
| 5+ screens of questions during sign-up | Sign-up = 4 steps max |
| Users drop off before seeing the app | Users in the app in under 2 minutes |
| Occupation asked of everyone including agents | Agents skip occupation — already identified |
| No way to complete profile later | Profile completion screen + banner + % tracker |
| All-or-nothing questionnaire | Complete one section at a time, anytime |
