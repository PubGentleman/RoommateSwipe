# RoomDrx — Group Property Search: Smart Bedroom Filtering Update

## Problem

The `GroupPropertySearchModal` currently shows all listings including 1BR studios. Since groups have multiple members who each need their own room, the search should only show listings that can actually fit the group.

## The Rule

From the established member limit logic: `bedrooms + 1 = max members`

Therefore: `minimum bedrooms needed = memberCount - 1`

| Group Size | Min Bedrooms | Logic |
|---|---|---|
| 2 members | 1 BR | 1BR supports 2 max — acceptable |
| 3 members | 2 BR | 1BR only fits 2, can't accommodate 3 |
| 4 members | 3 BR | 2BR only fits 3, can't accommodate 4 |
| 5 members | 4 BR | 3BR only fits 4, can't accommodate 5 |

## Changes Required

### 1. Update `GroupPropertySearchModal` Props

```tsx
// Before
interface GroupPropertySearchModalProps {
  groupId: string;
  onSelect: (listing: Listing) => void;
  onClose: () => void;

// After
interface GroupPropertySearchModalProps {
  groupId: string;
  memberCount: number; // ← ADD THIS

  onClose: () => void;

```

### 2. Compute `minBedrooms` from `memberCount`

Add this near the top of the component, before state declarations:

```tsx
export function GroupPropertySearchModal({
  groupId,
  memberCount,
  onSelect,
  onClose,
}: GroupPropertySearchModalProps) {
  const { theme } = useTheme();

  // Minimum bedrooms this group needs based on member count
  const minBedrooms = Math.max(1, memberCount - 1);

  // Default bedroom filter starts at minBedrooms, not "Any"
  const [selectedBedrooms, setSelectedBedrooms] = useState<number>(minBedrooms);

  // ... rest of existing state
```

### 3. Filter the Bedroom Chips

Replace the static bedroom options array with a filtered one that hides options below the minimum:

```tsx
// Before — static options
const BEDROOM_OPTIONS = [
  { label: 'Any', value: 0 },
  { label: '1', value: 1 },
  { label: '2', value: 2 },
  { label: '3', value: 3 },
  { label: '4+', value: 4 },
];

// After — remove options the group can't use
const BEDROOM_OPTIONS = [
  { label: 'Any', value: 0 },
  { label: '1', value: 1 },
  { label: '2', value: 2 },
  { label: '3', value: 3 },
  { label: '4+', value: 4 },
].filter(opt => opt.value === 0 || opt.value >= minBedrooms);
// "Any" always shows but the query enforces minBedrooms when Any is selected
```

### 4. Enforce Minimum in the Supabase Query

When "Any" is selected, still apply `minBedrooms` as the floor:

```tsx
// Before
if (selectedBedrooms > 0) {
  query = query.eq('bedrooms', selectedBedrooms === 4 ? undefined : selectedBedrooms);
  if (selectedBedrooms === 4) query = query.gte('bedrooms', 4);
}

// After — always enforce minimum bedrooms
const bedroomFloor = selectedBedrooms > 0 ? selectedBedrooms : minBedrooms;

if (bedroomFloor === 4) {
  query = query.gte('bedrooms', 4);
} else {
  query = query.gte('bedrooms', bedroomFloor);
}
```

Full updated search function:

```tsx
async function fetchListings() {
  setSearchLoading(true);
  try {
    let query = supabase
      .from('listings')
      .select('id, title, address, city, neighborhood, price, bedrooms, photos, status')
      .neq('status', 'rented')
      .order('created_at', { ascending: false });

    // Text search
    if (searchText.trim()) {
      query = query.or(
        `title.ilike.%${searchText}%,address.ilike.%${searchText}%,city.ilike.%${searchText}%,neighborhood.ilike.%${searchText}%`
      );
    }

    // Price filter
    if (selectedPriceRange) {
      query = query
        .gte('price', selectedPriceRange.min)
        .lte('price', selectedPriceRange.max);
    }

    // Bedroom filter — ALWAYS enforce minBedrooms as the floor
    const bedroomFloor = selectedBedrooms > 0 ? selectedBedrooms : minBedrooms;
    if (bedroomFloor >= 4) {
      query = query.gte('bedrooms', 4);
    } else {
      query = query.gte('bedrooms', bedroomFloor);
    }

    const { data, error } = await query.limit(30);
    if (error) throw error;
    setListings(data || []);
  } catch (e) {
    console.error('Search error:', e);
  } finally {
    setSearchLoading(false);
  }
}
```

### 8. Edge Case: Group Being Created (0 or 1 Members)

When a user is creating a group and hasn't added any members yet, `memberCount` may be 0 or 1. Apply a safe default:

```tsx
// In GroupPropertySearchModal
const minBedrooms = Math.max(1, memberCount - 1);
// memberCount=0 → minBedrooms=1
// memberCount=1 → minBedrooms=1
// memberCount=2 → minBedrooms=1
// memberCount=3 → minBedrooms=2
// etc.
```

This ensures the minimum is never 0 bedrooms.
