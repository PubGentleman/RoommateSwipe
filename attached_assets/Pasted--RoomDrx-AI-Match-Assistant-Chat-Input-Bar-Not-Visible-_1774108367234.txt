# RoomDrx — AI Match Assistant: Chat Input Bar Not Visible Fix

## File: `screens/renter/AIMatchAssistantScreen.tsx`
(may also be `AIChatScreen.tsx` or `AIAssistantScreen.tsx`)

---

## Root Cause

The chat input bar is either:
1. **Rendered behind the tab bar** — the tab bar overlays the input at `bottom: 0`, making it invisible
2. **Missing `paddingBottom`** on the `ScrollView` so content fills the exact space where the input bar sits
3. **`KeyboardAvoidingView` not wrapping the screen**, so the input bar doesn't push up when the keyboard opens
4. **`useSafeAreaInsets` not applied** — the bottom safe area (home indicator on iPhone) is swallowing the input bar

---

## Fix — Full Screen Structure

Replace the current screen wrapper with this exact structure:

```tsx

      keyboardVerticalOffset={TAB_BAR_HEIGHT} // ← offsets keyboard above tab bar

            <Text style={styles.aiTitle}>AI Match Assistant</Text>

            </View>
          </View>

        </View>

      <ScrollView
        ref={scrollRef}
        style={styles.messageList}
        contentContainerStyle={{
          paddingTop: 16,
          // ⚠️ KEY FIX: paddingBottom must be large enough to
          // clear the input bar height so last message isn't hidden behind it
          paddingBottom: 20,
        }}
        showsVerticalScrollIndicator={false}
        keyboardShouldPersistTaps="handled"
        onContentSizeChange={() => scrollRef.current?.scrollToEnd({ animated: true })}
      >
        {/* AI greeting bubble */}
        <View style={styles.messageRow}>
          <LinearGradient colors={['#ff6b5b', '#ff8c7a']} style={styles.msgAvatar}>
            <Feather name="cpu" size={12} color="#fff" />
          </LinearGradient>
          <View style={styles.aiBubble}>
            <Text style={styles.aiBubbleText}>
              Hi {userName}! I'm your AI Match Assistant. I can help you find roommates,
              discover great neighborhoods, recommend restaurants, suggest activities,
              analyze zodiac compatibility, and give home decor tips.
            </Text>
            <Text style={styles.aiBubbleText}>What would you like to explore?</Text>
          </View>
        </View>

        {/* Quick action cards */}
        <View style={styles.quickActionsWrap}>
          <Text style={styles.quickActionsLabel}>Quick actions</Text>
          <View style={styles.quickActionsGrid}>
            {QUICK_ACTIONS.map((item) => (
              <Pressable

          msg.role === 'assistant' ? (
            <View key={msg.id} style={styles.messageRow}>

                <Feather name="cpu" size={12} color="#fff" />
              </LinearGradient>
              <View style={styles.aiBubble}>
                <Text style={styles.aiBubbleText}>{msg.content}</Text>
              </View>
            </View>
          ) : (
            <View key={msg.id} style={styles.userBubbleRow}>
              <View style={styles.userBubble}>
                <Text style={styles.userBubbleText}>{msg.content}</Text>

        )}

          <View style={styles.messageRow}>
            <LinearGradient colors={['#ff6b5b', '#ff8c7a']} style={styles.msgAvatar}>

            <View style={styles.aiBubble}>

      <View style={[
        styles.inputBar,
        { paddingBottom: Math.max(insets.bottom, 12) }
      ]}>
        <View style={styles.inputWrap}>
          <TextInput
            style={styles.inputField}
            placeholder="Ask me anything..."
            placeholderTextColor="rgba(255,255,255,0.25)"

            onChangeText={setInputText}

            blurOnSubmit={false}
          />

        <Pressable
          style={[styles.sendBtn, (!inputText.trim() || isTyping) && styles.sendBtnDisabled]}
          onPress={handleSend}
          disabled={!inputText.trim() || isTyping}
        >
          <Feather name="send" size={18} color="#fff" />
        </Pressable>
      </View>

    </KeyboardAvoidingView>
  );
}
```

## If the Screen Uses React Navigation Stack (Most Likely Cause)

If this screen is pushed onto a stack navigator that has a tab bar at the bottom, the tab bar height is already handled by React Navigation — but the `KeyboardAvoidingView` doesn't know about it. Add this:

```tsx
// Get the bottom tab bar height from navigation:
import { useBottomTabBarHeight } from '@react-navigation/bottom-tabs';

// Inside the component:
let tabBarHeight = 0;
try {
  tabBarHeight = useBottomTabBarHeight();
} catch {
  tabBarHeight = 80; // fallback if not inside a tab navigator
}

// Pass to KeyboardAvoidingView:
<KeyboardAvoidingView
  style={{ flex: 1, backgroundColor: '#111' }}
  behavior={Platform.OS === 'ios' ? 'padding' : 'height'}
  keyboardVerticalOffset={tabBarHeight}
>
```

---

## If the Tab Bar is Still Overlapping

If the input bar appears but is still partially hidden behind the tab bar, the screen needs to render **outside** the tab navigator. Promote the AI screen to a modal stack:

```tsx
// In your navigation setup (App.tsx or RootNavigator.tsx):
// Move AIMatchAssistantScreen OUT of the Tab.Screen group
// and into a Stack.Screen with presentation: 'modal' or fullScreenModal

<Stack.Screen
  name="AIMatchAssistant"
  component={AIMatchAssistantScreen}
  options={{
    presentation: 'fullScreenModal',
    headerShown: false,
  }}
/>
```

This makes the screen render as a full-screen modal that slides up over the tab bar, so there's no tab bar overlapping the input at all — which is the cleanest solution for a chat screen.

---

## Summary

| Issue | Fix |
|---|---|
| Input bar hidden behind tab bar | Wrap screen in `KeyboardAvoidingView` with `keyboardVerticalOffset` set to tab bar height |
| Input bar cut off by home indicator | Apply `insets.bottom` from `useSafeAreaInsets` to `inputBar` paddingBottom |
| Content scrolls behind input bar | `ScrollView` contentContainerStyle `paddingBottom: 20` (content clears the input bar) |
| Keyboard covers input bar | `behavior="padding"` on iOS, `behavior="height"` on Android inside `KeyboardAvoidingView` |
| Cleanest fix | Promote AI screen to `fullScreenModal` stack screen — tab bar disappears entirely |
