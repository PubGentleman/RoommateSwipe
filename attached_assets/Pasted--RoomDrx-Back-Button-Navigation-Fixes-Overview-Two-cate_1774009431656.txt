# RoomDrx ŌĆö Back Button & Navigation Fixes

## Overview

Two categories of problems found across the full navigation audit:

**Category A ŌĆö Missing back buttons:** 3 sub-screens have no back button at all, leaving users with no way to go back except the hardware gesture.

**Category B ŌĆö Wrong screen on back press:** When tapping upgrade/payment/plans buttons from the Messages, Groups, Roommates, or Explore tabs, the user lands on the Plans or Payment screen. Pressing back takes them to the Profile main screen instead of where they came from. Root cause: cross-tab navigation via `navigate('Profile', { screen: 'Plans' })` pushes the screen onto the Profile stack, so `goBack()` pops to ProfileMain ŌĆö the wrong destination.

## Category B ŌĆö Wrong Screen on Back Press (Cross-Tab Navigation)

### Root Cause

Whenever a screen in Messages, Groups, Roommates, or Explore calls:
```tsx
navigation.navigate('Profile', { screen: 'Plans' })
navigation.navigate('Profile', { screen: 'Payment' })
```
React Navigation switches to the Profile tab and pushes Plans/Payment onto the Profile stack. When the user presses back, `goBack()` pops Plans/Payment and reveals **ProfileMain** ŌĆö the root of the Profile stack ŌĆö instead of returning to where the user came from.

### Fix: Promote Plans and Payment to the Root Navigator as Modals

Add Plans and Payment as **modal screens in `RootNavigator.tsx`**, above the tab navigator. Because they're registered at the root level, any `navigate('Plans')` or `navigate('Payment')` call from anywhere in the app will present them as a modal overlay on top of everything. When dismissed (goBack), the user returns to exactly the screen they were on ŌĆö regardless of which tab they came from.

---

### Step B1 ŌĆö Update `RootNavigator.tsx`

```tsx
// navigation/RootNavigator.tsx

import { PlansScreen } from '../screens/shared/PlansScreen';
import { PaymentScreen } from '../screens/shared/PaymentScreen';

// Inside the root Stack.Navigator, add these AFTER the 'Main' screen:

<Stack.Screen
  name="Main"
  component={MainComponent}
/>

{/* ŌöĆŌöĆ Modal screens ŌĆö accessible from any tab ŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆŌöĆ */}
<Stack.Screen
  name="Plans"
  component={PlansScreen}
  options={{
    presentation: 'modal',
    headerShown: false,
    animation: 'slide_from_bottom',
  }}
/>
<Stack.Screen
  name="Payment"
  component={PaymentScreen}
  options={{
    presentation: 'modal',
    headerShown: false,
    animation: 'slide_from_bottom',
  }}
/>
```

Also update the root param list type:
```tsx
// In RootNavigator.tsx, update the root Stack type:
type RootStackParamList = {
  Login: undefined;
  OnboardingProfile: undefined;
  OnboardingHostType: undefined;
  HostCompanySetup: undefined;
  HostAgentSetup: undefined;
  OnboardingPlan: undefined;
  Main: undefined;
  Plans: undefined;           // ŌåÉ add
  Payment: undefined;         // ŌåÉ add
};
```

### Step B2 ŌĆö Update All Cross-Tab Navigation Calls

Replace every `navigate('Profile', { screen: 'Plans' })` and `navigate('Profile', { screen: 'Payment' })` called from **outside the Profile tab** with a root-level `navigate('Plans')` or `navigate('Payment')`. These will now hit the root navigator's modal screen instead of the Profile stack.

**`screens/shared/ChatScreen.tsx`** (3 occurrences):
```tsx
// BEFORE:
(navigation as any).navigate('Profile', { screen: 'Payment' })
navigation.getParent()?.navigate('Profile', { screen: 'Plans' })

// AFTER:
(navigation as any).navigate('Payment')
(navigation as any).navigate('Plans')
```

(navigation as any).navigate('Profile', { screen: 'Payment' })

### Step B3 ŌĆö Keep Within-Profile-Stack Calls As-Is

The following calls navigate to Plans/Payment **from within the Profile tab** (ProfileMain, WhoLikedMeScreen, MyInterestsScreen, ProfileViewsScreen). Since they're already in the Profile stack AND Plans/Payment are also registered in ProfileStackNavigator, these continue to work correctly ŌĆö `navigate('Plans')` from within ProfileStack finds Plans in the current stack first. **Do not change these:**

```tsx
// ProfileScreen.tsx ŌĆö stays as is
navigation.navigate('Plans')
navigation.navigate('HostSubscription')
navigation.navigate('Payment')

// WhoLikedMeScreen.tsx ŌĆö stays as is
navigation.navigate('Plans' as never)

// ProfileViewsScreen.tsx ŌĆö stays as is
navigation.navigate('Plans' as never)

// MyInterestsScreen.tsx ŌĆö stays as is

```

### Step B4 ŌĆö Fix Dashboard Notification Bell

The bell icon on `HostDashboardScreen` navigates cross-tab to Notifications:
```tsx
// BEFORE (wrong ŌĆö back from Notifications goes to ProfileMain, not Dashboard):
parent.navigate('Profile', { screen: 'Notifications' });

// AFTER ŌĆö add Notifications to DashboardStack so back works correctly:
navigation.navigate('Notifications' as any);
```

Then register `NotificationsScreen` in `DashboardStack` inside `HostTabNavigator.tsx`:
```tsx
// navigation/HostTabNavigator.tsx
import { NotificationsScreen } from '../screens/shared/NotificationsScreen';

// Inside DashboardStackNavigator():
<DashboardStack.Screen name="Notifications" component={NotificationsScreen} />

// Also update DashboardStackParamList type:
export type HostDashboardStackParamList = {
  DashboardMain: undefined;
  CreateEditListing: { propertyId?: string };
  Analytics: undefined;
  Inquiries: undefined;
  Notifications: undefined;   // ŌåÉ add
  HostPricing: undefined;
  HostSubscription: undefined;
  ListingBoost: { listingId: string };
};
```

Similarly, the same bell icon pattern on `RoommatesScreen` navigates to Notifications cross-tab:
```tsx
// screens/renter/RoommatesScreen.tsx ŌĆö 2 occurrences
// BEFORE:
(navigation as any).navigate('Profile', { screen: 'Notifications' })

// AFTER:
(navigation as any).navigate('Notifications')
```

Since Notifications is already in `ProfileStackNavigator`, this call from the Roommates tab would need Notifications added to the Renter tab navigators as well, OR use the same root-modal approach. The cleanest fix for the renter side is to add Notifications to `RoommatesStackNavigator`:
```tsx
// navigation/RoommatesStackNavigator.tsx
import { NotificationsScreen } from '../screens/shared/NotificationsScreen';

<Stack.Screen name="RoommatesList" component={RoommatesScreen} />
<Stack.Screen name="AIAssistant" component={AIAssistantScreen} />
<Stack.Screen name="Notifications" component={NotificationsScreen} />  // ŌåÉ add
```
