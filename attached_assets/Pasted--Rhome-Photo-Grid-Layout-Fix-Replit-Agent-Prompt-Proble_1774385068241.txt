# Rhome — Photo Grid Layout Fix
## Replit Agent Prompt

---

## Problem

In `screens/shared/EditProfileScreen.tsx`, the profile photo section uses a **horizontal ScrollView**. Photos are 120×150px laid out in a row. On a phone screen only 2 photos are visible at a time — users never scroll, think they're limited to 2 photos, and stop uploading. The app already supports 6 photos but the layout hides that completely.

## What To Fix

Replace the horizontal scroll with a **3-column grid** that shows all 6 slots at once — filled slots show the photo, empty slots show a dashed placeholder. This is the standard pattern used by Hinge, Bumble, and Airbnb. Users immediately see how many slots they have and are prompted to fill them.

Replace it with this grid layout:

```tsx
{/* Photo count progress */}
<View style={styles.photoCountRow}>
  <Text style={[styles.photoCountText, { color: theme.textSecondary }]}>
    {photos.length} of 6 photos added
  </Text>
  {photos.length >= 3 ? (
    <View style={styles.photoCountGood}>
      <Ionicons name="checkmark-circle" size={14} color="#4CAF50" />
      <Text style={styles.photoCountGoodText}>Looking good!</Text>
    </View>
  ) : (
    <Text style={[styles.photoCountWarning, { color: '#FF6B6B' }]}>
      Add {3 - photos.length} more to go live
    </Text>
  )}
</View>

{/* 3-column photo grid */}
<View style={styles.photoGrid}>
  {Array.from({ length: 6 }).map((_, index) => {
    const photo = photos[index]
    const isFirst = index === 0

    if (photo) {
      // Filled slot
      return (
        <View key={index} style={styles.photoGridItem}>
          <Image source={{ uri: photo }} style={styles.photoGridImage} />

          {/* Main photo badge on first slot */}
          {isFirst && (
            <View style={[styles.mainBadge, { backgroundColor: theme.primary }]}>
              <Text style={styles.mainBadgeText}>Main</Text>
            </View>
          )}

          {/* Remove button */}
          <TouchableOpacity
            style={[styles.removePhotoButton, { backgroundColor: theme.error }]}
            onPress={() => removePhoto(index)}
          >
            <Ionicons name="close" size={14} color="#fff" />
          </TouchableOpacity>

          {/* Reorder arrows */}
          <View style={styles.reorderButtons} pointerEvents="box-none">
            {index > 0 && (
              <TouchableOpacity
                style={[styles.reorderButton, { backgroundColor: 'rgba(0,0,0,0.5)' }]}
                onPress={() => {
                  const newPhotos = [...photos]
                  const temp = newPhotos[index - 1]
                  newPhotos[index - 1] = newPhotos[index]
                  newPhotos[index] = temp
                  setPhotos(newPhotos)
                }}
              >
                <Ionicons name="arrow-back" size={14} color="#fff" />
              </TouchableOpacity>
            )}
            {index < photos.length - 1 && (
              <TouchableOpacity
                style={[styles.reorderButton, { backgroundColor: 'rgba(0,0,0,0.5)' }]}
                onPress={() => {
                  const newPhotos = [...photos]
                  const temp = newPhotos[index + 1]
                  newPhotos[index + 1] = newPhotos[index]
                  newPhotos[index] = temp
                  setPhotos(newPhotos)
                }}
              >
                <Ionicons name="arrow-forward" size={14} color="#fff" />
              </TouchableOpacity>
            )}
          </View>
        </View>
      )
    }

    // Empty slot — only show add button on the next available slot, placeholders for the rest
    const isNextSlot = index === photos.length
    return (
      <TouchableOpacity
        key={index}
        style={[
          styles.photoGridItem,
          styles.photoGridEmpty,
          {
            backgroundColor: theme.backgroundSecondary,
            borderColor: isNextSlot ? theme.primary : theme.border,
            borderStyle: 'dashed',
            opacity: isNextSlot ? 1 : 0.5,
          }
        ]}
        onPress={isNextSlot ? pickImage : undefined}
        activeOpacity={isNextSlot ? 0.7 : 1}
      >
        {isNextSlot ? (
          <>
            <Ionicons name="add-circle-outline" size={28} color={theme.primary} />
            <Text style={[styles.addPhotoLabel, { color: theme.primary }]}>Add Photo</Text>
          </>
        ) : (
          <Ionicons name="image-outline" size={24} color={theme.border} />
        )}
      </TouchableOpacity>
    )
  })}
</View>
```

### 4. Remove the DraggablePhoto component

The old `DraggablePhoto` component at the top of the file used `react-native-gesture-handler` for drag-to-reorder. The new grid uses simple left/right arrow buttons instead, which are more reliable on mobile and don't require gesture setup.

Delete the entire `DraggablePhoto` component (roughly lines 24–110) and remove any imports that were only used by it (check for `PanGestureHandler`, `Animated`, gesture handler imports).

If `Animated` or gesture handler imports are used elsewhere in the file, keep them — only remove if they were exclusively used by `DraggablePhoto`.

## Summary

| What changes | Why |
|---|---|
| Horizontal scroll → 3-column grid | Users can see all 6 slots at once |
| Empty slots shown as placeholders | Makes it obvious more photos can be added |
| Next available slot has Add button | Clear call to action |
| Future slots are dimmed | Shows capacity without being confusing |
| Left/right reorder arrows | Replaces drag gesture, more reliable |
| 3-photo minimum guard on save | Enforces quality before going live |
| Onboarding photo step updated | Consistent experience from day one |
