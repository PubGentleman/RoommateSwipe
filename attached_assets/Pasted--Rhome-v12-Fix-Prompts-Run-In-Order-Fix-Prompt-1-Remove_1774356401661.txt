**2. Fix `screens/shared/PaymentScreen.tsx`:**

const handlePayment = async () => {
  // 1. Create payment intent from backend
  const { data } = await supabase.functions.invoke('create-payment-intent', {
    body: { amount, currency: 'usd', userId: user.id }

  // 2. Initialize payment sheet

## Fix Prompt 3 — Fix Stripe Webhook Subscription Lookup

Paste this into Replit Agent:

---

In `supabase/functions/stripe-webhook/index.ts`, fix an inconsistency where subscription updates sometimes query the `profiles` table and sometimes query the `subscriptions` table for `stripe_customer_id`.

Standardize ALL subscription lookups to use this pattern consistently:

```typescript
// Step 1: Find user by stripe_customer_id in profiles table
const { data: profile } = await supabase
  .from('profiles')
  .select('id, role')
  .eq('stripe_customer_id', customerId)
  .single()

if (!profile) {
  console.error('No profile found for customer:', customerId)
  return
}

// Step 2: Update the subscriptions table using the profile's user id
await supabase
  .from('subscriptions')
  .upsert({
    user_id: profile.id,
    stripe_customer_id: customerId,
    stripe_subscription_id: subscriptionId,
    status: subscriptionStatus,
    plan: planName,
    current_period_end: periodEnd,
    updated_at: new Date().toISOString()
  }, { onConflict: 'user_id' })

// Step 3: Also update is_premium flag on profiles
await supabase
  .from('profiles')
  .update({
    is_premium: subscriptionStatus === 'active',
    plan: planName
  })
  .eq('id', profile.id)
```

Apply this pattern consistently to ALL webhook event handlers:
- `customer.subscription.created`
- `customer.subscription.updated`
- `customer.subscription.deleted`
- `invoice.payment_succeeded`
- `invoice.payment_failed`

Do NOT change the webhook signature verification logic at the top of the file.

## Supabase Setup — Add Stripe Secret Key

Do this manually (not via Replit Agent):

1. Go to your Supabase project dashboard
2. Click **Edge Functions** in the left sidebar
3. Click **Secrets** (or "Manage secrets")
4. Click **Add new secret**
5. Add:
   - Name: `STRIPE_SECRET_KEY`
   - Value: your `sk_test_...` key from Stripe (or `sk_live_...` when ready for production)
6. Click Save
7. Also add:
   - Name: `ANTHROPIC_API_KEY`
   - Value: your Anthropic API key (from console.anthropic.com)

These two secrets power all payment processing and AI features.
