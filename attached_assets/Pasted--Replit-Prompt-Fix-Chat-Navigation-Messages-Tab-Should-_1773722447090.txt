# Replit Prompt: Fix Chat Navigation — Messages Tab Should Always Show List First

---

## THE BUGS

**Bug 1 — React Navigation restores Chat instead of MessagesList when tapping the Messages tab.**
React Navigation remembers the last active screen in each tab stack. If a user was inside a Chat, switches tabs, and taps Messages again — they land directly in Chat instead of the conversation list. The user expects to see their inbox but sees a specific chat instead.

**Bug 2 — Many screens deep-link directly into Chat, skipping MessagesList entirely.**
Screens like RoommatesScreen, GroupsScreen, ExploreScreen, NotificationsScreen all navigate with `{ screen: 'Chat', params: {...} }` which bypasses the Messages list. While this is sometimes intentional (e.g. tapping a notification for a specific chat), the stack state gets left in a broken position when the user navigates back.

---

## FIX 1 — Reset Messages stack to list when tab is pressed

### `navigation/RenterTabNavigator.tsx`

Find the custom tab bar `onPress` handler inside `CustomTabBar`:

```tsx
const onPress = () => {
  const event = navigation.emit({
    type: 'tabPress',
    target: route.key,
    canPreventDefault: true,
  });
  if (!isFocused && !event.defaultPrevented) {
    navigation.navigate(route.name);
  }
};
```

**Replace with:**

```tsx
const onPress = () => {
  const event = navigation.emit({
    type: 'tabPress',
    target: route.key,
    canPreventDefault: true,
  });
  if (!event.defaultPrevented) {
    if (route.name === 'Messages') {
      // Always reset Messages stack to the list screen when tab is tapped
      // This prevents getting stuck inside a chat when returning to the tab
      navigation.navigate('Messages' as any, { screen: 'MessagesList' } as any);
    } else if (!isFocused) {
      navigation.navigate(route.name);
    }
  }
};
```

## FIX 2 — Update all screens that navigate directly to Chat

Every screen that does `navigate('Messages', { screen: 'Chat', params: {...} })` should instead navigate to the Messages tab first and then push Chat on top of MessagesList — so the back button always returns to the list, not an empty stack.

The correct pattern is a two-step navigation:
```tsx
// Step 1: Go to Messages tab and reset to MessagesList
navigation.navigate('Messages' as any, { screen: 'MessagesList' });
// Step 2: Then navigate to the specific Chat
navigation.navigate('Messages' as any, { screen: 'Chat', params: { conversationId, otherUser, inquiryGroup } });
```

Or use a helper function. Add this helper to each affected screen:

```tsx
const navigateToChat = (conversationId: string, otherUser?: any, inquiryGroup?: any) => {
  // First ensure MessagesList is in the stack, then push Chat on top
  (navigation as any).navigate('Messages', {
    screen: 'MessagesList',
  });
  setTimeout(() => {
    (navigation as any).navigate('Messages', {
      screen: 'Chat',
      params: { conversationId, otherUser, inquiryGroup },
    });
  }, 50);
};
```
