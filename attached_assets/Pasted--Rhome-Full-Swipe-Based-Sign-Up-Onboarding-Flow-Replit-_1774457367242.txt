# Rhome — Full Swipe-Based Sign-Up Onboarding Flow
## Replit Agent Prompt

---

## Overview

Replace the current sign-up screen with a fully swipe-driven multi-step onboarding flow. Each step is a full screen. When a user makes a definitive selection (tapping a card, choosing an option), the screen automatically slides left and the next step slides in from the right. Users can swipe right or tap the back chevron to go to the previous step at any time.

This pattern is used by Hinge, Bumble, and other top consumer apps. It feels native, fast, and focused — one question at a time, no overwhelming forms.

---

## Architecture

Create a single `SignUpFlow` component that manages all steps using a horizontal pager. Use `react-native-pager-view` if already installed, otherwise implement with `Animated.Value` + `PanResponder`.

```typescript
type SignUpStep =
  | 'account_type'    // Step 1 — Who are you?
  | 'credentials'     // Step 2 — Name, email, password
  | 'location'        // Step 3 — Where are you? (renter) / Property location (host)
  | 'details'         // Step 4 — Type-specific details
  | 'photo'           // Step 5 — Profile photo
  | 'complete';       // Done

interface SignUpState {
  accountType: 'renter' | 'individual' | 'agent' | 'company' | null;
  name: string;
  email: string;
  password: string;
  city: string;
  // Agent-specific
  licenseNumber: string;
  agencyName: string;
  // Company-specific
  companyName: string;
  // Individual host
  propertyType: string;
  // Photo
  profilePhoto: string | null;
}
```

## Navigation Rules

**Auto-advance (no button tap needed):**
- Step 1: Tapping an account type card
- Step 3: Tapping a city / location option
- Step 4: Tapping a single-choice option (property type, etc.)

**Manual advance (user taps Continue/Next):**
- Step 2: Credentials form (name, email, password)
- Step 4: If it contains text input fields (license number, company name)
- Step 5: Photo upload (also skippable)

**Going back:**
- Swipe right anywhere on the screen (PanResponder drag threshold: 80px)
- Tap the back chevron (top left)
- Back gesture snaps to previous step with spring animation, or snaps forward if not dragged far enough

**Never go back from Step 1** — no back button, no back swipe. Step 1 is the entry point.

## Swipe Gesture Implementation

Use `PanResponder` to handle the back swipe on every step after Step 1:

```typescript
const panResponder = useRef(
  PanResponder.create({
    onMoveShouldSetPanResponder: (_, gestureState) => {
      // Only capture horizontal right swipes
      return gestureState.dx > 10 && Math.abs(gestureState.dy) < 20;
    },
    onPanResponderMove: (_, gestureState) => {
      if (gestureState.dx > 0) {
        dragAnim.setValue(gestureState.dx);
      }
    },
    onPanResponderRelease: (_, gestureState) => {
      if (gestureState.dx > 80) {
        // Crossed threshold — snap back to previous step
        goBack();
      } else {
        // Not far enough — spring back to current step
        Animated.spring(dragAnim, {
          toValue: 0,
          useNativeDriver: true,
        }).start();
      }
    },
  })
).current;
```

## Progress Indicator

Show dots at the top of every step (except Step 1). Active dot is filled, completed dots are filled with lower opacity, future dots are empty.

```typescript
function ProgressDots({ current, total }: { current: number; total: number }) {
  return (
    <View style={styles.dotsRow}>
      {Array.from({ length: total }).map((_, i) => (
        <View
          key={i}
          style={[
            styles.dot,
            i < current && styles.dotComplete,
            i === current && styles.dotActive,
          ]}
        />
      ))}
    </View>
  );
}
```

Total steps: 5 (Steps 2–5 show dots, Step 1 shows none since it's the entry screen).

## Account Creation Timing

Create the Supabase auth user at the end of Step 2 (after name/email/password are validated), but mark the profile as `onboarding_complete = false`. This way if the user drops off mid-flow, you have their account and can resume. Set `onboarding_complete = true` only when they hit "Let's Go" on the final step.

---

## Animation Spec

All slide transitions:
- Duration: 280ms
- Easing: `Easing.out(Easing.cubic)`
- Direction: forward = slide left, back = slide right
- `useNativeDriver: true` on all animations

Back swipe spring (when released without crossing threshold):
- `tension: 120, friction: 10`
