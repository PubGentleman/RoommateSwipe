# Rhome — Safety Mode + Background Check Integration
## Replit Agent Prompt

---

## Overview

This prompt adds two deeply connected features:

1. **Safety Mode** — an opt-in toggle in profile settings. When enabled, incoming messages that contain phone numbers, Instagram handles, Snapchat, or WhatsApp info are automatically blurred with a tap-to-reveal option. Users feel protected without the app feeling restrictive. Designed especially for renters (often women) messaging unknown hosts.

2. **Background & Credit Check** — powered by Stripe Identity (already set up) + a credit bureau API (Checkr or Persona). A renter can request a check on themselves and share a verified badge with hosts. A host can require a background check before accepting a roommate. This is the #1 feature that cannot be replicated outside the app — it is Rhome's strongest retention and trust tool.

## Part 1 — Database Migration

Create file: `supabase/migrations/027_safety_background.sql`

```sql
-- Safety Mode setting on profiles
ALTER TABLE profiles
  ADD COLUMN IF NOT EXISTS safety_mode_enabled BOOLEAN DEFAULT FALSE;

-- Background checks table
CREATE TABLE IF NOT EXISTS background_checks (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID REFERENCES profiles(id) ON DELETE CASCADE NOT NULL,
  provider TEXT NOT NULL DEFAULT 'persona',       -- 'persona' | 'checkr'
  provider_inquiry_id TEXT,                        -- external ID from provider
  status TEXT NOT NULL DEFAULT 'pending',          -- pending | processing | approved | declined | expired
  check_type TEXT NOT NULL DEFAULT 'standard',     -- standard | premium (includes credit)
  identity_verified BOOLEAN DEFAULT FALSE,
  criminal_clear BOOLEAN DEFAULT FALSE,
  credit_score_range TEXT,                         -- 'excellent' | 'good' | 'fair' | 'poor' — never exact score
  report_url TEXT,                                 -- private signed URL, only shown to the user
  expires_at TIMESTAMPTZ DEFAULT (NOW() + INTERVAL '12 months'),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);

-- Index for fast lookup
CREATE INDEX IF NOT EXISTS idx_background_checks_user_id ON background_checks(user_id);
CREATE INDEX IF NOT EXISTS idx_background_checks_status ON background_checks(status);

-- Listing requirement: host can require background check before accepting
ALTER TABLE listings
  ADD COLUMN IF NOT EXISTS requires_background_check BOOLEAN DEFAULT FALSE;

-- RLS
ALTER TABLE background_checks ENABLE ROW LEVEL SECURITY;

-- Users can see their own checks
CREATE POLICY "Users see own checks" ON background_checks
  FOR SELECT USING (auth.uid() = user_id);

-- Users can insert their own checks
CREATE POLICY "Users insert own checks" ON background_checks
  FOR INSERT WITH CHECK (auth.uid() = user_id);

-- Service role can update (webhook processing)
CREATE POLICY "Service role updates" ON background_checks
  FOR UPDATE USING (true);

-- Verified badge view: what other users can see (no PII, no report URL)
CREATE OR REPLACE VIEW public_background_badges AS
SELECT
  user_id,
  status,
  identity_verified,
  criminal_clear,
  credit_score_range,
  check_type,
  expires_at,
  completed_at
FROM background_checks
WHERE status = 'approved'
  AND expires_at > NOW();
```

## Part 2 — Background Check Service (Persona Integration)

Persona (persona.com) is the recommended provider — it handles identity verification (government ID + selfie) and criminal background checks in one flow, has a React Native SDK, and is FCRA compliant.

Create file: `supabase/functions/create-background-check/index.ts`

```typescript
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

const corsHeaders = {
  'Access-Control-Allow-Origin': '*',
  'Access-Control-Allow-Headers': 'authorization, x-client-info, apikey, content-type',
}

serve(async (req) => {
  if (req.method === 'OPTIONS') return new Response('ok', { headers: corsHeaders })

  try {
    const supabase = createClient(
      Deno.env.get('SUPABASE_URL')!,
      Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
    )

    const authHeader = req.headers.get('Authorization')!
    const { data: { user } } = await supabase.auth.getUser(authHeader.replace('Bearer ', ''))
    if (!user) return new Response(JSON.stringify({ error: 'Unauthorized' }), { status: 401, headers: corsHeaders })

    const { checkType = 'standard' } = await req.json()

    // Get user profile for name/email
    const { data: profile } = await supabase
      .from('profiles')
      .select('full_name, email')
      .eq('id', user.id)
      .single()

    // Create Persona inquiry
    const personaRes = await fetch('https://withpersona.com/api/v1/inquiries', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${Deno.env.get('PERSONA_API_KEY')}`,
        'Content-Type': 'application/json',
        'Persona-Version': '2023-01-05',
      },
      body: JSON.stringify({
        data: {
          attributes: {
            'inquiry-template-id': checkType === 'premium'
              ? Deno.env.get('PERSONA_PREMIUM_TEMPLATE_ID')
              : Deno.env.get('PERSONA_STANDARD_TEMPLATE_ID'),
            'reference-id': user.id,
            fields: {
              'name-first': profile?.full_name?.split(' ')[0] || '',
              'name-last': profile?.full_name?.split(' ').slice(1).join(' ') || '',
              'email-address': user.email || '',
            },
          },
        },
      }),
    })

    const personaData = await personaRes.json()
    const inquiryId = personaData.data?.id
    const sessionToken = personaData.data?.attributes?.['session-token']

    if (!inquiryId) throw new Error('Failed to create Persona inquiry')

    // Create record in our DB
    const { data: check, error } = await supabase
      .from('background_checks')
      .insert({
        user_id: user.id,
        provider: 'persona',
        provider_inquiry_id: inquiryId,
        status: 'pending',
        check_type: checkType,
      })
      .select()
      .single()

    if (error) throw error

    return new Response(
      JSON.stringify({
        checkId: check.id,
        sessionToken,   // used to open Persona SDK in the app
        inquiryId,
      }),
      { headers: { ...corsHeaders, 'Content-Type': 'application/json' } }
    )
  } catch (err) {
    return new Response(JSON.stringify({ error: err.message }), { status: 500, headers: corsHeaders })
  }
})
```

## Part 3 — Persona Webhook Handler

Create file: `supabase/functions/persona-webhook/index.ts`

```typescript
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts'
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

serve(async (req) => {
  const payload = await req.json()
  const eventName = payload.data?.type  // 'inquiry.completed', 'inquiry.failed', etc.
  const attributes = payload.data?.attributes || {}
  const referenceId = attributes['reference-id']  // this is our user_id
  const inquiryId = payload.data?.id

  if (!referenceId || !inquiryId) {
    return new Response('Missing reference', { status: 400 })
  }

  const supabase = createClient(
    Deno.env.get('SUPABASE_URL')!,
    Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
  )

  if (eventName === 'inquiry.completed') {
    const status = attributes.status  // 'approved' or 'declined'
    const identityVerified = attributes['identity-verified'] === true
    const criminalClear = attributes['criminal-clear'] !== false  // default true unless flagged

    await supabase
      .from('background_checks')
      .update({
        status: status === 'approved' ? 'approved' : 'declined',
        identity_verified: identityVerified,
        criminal_clear: criminalClear,
        completed_at: new Date().toISOString(),
      })
      .eq('provider_inquiry_id', inquiryId)
      .eq('user_id', referenceId)

    // Send push notification to user
    const { data: profile } = await supabase
      .from('profiles')
      .select('push_token')
      .eq('id', referenceId)
      .single()

    if (profile?.push_token) {
      await fetch('https://exp.host/--/api/v2/push/send', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          to: profile.push_token,
          title: status === 'approved' ? '✅ Background Check Approved' : '❌ Background Check Declined',
          body: status === 'approved'
            ? 'Your verified badge is now showing on your profile.'
            : 'Your background check was not approved. Tap for details.',
          data: { type: 'background_check', status },
        }),
      })
    }
  }

  return new Response('ok', { status: 200 })
})
```

## Part 4 — Background Check Service (App Side)

Create file: `services/backgroundCheckService.ts`

```typescript
import { supabase } from './supabase'

export interface BackgroundCheckBadge {
  status: 'approved' | 'pending' | 'processing' | 'declined' | 'none'
  identityVerified: boolean
  criminalClear: boolean
  creditScoreRange?: string
  checkType: 'standard' | 'premium'
  expiresAt?: string
}

export const startBackgroundCheck = async (
  checkType: 'standard' | 'premium' = 'standard'
): Promise<{ sessionToken: string; inquiryId: string; checkId: string }> => {
  const { data: { session } } = await supabase.auth.getSession()

  const response = await fetch(
    `${process.env.EXPO_PUBLIC_SUPABASE_URL}/functions/v1/create-background-check`,
    {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${session?.access_token}`,
      },
      body: JSON.stringify({ checkType }),
    }
  )

  const data = await response.json()
  if (data.error) throw new Error(data.error)
  return data
}

export const getMyBackgroundCheck = async (): Promise<BackgroundCheckBadge> => {
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) return { status: 'none', identityVerified: false, criminalClear: false, checkType: 'standard' }

  const { data } = await supabase
    .from('background_checks')
    .select('*')
    .eq('user_id', user.id)
    .order('created_at', { ascending: false })
    .limit(1)
    .single()

  if (!data) return { status: 'none', identityVerified: false, criminalClear: false, checkType: 'standard' }

  return {
    status: data.status,
    identityVerified: data.identity_verified,
    criminalClear: data.criminal_clear,
    creditScoreRange: data.credit_score_range,
    checkType: data.check_type,
    expiresAt: data.expires_at,
  }
}

export const getUserBackgroundBadge = async (userId: string): Promise<BackgroundCheckBadge | null> => {
  const { data } = await supabase
    .from('public_background_badges')
    .select('*')
    .eq('user_id', userId)
    .single()

  if (!data) return null

  return {
    status: data.status,
    identityVerified: data.identity_verified,
    criminalClear: data.criminal_clear,
    creditScoreRange: data.credit_score_range,
    checkType: data.check_type,
    expiresAt: data.expires_at,
  }
}
```

### 7c — Message Blurring Component

Create file: `components/SafeMessageText.tsx`

```tsx
import React, { useState } from 'react'
import { Text, TouchableOpacity, StyleSheet, View } from 'react-native'
import { Ionicons } from '@expo/vector-icons'

interface Props {
  text: string
  safetyModeEnabled: boolean
  isOwnMessage: boolean
}

// Patterns to blur in Safety Mode
const SENSITIVE_PATTERNS = [
  /\b\d{3}[-.\s]?\d{3}[-.\s]?\d{4}\b/g,          // US phone numbers
  /\b\d{10}\b/g,                                     // 10-digit phone
  /@[\w.]{2,30}\b/g,                                 // @instagram @snapchat
  /\bsnapchat\s*:?\s*[\w.]+/gi,                     // Snapchat: username
  /\bwhatsapp\s*:?\s*[\d+\s()-]+/gi,               // WhatsApp numbers
  /\btelegram\s*:?\s*@?[\w.]+/gi,                   // Telegram handles
]

const containsSensitiveInfo = (text: string): boolean => {
  return SENSITIVE_PATTERNS.some(p => p.test(text))
}

export const SafeMessageText: React.FC<Props> = ({ text, safetyModeEnabled, isOwnMessage }) => {
  const [revealed, setRevealed] = useState(false)

  // Never blur your own messages
  const shouldBlur = safetyModeEnabled && !isOwnMessage && containsSensitiveInfo(text) && !revealed

  if (shouldBlur) {
    return (
      <View>
        <Text style={styles.blurredText}>
          {text.replace(/./g, '•')}
        </Text>
        <TouchableOpacity
          style={styles.revealButton}
          onPress={() => setRevealed(true)}
        >
          <Ionicons name="eye-outline" size={14} color="#FF6B6B" />
          <Text style={styles.revealText}>Tap to reveal contact info</Text>
        </TouchableOpacity>
      </View>
    )
  }

  return <Text style={styles.messageText}>{text}</Text>
}

const styles = StyleSheet.create({
  messageText: { color: '#fff', fontSize: 15, lineHeight: 22 },
  blurredText: {
    color: 'transparent',
    textShadowColor: '#aaa',
    textShadowOffset: { width: 0, height: 0 },
    textShadowRadius: 6,
    fontSize: 15,
    lineHeight: 22,
    letterSpacing: 2,
  },
  revealButton: {
    flexDirection: 'row', alignItems: 'center', gap: 4,
    marginTop: 4,
  },
  revealText: { color: '#FF6B6B', fontSize: 12, fontWeight: '600' },
})
```

## Why This Is Rhome's Strongest Retention Feature

A background check cannot be done over Instagram DMs. It cannot be done over text. It requires the app, it requires identity verification, it requires a payment ($15–$29 one-time). Once a renter has done it, their badge lives in Rhome. Once a host has seen it, they trust the platform. This is the feature that makes Rhome a serious product — not just a swipe app.
