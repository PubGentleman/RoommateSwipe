# Rhome — Shareable Profile Notes ("In Their Own Words")
## Replit Agent Prompt

---

## Overview

Add a free-text field where users write something about themselves that they want potential roommates to know — beyond what the structured questionnaire captures. They control exactly what gets shared. Claude reads this field when answering questions about that person, so matches get richer, more honest answers than the questionnaire data alone can provide.

This is the only cross-user AI data bridge in the app. Everything else stays private.

The feature appears at the end of the questionnaire as a final optional step, and in the Edit Profile screen. It shows on the profile card as an "In their own words" section.

---

## Part 1 — Database Migration

Run in Supabase SQL Editor:

```sql
-- Migration 030: Shareable profile notes

ALTER TABLE profiles
ADD COLUMN IF NOT EXISTS profile_note TEXT,
ADD COLUMN IF NOT EXISTS profile_note_updated_at TIMESTAMPTZ;

-- Add character limit constraint
ALTER TABLE profiles
ADD CONSTRAINT profile_note_max_length CHECK (char_length(profile_note) <= 500);

-- Index for searching notes (optional, for future search feature)
CREATE INDEX IF NOT EXISTS idx_profiles_profile_note
  ON profiles USING gin(to_tsvector('english', COALESCE(profile_note, '')));

COMMENT ON COLUMN profiles.profile_note IS
  'User-written free text shown to potential matches and readable by AI when answering questions about this user. User controls content entirely.';
```

## Part 6 — Update System Prompt Language

In `ai-ask-about-profile/index.ts`, update the system prompt to explicitly tell Claude how to use the note:

Find the `YOUR ROLE:` section and add:

```
- If the profile has an "In their own words" note, treat it as the most authentic source of information about this person — they wrote it themselves to describe how they actually live. Reference it directly when answering questions about their habits or vibe.
- Distinguish between structured profile data (what they answered in the questionnaire) and their personal note (what they chose to say in their own voice). The note often has nuance the questionnaire misses.
```

## Part 7 — Privacy Label in App

Anywhere the profile note appears in the UI (profile card, edit screen), add a small label making clear this is intentionally shared:

```tsx
<View style={{ flexDirection: 'row', alignItems: 'center', gap: 4, marginTop: 4 }}>
  <Ionicons name="eye-outline" size={11} color={theme.textSecondary} />
  <Text style={{ color: theme.textSecondary, fontSize: 11 }}>
    Visible to your matches · You control this
  </Text>
</View>
```

This label matters. Users need to know this field is intentionally public to matches — not a private note.

---

## What NOT to Do

- Do NOT pull private message history between users A and B into any other user's context
- Do NOT store anything from User A's conversations into User B's `user_ai_memory`
- Do NOT expose User B's `user_ai_memory` when User C asks about User B — that table is only loaded for the authenticated user's own context
- The `profile_note` is the ONLY cross-user AI data bridge. Everything else stays scoped to the user who created it.

## Summary

| What | Detail |
|---|---|
| New field | `profile_note TEXT` on `profiles` table, max 500 chars |
| Where users write it | Final step of questionnaire + Edit Profile screen |
| Where it shows | Profile card under bio — "IN THEIR OWN WORDS" with left border accent |
| What Claude does with it | References it directly when answering questions about that person — treats it as the most authentic data source |
| Privacy | Labeled "Visible to your matches · You control this" everywhere it appears |
| What it replaces | Nothing — it adds a human voice layer on top of the structured questionnaire data |
| Cross-user AI firewall | This is the only intentional cross-user data. Private messages, AI memory, and chat history remain scoped to their owner. |
