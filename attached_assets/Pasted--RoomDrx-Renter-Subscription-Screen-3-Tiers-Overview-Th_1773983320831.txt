# RoomDrx — Renter Subscription Screen (3 Tiers)

## Overview

The renter subscription screen is redesigned to visually match the host subscription screen — same card layout, same `PurchaseConfirmModal`, same tier card structure with feature rows, plan badges, and CTA buttons. The pricing, feature restrictions, and plan names are renter-specific and must not use host plan prices.

### Renter tiers

| Plan | Price | Audience |
|---|---|---|
| **Free** | $0 | Default for all new renter accounts |
| **Plus** | $14.99/mo | Renters who want more swipes, groups, and visibility |
| **Elite** | $29.99/mo | Power users who want every advantage in their search |

### Key differences from host plans

| | Host Plans | Renter Plans |
|---|---|---|
| **Tiers** | Free / Starter / Pro / Business | Free / Plus / Elite |
| **Pricing** | $19.99 / $49.99 / $99.99 | $14.99 / $29.99 |
| **Primary gate** | Listings, outreach, analytics | Swipes, groups, profile tools |
| **Business features** | Company branding, placement rank | N/A |

## Step 1 — Renter Plan Limits Constant

```ts
// constants/renterPlanLimits.ts

export type RenterPlan = 'free' | 'plus' | 'elite';

export interface RenterPlanLimits {
  plan: RenterPlan;
  label: string;
  price: string;
  dailySwipes: number;           // -1 = unlimited
  maxGroups: number;             // -1 = unlimited
  hasAdvancedFilters: boolean;
  canSeeWhoLiked: boolean;
  hasVerifiedBadge: boolean;
  hasMatchBreakdown: boolean;
  hasProfileBoost: boolean;
  hasPriorityInSearch: boolean;
  hasReadReceipts: boolean;
  hasIncognito: boolean;
  hasDedicatedSupport: boolean;
}

export const RENTER_PLAN_LIMITS: Record<RenterPlan, RenterPlanLimits> = {
  free: {
    plan: 'free',
    label: 'Free',
    price: '$0',
    dailySwipes: 10,
    maxGroups: 1,
    hasAdvancedFilters: false,
    canSeeWhoLiked: false,
    hasVerifiedBadge: false,
    hasMatchBreakdown: false,
    hasProfileBoost: false,
    hasPriorityInSearch: false,
    hasReadReceipts: false,
    hasIncognito: false,
    hasDedicatedSupport: false,
  },
  plus: {
    plan: 'plus',
    label: 'Plus',
    price: '$14.99/mo',
    dailySwipes: -1,
    maxGroups: 3,
    hasAdvancedFilters: true,
    canSeeWhoLiked: true,
    hasVerifiedBadge: true,
    hasMatchBreakdown: false,
    hasProfileBoost: false,
    hasPriorityInSearch: false,
    hasReadReceipts: false,
    hasIncognito: false,
    hasDedicatedSupport: false,
  },
  elite: {
    plan: 'elite',
    label: 'Elite',
    price: '$29.99/mo',
    dailySwipes: -1,
    maxGroups: -1,
    hasAdvancedFilters: true,
    canSeeWhoLiked: true,
    hasVerifiedBadge: true,
    hasMatchBreakdown: true,
    hasProfileBoost: true,
    hasPriorityInSearch: true,
    hasReadReceipts: true,
    hasIncognito: true,
    hasDedicatedSupport: true,
  },
};

export function getRenterPlanLimits(plan: RenterPlan): RenterPlanLimits {
  return RENTER_PLAN_LIMITS[plan] ?? RENTER_PLAN_LIMITS.free;
}

export function canSwipe(plan: RenterPlan, swipesToday: number): boolean {
  const { dailySwipes } = getRenterPlanLimits(plan);
  if (dailySwipes === -1) return true;
  return swipesToday < dailySwipes;
}

export function canJoinGroup(plan: RenterPlan, currentGroupCount: number): boolean {
  const { maxGroups } = getRenterPlanLimits(plan);
  if (maxGroups === -1) return true;
  return currentGroupCount < maxGroups;
}
```

## Step 3 — DB: Renter Plan Column

```sql
ALTER TABLE profiles
  ADD COLUMN IF NOT EXISTS renter_plan TEXT DEFAULT 'free'
    CHECK (renter_plan IN ('free', 'plus', 'elite')),
  ADD COLUMN IF NOT EXISTS renter_plan_expires_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS renter_stripe_subscription_id TEXT;
```

Stripe webhook resets `renter_plan` to `'free'` on `customer.subscription.deleted`.

## Step 5 — Renter Subscription Service

```ts
// services/renterSubscriptionService.ts

import { supabase } from '../lib/supabase';
import { useStripe } from '@stripe/stripe-react-native';

export async function subscribeToRenterPlan(plan: string): Promise<void> {
  const { data, error } = await supabase.functions.invoke('create-renter-subscription', {
    body: { plan },
  });
  if (error) throw error;

  const { initPaymentSheet, presentPaymentSheet } = useStripe();

  await initPaymentSheet({
    paymentIntentClientSecret: data.clientSecret,
    customerEphemeralKeySecret: data.ephemeralKey,
    customerId: data.customerId,
    merchantDisplayName: 'RoomDrx',
  });

  const { error: paymentError } = await presentPaymentSheet();
  if (paymentError) throw new Error(paymentError.message);
  // renter_plan updated by Stripe webhook on success
}
```

## Step 7 — Stripe Webhook (renter plan update)

The existing `stripe-webhook` edge function handles both host and renter subscriptions using `plan_type` metadata on the Stripe product:

```ts
case 'customer.subscription.updated':
case 'customer.subscription.deleted': {
  const sub      = event.data.object as Stripe.Subscription;
  const planType = sub.metadata.plan_type;  // 'host' or 'renter'
  const planTier = sub.metadata.plan_tier;  // 'plus' | 'elite' (renter) or 'starter' | 'pro' | 'business' (host)
  const userId   = sub.metadata.user_id;
  const isActive = sub.status === 'active';

  if (planType === 'renter') {
    await supabaseAdmin.from('profiles').update({
      renter_plan: isActive ? planTier : 'free',
      renter_stripe_subscription_id: sub.id,
    }).eq('id', userId);
  } else if (planType === 'host') {
    await supabaseAdmin.from('profiles').update({
      host_plan: isActive ? planTier : 'free',
      stripe_subscription_id: sub.id,
    }).eq('id', userId);
  }
  break;
}
```

## Complete Enforcement Map

| Restriction | Enforced in | What happens |
|---|---|---|
| 10 swipes/day (Free) | `canSwipe()` in swipe handler | Navigate to `RenterSubscription` |
| 1 group max (Free) | `canJoinGroup()` in `groupService` | Toast + navigate to `RenterSubscription` |
| 3 groups max (Plus) | `canJoinGroup()` in `groupService` | Toast + navigate to `RenterSubscription` (Elite) |
| See who liked (Plus+) | `canSeeWhoLiked()` | Lock overlay on likes list |
| Match breakdown (Elite) | `canSeeMatchBreakdown()` | Breakdown blurred + `LockedFeatureWall` |
| Profile boost (Elite) | `hasProfileBoost()` | Boost toggle locked |
| Read receipts (Elite) | `hasReadReceipts()` | Tick icons hidden |
| Incognito (Elite) | `canBrowseIncognito()` | Toggle locked + `LockedFeatureWall` |
| Priority in search (Elite) | DB query order | Automatic — Elite profiles ranked higher |
