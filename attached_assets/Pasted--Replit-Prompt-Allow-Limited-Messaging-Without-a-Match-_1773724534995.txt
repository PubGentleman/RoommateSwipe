# Replit Prompt — Allow Limited Messaging Without a Match (Open Cold Messaging)

## Overview
Currently non-Elite users see a "Match Required" paywall when trying to message someone they haven't mutually matched with. This change removes that hard wall and gives all users a daily cold message limit instead. Matched conversations remain prioritized and unlimited.

## New Messaging Rules

| Plan | Cold messages (non-matched) | Matched messages |
|------|-----------------------------|-----------------|
| Basic | 3 per day | Unlimited |
| Starter | 5 per day | Unlimited |
| Pro | 10 per day | Unlimited |
| Elite | Unlimited | Unlimited |

Cold message count resets at midnight. If the user has used their daily limit, show a friendly counter — not a paywall.

Also add (or update) a `getDailyColdMessageCount` function that reads from AsyncStorage using a key like `cold_msg_count_${userId}_${dateString}` so it resets automatically at midnight:

```ts
export async function getDailyColdMessageCount(userId: string): Promise<number> {
  const today = new Date().toISOString().split('T')[0]; // YYYY-MM-DD
  const key = `cold_msg_count_${userId}_${today}`;
  const stored = await AsyncStorage.getItem(key);
  return stored ? parseInt(stored, 10) : 0;
}

export async function incrementDailyColdMessageCount(userId: string): Promise<void> {
  const today = new Date().toISOString().split('T')[0];
  const key = `cold_msg_count_${userId}_${today}`;
  const current = await getDailyColdMessageCount(userId);
  await AsyncStorage.setItem(key, String(current + 1));
}
```

### 2. `contexts/AuthContext.tsx` — Update `canSendColdMessage()`

Find the `canSendColdMessage` function and replace its logic so that:
- All plans can send cold messages (just different limits)
- It checks today's count vs the plan limit
- Returns `{ canSend: boolean, reason?: string, remaining: number }`

```ts
const canSendColdMessage = async (): Promise<{ canSend: boolean; reason?: string; remaining: number }> => {
  if (!user) return { canSend: false, reason: 'Not logged in', remaining: 0 };

  const plan = user.subscription?.plan || 'basic';
  const limit = getDailyColdMessageLimit(plan);

  if (limit === Infinity) return { canSend: true, remaining: Infinity };

  const count = await getDailyColdMessageCount(user.id);
  const remaining = Math.max(0, limit - count);

  if (remaining <= 0) {
    return {
      canSend: false,
      reason: `You've used all ${limit} messages for today. Resets at midnight.`,
      remaining: 0,
    };
  }

  return { canSend: true, remaining };
};
```

Make sure `getDailyColdMessageLimit` and `getDailyColdMessageCount` are imported from `messagingUtils.ts`.

## What Does NOT Change

- Matched conversations remain unlimited for all plans
- Elite users keep unlimited cold messages as a premium perk
- The daily limit for regular messages (non-cold) is unchanged
- Super Interest functionality is unchanged
- The Explore/listing inquiry flow is unchanged

## Summary

After this change:
- Any user can tap "Send Message" on any profile and it will open a chat
- Free users get 3 attempts per day before hitting the limit
- Hitting the limit shows a friendly Alert (not a paywall modal) with an option to upgrade
- Paid plans get proportionally more daily cold messages
- The chat screen shows a subtle counter so users know their remaining cold message budget
