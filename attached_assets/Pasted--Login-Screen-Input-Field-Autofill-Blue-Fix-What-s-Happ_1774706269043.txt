# Login Screen — Input Field Autofill Blue Fix

## What's Happening

The email input field is showing a light blue/white background caused by iOS autofill styling overriding the custom dark theme. The password field is already fixed. This prompt targets the email field specifically and also cleans up the red border on the left icon container.

---

## Fix 1 — Kill iOS Autofill Blue Background on Email Input

**File:** `screens/auth/LoginScreen.tsx`

Find the email `TextInput` and add these props:

```tsx
<TextInput
  style={styles.input}
  placeholder="Email"
  placeholderTextColor="rgba(255,255,255,0.35)"
  keyboardType="email-address"
  autoCapitalize="none"
  autoCorrect={false}
  autoComplete="off"
  textContentType="none"          {/* ← stops iOS autofill blue tint */}
  importantForAutofill="no"       {/* ← Android equivalent */}
  value={email}
  onChangeText={setEmail}
  onFocus={() => setEmailFocused(true)}
  onBlur={() => setEmailFocused(false)}
/>
```

The key props are `textContentType="none"` and `autoComplete="off"` — these tell iOS not to apply its autofill UI styling to the field.

---

## Fix 2 — Force Transparent Background on TextInput

iOS sometimes ignores `textContentType="none"` and still applies the blue. Add a style override directly on the TextInput to force transparency:

```ts
input: {
  flex: 1,
  color: '#FFFFFF',
  fontSize: 15,
  paddingVertical: 14,
  paddingHorizontal: 12,
  backgroundColor: 'transparent',  // ← force transparent, let container handle bg
},
```

And make sure the parent container has the dark background:

```ts
inputContainer: {
  flexDirection: 'row',
  alignItems: 'center',
  backgroundColor: '#242538',           // ← dark background lives here
  borderWidth: 1,
  borderColor: 'rgba(255,255,255,0.1)',
  borderRadius: 14,
  marginBottom: 14,
  overflow: 'hidden',                    // ← clips any iOS autofill bleed
},
inputContainerFocused: {
  borderColor: '#F06464',               // ← coral on focus
},
```

The `overflow: 'hidden'` on the container clips iOS's autofill background before it bleeds outside the styled bounds.

---

## Fix 3 — Remove Red Border on Left Icon Container

The left icon (mail icon) has a red/orange border around it that shouldn't be there. Find the icon wrapper View and remove any `borderColor`, `borderWidth`, or `borderStyle` from it:

```tsx
{/* Before — has unwanted border */}
<View style={styles.iconWrapper}>
  <Feather name="mail" size={16} color="rgba(255,255,255,0.35)" />
</View>

{/* After — no border, just spacing */}
<View style={{ paddingLeft: 12 }}>
  <Feather name="mail" size={16} color="rgba(255,255,255,0.35)" />
</View>
```

Delete the `iconWrapper` style entirely from the StyleSheet if it exists — the icon just needs left padding, nothing else.

## Fix 4 — Full Email Input Assembly

Here is the complete correct email input block after all fixes:

```tsx
<View style={[styles.inputContainer, emailFocused && styles.inputContainerFocused]}>
  <View style={{ paddingLeft: 12 }}>
```

## Summary

| Fix | Change |
|---|---|
| iOS autofill blue | Add `textContentType="none"` and `autoComplete="off"` to email TextInput |
| Background bleed | Add `backgroundColor: 'transparent'` to input style, `overflow: 'hidden'` to container |
| Red icon border | Remove `borderColor`/`borderWidth` from left icon wrapper View |
| Container bg | Set `backgroundColor: '#242538'` on the inputContainer, not on the TextInput |
