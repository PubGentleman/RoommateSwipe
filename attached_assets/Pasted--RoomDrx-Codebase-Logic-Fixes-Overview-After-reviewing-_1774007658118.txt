## Fix 1 — Business Plan Price Bug (`utils/hostPricing.ts`)

**Problem:** `HOST_PLANS.business.price` is `99` instead of `99.99`. The purchase config correctly shows `$99.99/mo` in the modal, but the Monthly Cost Summary on the host subscription screen calculates from this value, so it displays **$99.00/mo** instead of **$99.99/mo**.

```ts
// utils/hostPricing.ts
// BEFORE:
business: {
  label: 'Host Business',
  price: 99,   // ❌ wrong

// AFTER:
business: {
  label: 'Host Business',
  price: 99.99, // ✅ correct
```

No other changes needed — `purchaseConfig.ts`, `planLimits.ts`, and all modals already have `$99.99`.

## Fix 2 — Stripe Webhook (`supabase/functions/stripe-webhook/index.ts`)

**Problem:** The current webhook has two issues:
1. It sets `plan = 'basic'` on cancellation instead of `'free'`. This is why `normalizeRenterPlan()` exists — it's a band-aid for the mismatch. The fix eliminates the need for that workaround.
2. It only handles renter subscriptions. Host subscriptions are stored in `AsyncStorage` locally and never synced to Supabase through the webhook, meaning a host who reinstalls the app loses their plan.

**Fix:** Add host plan handling using `plan_type` metadata on the Stripe subscription, and change all `'basic'` references to `'free'`.

```ts
// supabase/functions/stripe-webhook/index.ts

import Stripe from 'https://esm.sh/stripe@12.0.0?target=deno';
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2';

const stripe = new Stripe(Deno.env.get('STRIPE_SECRET_KEY')!, { apiVersion: '2023-10-16' });
const supabase = createClient(
  Deno.env.get('SUPABASE_URL')!,
  Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!
);

Deno.serve(async (req) => {
  const signature = req.headers.get('stripe-signature')!;
  const body = await req.text();

  let event: Stripe.Event;
  try {
    event = stripe.webhooks.constructEvent(body, signature, Deno.env.get('STRIPE_WEBHOOK_SECRET')!);
  } catch {
    return new Response('Webhook signature invalid', { status: 400 });
  }

  // ── Identity verification ────────────────────────────────────────────────
  if (event.type === 'identity.verification_session.verified') {
    const session = event.data.object as any;
    await supabase.from('users')
      .update({
        identity_verified: true,
        identity_verified_at: new Date().toISOString(),
      })
      .eq('id', session.metadata.user_id);
    return new Response(JSON.stringify({ received: true }), { status: 200 });
  }

  // ── Subscription events ──────────────────────────────────────────────────
  const subscription = event.data.object as Stripe.Subscription;
  const customerId = subscription.customer as string;

  // planType tells us whether this is a renter or host subscription
  const planType = subscription.metadata?.plan_type ?? 'renter'; // default to renter for legacy
  const planTier = subscription.metadata?.plan ?? 'free';
  const billingCycle = subscription.metadata?.billing_cycle ?? 'monthly';

  // ── Renter subscription ──────────────────────────────────────────────────
  if (planType === 'renter') {
    const { data: sub } = await supabase
      .from('subscriptions')
      .select('user_id')
      .eq('stripe_customer_id', customerId)
      .single();

    if (!sub) return new Response('User not found', { status: 404 });

    const updates: Record<string, any> = {
      stripe_subscription_id: subscription.id,
      current_period_end: new Date(subscription.current_period_end * 1000).toISOString(),
      cancel_at_period_end: subscription.cancel_at_period_end,
      updated_at: new Date().toISOString(),
    };

    if (
      event.type === 'customer.subscription.created' ||
      event.type === 'customer.subscription.updated'
    ) {
      updates.status = 'active';
      updates.plan = planTier;           // ✅ 'plus' | 'elite' — no more 'basic'
      updates.billing_cycle = billingCycle;
    } else if (event.type === 'customer.subscription.deleted') {
      updates.status = 'cancelled';
      updates.plan = 'free';             // ✅ 'free' — not 'basic'
    } else if (event.type === 'invoice.payment_failed') {
      updates.status = 'past_due';
    }

    await supabase.from('subscriptions').update(updates).eq('user_id', sub.user_id);
  }

  // ── Host subscription ────────────────────────────────────────────────────
  if (planType === 'host') {
    // Look up user by Stripe customer ID (stored on profiles)
    const { data: profile } = await supabase
      .from('profiles')
      .select('id')
      .eq('stripe_customer_id', customerId)
      .single();

    if (!profile) return new Response('Host profile not found', { status: 404 });

    if (
      event.type === 'customer.subscription.created' ||
      event.type === 'customer.subscription.updated'
    ) {
      await supabase.from('profiles').update({
        host_plan: planTier,                // 'starter' | 'pro' | 'business'
        host_stripe_subscription_id: subscription.id,
        host_plan_billing_cycle: billingCycle,
        host_plan_expires_at: new Date(subscription.current_period_end * 1000).toISOString(),
        updated_at: new Date().toISOString(),
      }).eq('id', profile.id);
    } else if (event.type === 'customer.subscription.deleted') {
      await supabase.from('profiles').update({
        host_plan: 'free',                  // ✅ reset to free on cancel
        host_stripe_subscription_id: null,
        host_plan_expires_at: null,
        updated_at: new Date().toISOString(),
      }).eq('id', profile.id);
    }
  }

  return new Response(JSON.stringify({ received: true }), { status: 200 });
});
```

**DB columns to add for host plan persistence:**

```sql
ALTER TABLE profiles
  ADD COLUMN IF NOT EXISTS stripe_customer_id TEXT,
  ADD COLUMN IF NOT EXISTS host_plan TEXT DEFAULT 'free'
    CHECK (host_plan IN ('free', 'starter', 'pro', 'business')),
  ADD COLUMN IF NOT EXISTS host_stripe_subscription_id TEXT,
  ADD COLUMN IF NOT EXISTS host_plan_billing_cycle TEXT DEFAULT 'monthly',
  ADD COLUMN IF NOT EXISTS host_plan_expires_at TIMESTAMPTZ;
```

**When creating a host Stripe subscription**, include `plan_type` in metadata:

```ts
// In create-subscription edge function (host flow):
const subscription = await stripe.subscriptions.create({
  customer: customerId,
  items: [{ price: priceId }],
  metadata: {
    plan_type: 'host',           // ← required for webhook routing
    plan: 'starter',             // 'starter' | 'pro' | 'business'
    billing_cycle: 'monthly',
    user_id: userId,
  },
});
```

**When creating a renter Stripe subscription**, include `plan_type`:

```ts
// In create-subscription edge function (renter flow):
const subscription = await stripe.subscriptions.create({
  customer: customerId,
  items: [{ price: priceId }],
  metadata: {
    plan_type: 'renter',         // ← required for webhook routing
    plan: 'plus',                // 'plus' | 'elite'
    billing_cycle: 'monthly',
    user_id: userId,
  },
});
```

## Fix 4 — `normalizeRenterPlan` Legacy Cleanup (`constants/renterPlanLimits.ts`)

**Problem:** `normalizeRenterPlan` maps `'basic' → 'free'` because the webhook used to store `'basic'` on cancellation. After Fix 2, the webhook correctly stores `'free'`. However, existing users in the DB might still have `plan = 'basic'`. The function should stay for backward compatibility but be clearly documented.

```ts
// constants/renterPlanLimits.ts

/**
 * Normalizes legacy plan values from the database.
 * 'basic' was used in older webhook versions — now the webhook uses 'free'.
 * Keep this function to handle any existing 'basic' rows in the subscriptions table.
 */
export function normalizeRenterPlan(plan: string | null | undefined): RenterPlan {
  if (!plan || plan === 'basic') return 'free';   // 'basic' is legacy for 'free'
  if (plan === 'plus' || plan === 'elite') return plan;
  return 'free';
}
```

Run a one-time migration to clean up existing `'basic'` rows in the database:

```sql
-- Run once in Supabase SQL editor to fix legacy 'basic' plan rows
UPDATE subscriptions
SET plan = 'free'
WHERE plan = 'basic';
```

## Fix 6 — Outreach Quota Resets at User's Local Midnight (`services/hostOutreachService.ts`)

**Problem:** `todayKey()` uses `new Date().toISOString().slice(0, 10)` which returns the current date in **UTC**. This means a host in New York sees their quota reset at 7–8 PM local time instead of midnight. Hosts in timezones ahead of UTC (e.g. London, Tokyo) see it reset even earlier in their day.

**Fix:** Use `Intl.DateTimeFormat` with the device's local timezone so the date key reflects the user's local calendar day, causing the quota to reset at **midnight in their timezone**.

```ts
// services/hostOutreachService.ts

// BEFORE:
function todayKey(): string {
  return new Date().toISOString().slice(0, 10); // UTC date — resets at midnight UTC
}

// AFTER:
function todayKey(): string {
  const tz = Intl.DateTimeFormat().resolvedOptions().timeZone;
  return new Intl.DateTimeFormat('en-CA', {
    timeZone: tz,
    year: 'numeric',
    month: '2-digit',
    day: '2-digit',
  }).format(new Date());
  // Returns 'YYYY-MM-DD' in the device's local timezone
  // Resets at midnight local time regardless of the user's timezone
}
```

**Why `en-CA`?** The Canada English locale formats dates as `YYYY-MM-DD` natively — matching the existing key format used in Supabase (`host_outreach_daily_quota.date_key`) and AsyncStorage. No key format change is needed.

**Behavior after this fix:**
- A host in New York (UTC-5) sees their quota reset at **midnight EST/EDT** ✅
- A host in London (UTC+0/+1) sees their quota reset at **midnight GMT/BST** ✅
- A host in Tokyo (UTC+9) sees their quota reset at **midnight JST** ✅
- Each user's `date_key` differs if they're in different timezones — this is correct and expected

**No DB schema change required.** The `date_key` column in `host_outreach_daily_quota` stays as `TEXT` — it already accepts any `YYYY-MM-DD` string. The RLS policy and index on `date_key` are unaffected.

> **Edge case:** If a user travels to a different timezone, their local `todayKey()` changes immediately. This is acceptable — they get a fresh counter in the new timezone. This is the same behavior as virtually every app with daily limits (Duolingo, Twitter, etc.).
