# RoomDrx — Host Analytics: Basic vs Advanced Differentiation

## Overview

The `analyticsLevel` field exists in `planLimits.ts` with values `'none' | 'basic' | 'advanced'` but is never read anywhere. Both Pro and Business hosts currently see the identical analytics screen. This prompt implements the split:

- **Free / Starter** → `LockedFeatureWall` (already working ✅)
- **Pro** → Basic Analytics (what's currently built — clean it up slightly)
- **Business** → Advanced Analytics (everything in Basic + 4 new sections)

---

## Tier Breakdown

### Basic — Pro ($49.99/mo)
All currently-existing sections, unchanged:
- Overview grid (Total Listings, Active, Inquiries, Accept Rate)
- Status pills (Pending, Accepted, Passed, Rented, Paused)
- Conversion funnel — **change estimated views/saves to show a clear disclaimer label "Estimated" in amber**
- Per-listing inquiry breakdown (bar chart)
- 30-day Monthly Trend

### Advanced — Business ($99.99/mo)
Everything in Basic, plus sections gated by plan AND host type:

**All Business hosts (Individual, Company, Agent):**
1. **Real Listing Views** — replaces estimated views with actual tracked impressions
2. **Renter Demographics** — budget ranges, move-in dates, lifestyle tags
3. **Outreach Performance** — messages sent, outreach-to-inquiry conversion
4. **90-Day History Toggle** — extends the trend window from 30 to 90 days

**Company hosts only (`host_type === 'company'`):**
5. **Vacancy Rate + Occupancy** — % of portfolio occupied, vacant, paused
6. **Projected Monthly Revenue** — sum of rent across all rented units
7. **Avg. Days to Fill** — listing creation → first accepted inquiry
8. **Portfolio Comparison Table** — all listings side-by-side: views, inquiries, status
9. **Portfolio CSV Export** — full portfolio performance data

**Agent hosts only (`host_type === 'agent'`):**
5. **Response Time Tracker** — avg hours from inquiry received → host responds
6. **Listing Scorecard** — health score (0–100) per listing: view-to-inquiry rate, accept rate, listing age
7. **Client Report CSV Export** — formatted for sharing with property owner clients

## Step 1 — New DB Table: `listing_views`

Mirrors the existing `profile_views` table structure.

```sql
CREATE TABLE IF NOT EXISTS listing_views (
  id          UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  listing_id  UUID NOT NULL REFERENCES listings(id) ON DELETE CASCADE,
  viewer_id   UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  viewed_at   TIMESTAMPTZ DEFAULT now() NOT NULL,
  UNIQUE (listing_id, viewer_id, date_trunc('day', viewed_at)) -- one impression per renter per day per listing
);

-- RLS
ALTER TABLE listing_views ENABLE ROW LEVEL SECURITY;

-- Hosts can read views on their own listings
CREATE POLICY "hosts_read_own_listing_views" ON listing_views
  FOR SELECT USING (
    listing_id IN (
      SELECT id FROM listings WHERE host_id = auth.uid()
    )
  );

-- Renters can insert their own views
CREATE POLICY "renters_insert_own_views" ON listing_views
  FOR INSERT WITH CHECK (viewer_id = auth.uid());

-- Index for fast analytics queries
CREATE INDEX IF NOT EXISTS idx_listing_views_listing_id ON listing_views(listing_id);
CREATE INDEX IF NOT EXISTS idx_listing_views_viewed_at ON listing_views(viewed_at);
```

## Step 2 — New Service Function: `recordListingView` + `getListingViews`

Add to `services/listingService.ts`:

```ts
/**
 * Called when a renter opens a listing detail. Records one impression per
 * renter per listing per calendar day (enforced by DB UNIQUE constraint).
 */
export async function recordListingView(listingId: string): Promise<void> {
  const { data: { user } } = await supabase.auth.getUser();
  if (!user) return;

  // Silently ignore errors — analytics tracking should never break the UI
  await supabase
    .from('listing_views')
    .upsert(
      { listing_id: listingId, viewer_id: user.id, viewed_at: new Date().toISOString() },
      { onConflict: 'listing_id,viewer_id,date_trunc' }
    )
    .then(() => {});
}

export interface ListingViewStats {
  listingId: string;
  totalViews: number;
  last30Days: number;
  last90Days: number;
}

/**
 * Returns view counts for an array of listing IDs (host's own listings).
 * Called from HostAnalyticsScreen for Business-tier hosts only.
 */
export async function getListingViewStats(listingIds: string[]): Promise<ListingViewStats[]> {
  if (!listingIds.length) return [];

  const ninety = new Date(Date.now() - 90 * 24 * 60 * 60 * 1000).toISOString();
  const thirty  = new Date(Date.now() - 30 * 24 * 60 * 60 * 1000).toISOString();

  const { data } = await supabase
    .from('listing_views')
    .select('listing_id, viewed_at')
    .in('listing_id', listingIds)
    .gte('viewed_at', ninety);

  if (!data) return listingIds.map(id => ({ listingId: id, totalViews: 0, last30Days: 0, last90Days: 0 }));

  return listingIds.map(id => {
    const rows = data.filter(r => r.listing_id === id);
    return {
      listingId: id,
      totalViews: rows.length,
      last30Days: rows.filter(r => r.viewed_at >= thirty).length,
      last90Days: rows.length,
    };
  });
}
```

## Step 3 — Record Views in ExploreScreen

In `screens/renter/ExploreScreen.tsx`, whenever the renter opens a listing detail sheet, call `recordListingView`. Look for where the listing detail modal opens — typically when a card is pressed or the detail sheet becomes visible:

```ts
// screens/renter/ExploreScreen.tsx
import { recordListingView } from '../../services/listingService';

// When the renter opens a listing detail (where selectedProperty is set):
useEffect(() => {
  if (selectedProperty?.id) {
    recordListingView(selectedProperty.id);
  }
}, [selectedProperty?.id]);
```

This fires once per listing open. The DB UNIQUE constraint ensures at most one impression per renter per listing per calendar day, so rapid re-opens don't inflate counts.

```tsx

const ACCENT    = '#ff6b5b';

const GREEN     = '#3ECF8E';
const AMBER     = '#f59e0b';
const CARD_BG   = '#1a1a1a';
const BG        = '#111';

export const HostAnalyticsScreen = () => {
  const { user, getHostPlan } = useAuth();
  const navigation = useNavigation<any>();
  const hostPlan = getHostPlan() as HostPlan;
  const analyticsLocked = !canAccessAnalytics(hostPlan);
  const analyticsLevel  = getPlanLimits(hostPlan).analyticsLevel; // 'basic' | 'advanced'

  const { theme } = useTheme();

  const [outreachLog, setOutreachLog] = useState<any[]>([]); // OutreachLogEntry[]
  const [trendWindow, setTrendWindow] = useState<30 | 90>(30); // Business-only toggle

  useFocusEffect(
    useCallback(() => { loadData(); }, [user?.id])

  const loadData = async () => {

        if (isAdvanced) {
          const stats = await getListingViewStats(mapped.map(p => p.id));

        }
      } else {
        setProperties([]);
      }
    } catch {
      const allProperties = await StorageService.getProperties();
      setProperties(allProperties.filter(p => p.hostId === user.id));
    }

    // — Interest cards (inquiries) —
    try {

      const mapped: InterestCard[] = (supaCards || []).map((c: any) => ({
        id:                c.id,
        renterId:          c.sender?.id || c.sender_id,
        renterName:        c.sender?.full_name || 'Unknown',
        renterPhoto:       c.sender?.avatar_url,
        hostId:            c.recipient_id || user.id,
        propertyId:        c.listing_id || '',
        propertyTitle:     c.listing_title || '',
        status:            c.status || 'pending',
        isSuperInterest:   c.action === 'super_interest',
        compatibilityScore: c.compatibility_score || 0,
        budgetRange:       c.budget_range || '',
        moveInDate:        c.move_in_date || '',
        lifestyleTags:     c.lifestyle_tags || [],
        personalNote:      c.personal_note || '',
        createdAt:         c.created_at || new Date().toISOString(),
        respondedAt:       c.responded_at,
      }));
      setInquiries(mapped);
    } catch {
      const cards = await StorageService.getInterestCardsForHost(user.id);
      setInquiries(cards);
    }

    if (isAdvanced) {
      try {
        const log = await getOutreachLogForHost(user.id);
        setOutreachLog(log);
      } catch {
        setOutreachLog([]);
      }
    }
  };

  // ── Computed values ────────────────────────────────────────────────────────

  const totalListings    = properties.length;
  const activeListings   = properties.filter(p => p.available && !p.rentedDate).length;
  const rentedListings   = properties.filter(p => !!p.rentedDate).length;
  const pausedListings   = properties.filter(p => !p.available && !p.rentedDate).length;

  const totalInquiries    = inquiries.length;
  const pendingInquiries  = inquiries.filter(i => i.status === 'pending').length;
  const acceptedInquiries = inquiries.filter(i => i.status === 'accepted').length;

    ? Math.round((acceptedInquiries / totalInquiries) * 100) : 0;

  const estimatedSaves     = totalInquiries * 8;

  const displayViewsLabel  = isAdvanced ? 'Listing Views' : 'Est. Views';

  const now = Date.now();
  const windowMs      = trendWindow * 24 * 60 * 60 * 1000;
  const priorWindowMs = windowMs * 2;
  const inquiriesInWindow = inquiries.filter(i =>
    now - new Date(i.createdAt).getTime() < windowMs
  ).length;
  const inquiriesPrior = inquiries.filter(i => {
    const age = now - new Date(i.createdAt).getTime();
    return age >= windowMs && age < priorWindowMs;
  }).length;
  const trendDelta    = inquiriesInWindow - inquiriesPrior;
  const trendPositive = trendDelta > 0;
  const trendNeutral  = trendDelta === 0;

  const perListingInquiries = properties.map(p => {

    const label = i.budgetRange; // use raw label for now

  const topBudgets = Object.entries(budgetBuckets)
    .sort((a, b) => b[1] - a[1])
    .slice(0, 5);

  // Move-in date distribution — group by month
  const moveInMonths: Record<string, number> = {};
  inquiries.forEach(i => {
    if (!i.moveInDate) return;
    const month = new Date(i.moveInDate).toLocaleDateString('en-US', { month: 'short', year: 'numeric' });

  });
  const moveInSorted = Object.entries(moveInMonths).sort((a, b) => b[1] - a[1]).slice(0, 5);

  // Top lifestyle tags
  const tagCounts: Record<string, number> = {};
  inquiries.forEach(i => {

      tagCounts[tag] = (tagCounts[tag] || 0) + 1;
    });
  });
  const topTags = Object.entries(tagCounts).sort((a, b) => b[1] - a[1]).slice(0, 8);

  // (Compatibility score not shown to hosts — renter-facing metric only)

  // ── Advanced: Outreach performance ────────────────────────────────────────

  const thirtyDaysAgo = Date.now() - 30 * 24 * 60 * 60 * 1000;
  const outreachThisMonth = outreachLog.filter(
    e => new Date(e.sentAt).getTime() > thirtyDaysAgo
  ).length;
  // Outreach-to-inquiry conversion: inquiries whose createdAt is within 7 days of an outreach send to same group
  const outreachConversions = inquiries.filter(i => {
    const inquiryTime = new Date(i.createdAt).getTime();
    return outreachLog.some(e =>
      e.groupId === i.renterId && // groupId on outreach, renterId on inquiry (renter who inquired after outreach)
      Math.abs(inquiryTime - new Date(e.sentAt).getTime()) < 7 * 24 * 60 * 60 * 1000
    );
  }).length;
  const outreachConversionRate = outreachThisMonth > 0
    ? Math.round((outreachConversions / outreachThisMonth) * 100) : 0;

  // ── CSV Export (Business only) ─────────────────────────────────────────────

  const handleExport = async () => {
    const header = 'Renter,Listing,Status,Budget,Move-In,Compatibility,Date\n';
    const rows = inquiries.map(i =>
      [
        i.renterName,
        i.propertyTitle,
        i.status,
        i.budgetRange,

      title: 'RoomDrx Analytics Export',
```

```ts
/**
 * Returns the host's full outreach log entries for analytics use.
 * Tries Supabase first, falls back to AsyncStorage.
 */
export async function getOutreachLogForHost(hostId: string): Promise<OutreachLogEntry[]> {
  try {
    const { data } = await supabase
      .from('host_outreach_log')
      .select('id, host_id, group_id, listing_id, message, sent_at')
      .eq('host_id', hostId)
      .order('sent_at', { ascending: false })

        id:        r.id,
        hostId:    r.host_id,
        groupId:   r.group_id,
        listingId: r.listing_id,
        message:   r.message,
        sentAt:    r.sent_at,
      }));
    }
  } catch {}

  // Fallback to local log
  return getLocalLog();
}
```

#### 2 — Listing Performance Scorecard

A per-listing "health score" combining view-to-inquiry rate, accept rate, and days listed. Agents can use this to identify listings that need attention and to show property owners objective performance data.

```tsx
// Health score per listing (0–100)
// Weighted: 40% view-to-inquiry rate, 40% accept rate, 20% recency (newer = better)
function getListingHealthScore(
  viewCount: number,
  inquiryCount: number,
  accepted: number,
  daysListed: number,
): number {
  const vtiRate    = viewCount > 0 ? (inquiryCount / viewCount) : 0;      // 0–1
  const acceptRate = inquiryCount > 0 ? (accepted / inquiryCount) : 0;    // 0–1
  const recency    = Math.max(0, 1 - daysListed / 90);                    // 1 if new, 0 if 90+ days old

  const score = (vtiRate * 40) + (acceptRate * 40) + (recency * 20);
  return Math.min(100, Math.round(score * 100));
}
```
