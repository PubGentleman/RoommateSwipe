# Rhome — Host Pricing Filtered by Host Type
## Replit Agent Prompt

---

## Overview

When a user selects their host type during onboarding (individual, agent, or company), they should only ever see pricing plans that are relevant to them. Right now `HostSubscriptionScreen.tsx` and `HostPricingScreen.tsx` show all plans to everyone. This prompt filters the displayed plans based on the `hostType` stored in the user's auth context.

**Rules:**
- `individual` → sees **Starter**, **Pro**, **Business** (personal landlord tiers)
- `agent` → sees **Agent Starter**, **Agent Pro**, **Agent Business** (agent-specific tiers)
- `company` → sees **Company Starter**, **Company Pro**, **Company Enterprise** (company tiers)

If for any reason `hostType` is undefined, default to showing individual plans.

---

## Part 1 — Read `hostType` from AuthContext

In `contexts/AuthContext.tsx`, confirm that `hostType` is stored on the user/profile object and exported. It should already exist from the host onboarding flow. If it is on the `profiles` table, it will be something like:

```typescript
const hostType = profile?.host_type as 'individual' | 'agent' | 'company' | undefined;
```

Make sure `hostType` (or `host_type`) is included in the profile SELECT query and available via the auth context throughout the app.

```typescript
const { user, profile } = useAuth();
```


### Step 2c — Filter plans in the component

Replace any hardcoded plan array with the filtered set:

```typescript
const plans = HOST_PLANS_BY_TYPE[hostType] ?? HOST_PLANS_BY_TYPE['individual'];
```

Then render `plans.map(...)` instead of the current hardcoded list.

### Step 2d — Update the screen title to reflect host type

Show a subtitle under the screen title so users know they're seeing the right plans:

## Part 4 — Handle Enterprise "Contact Sales"

For company Enterprise tier, the CTA is "Contact Sales" rather than a payment button. When this plan is tapped:

```typescript
if (plan.id === 'host_company_enterprise') {
  // Open email or navigate to contact screen
  Linking.openURL('mailto:enterprise@rhomeapp.com?subject=Enterprise%20Plan%20Inquiry');
  return;
}
```

In the RevenueCat dashboard, add new entitlements for agent and company tiers:

- `host_agent_starter`
- `host_agent_pro`
- `host_agent_business`
- `host_company_starter`
- `host_company_pro`

In `utils/revenueCatService.ts`, update the `syncSubscriptionToSupabase` function to map these new entitlements:

```typescript
// Add to the entitlement mapping block:
else if (activeEntitlements.includes('host_agent_business')) { plan = 'agent_business'; planType = 'agent'; }
else if (activeEntitlements.includes('host_agent_pro')) { plan = 'agent_pro'; planType = 'agent'; }
else if (activeEntitlements.includes('host_agent_starter')) { plan = 'agent_starter'; planType = 'agent'; }
else if (activeEntitlements.includes('host_company_pro')) { plan = 'company_pro'; planType = 'company'; }
else if (activeEntitlements.includes('host_company_starter')) { plan = 'company_starter'; planType = 'company'; }
```

## Summary

| Host Type | Plans Shown | App Store Group |
|---|---|---|
| Individual | Starter · Pro · Business | Rhome Host |
| Agent | Agent Starter · Agent Pro · Agent Business | Rhome Agent |
| Company | Company Starter · Company Pro · Enterprise (contact) | Rhome Company |

No host type ever sees another type's pricing. The filter happens at the component level using `hostType` from `AuthContext` — one line of logic, clean and maintainable.
