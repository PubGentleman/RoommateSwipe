**Fix swipe delay on RoommatesScreen and GroupsScreen — fire Supabase calls in background**

Both screens block the swipe animation waiting for Supabase to respond. The fix is to advance the card immediately and fire the database call in the background.

**In `screens/renter/RoommatesScreen.tsx`:**

In `handleSwipeAction`, advance to the next card immediately before calling `processSwipeAsync`:

```typescript
const handleSwipeAction = async (action: 'like' | 'nope' | 'superlike') => {
  if (!currentProfile) return;
  const profileId = currentProfile.profile.id;
  const userId = user!.id;

  // Advance the card immediately — do NOT await the swipe processing
  setCurrentIndex(prev => prev + 1);
  setLastSwipedProfile({ profile: currentProfile, action });

  // Fire Supabase in background — no await
  processSwipeAsync(action, profileId, userId).catch(err =>
    console.warn('[RoommatesScreen] Background swipe error:', err)
  );
};
```

**In `screens/renter/GroupsScreen.tsx`:**

In `handleSwipeAction`, advance the group index immediately before the async like call:

```typescript
const handleSwipeAction = async (action: 'like' | 'skip') => {
  if (!currentGroup) return;
  const groupSnapshot = currentGroup;

  // Advance immediately
  setCurrentGroupIndex(prev => prev + 1);
  setLastSwipedGroup({ group: groupSnapshot, action });

  // Fire in background — no await
  if (action === 'like') {
    handleLikeGroup(groupSnapshot).catch(err =>
      console.warn('[GroupsScreen] Background like error:', err)
    );
  }
};
```

This makes both swipes feel instant. The match detection and notifications still happen — just in the background after the card has already animated away. Match celebration modals should still appear since they're triggered inside `processSwipeAsync` which still runs, just not blocking the UI.

No DB changes needed.
