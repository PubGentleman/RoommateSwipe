### J1 — Fix AgentGroupBuilderScreen Loading Renters from StorageService Instead of Supabase

**File:** `screens/host/AgentGroupBuilderScreen.tsx`

**Bug:** In `loadData` (around line 72-106), the shortlisted renter profiles are loaded from `StorageService.getRoommateProfiles()` and filtered by the IDs from Supabase. If local storage is empty (any device other than the one that seeded mock data), no renters appear in the group builder — the agent's shortlist is always empty.

```ts
// WRONG — local storage only:
const profiles: RoommateProfile[] = await StorageService.getRoommateProfiles();
const shortlistedIds = await getShortlistedRenterIds(user.id);
const mapped = profiles.filter(p => shortlistedIds.includes(p.id)).map(p => ({ ... }));
```

**Fix:** After getting `shortlistedIds` from Supabase, fetch the renter profiles directly from Supabase by ID:
```ts
const shortlistedIds = await getShortlistedRenterIds(user.id);

if (shortlistedIds.length > 0) {
  const { data: supaRenters } = await supabase
    .from('users')
    .select(`
      id, full_name, avatar_url, age, occupation, gender, bio,
      profile:profiles(
        budget_max, budget_min, budget_per_person_min, budget_per_person_max,
        move_in_date, room_type, cleanliness, sleep_schedule, smoking, pets, interests, photos, bio
      )
    `)
    .in('id', shortlistedIds);

  const mapped: AgentRenter[] = (supaRenters || []).map((u: any) => {
    const p = Array.isArray(u.profile) ? u.profile[0] : u.profile;
    return {
      id: u.id,
      name: u.full_name || 'Unknown',
      age: u.age || 0,
      occupation: u.occupation || '',
      photos: p?.photos || (u.avatar_url ? [u.avatar_url] : []),
      budgetMin: p?.budget_per_person_min ?? (p?.budget_max ? p.budget_max * 0.8 : undefined),
      budgetMax: p?.budget_per_person_max ?? p?.budget_max,
      moveInDate: p?.move_in_date,
      cleanliness: p?.cleanliness,
      sleepSchedule: p?.sleep_schedule,
      smoking: p?.smoking,
      pets: p?.pets === 'yes' || p?.pets === true,
      interests: p?.interests || [],
      roomType: p?.room_type,
      gender: u.gender,
      bio: p?.bio || u.bio,
    };
  });
  setAllRenters(mapped);
}
```

Import `supabase` at the top of the file if not already imported.

### J3 — Fix createAgentGroup Return Value Not Captured — Wrong Group ID Sent to Renters

**File:** `screens/host/AgentGroupBuilderScreen.tsx`

**Bug:** In `handleSendInvites` (around lines 141-178), the result of `createAgentGroup` is discarded:
```ts
await createAgentGroup(group);          // returns { ...group, id: realSupabaseUUID }
await sendAgentInvites(
  user.id,
  user.name,
  group.id,                             // BUG: still 'ag_1234567...' — the local temp ID
  ...
);
```
The `createAgentGroup` service creates the group in Supabase and returns a new object with the real UUID. But because the return value isn't captured, `sendAgentInvites` inserts invite rows with `group_id = 'ag_1234567...'` which matches nothing in the database. All renter invites are orphaned.

**Fix:** Capture the return value and use the real ID:
```ts
const createdGroup = await createAgentGroup(group);   // capture the real Supabase group
await sendAgentInvites(
  user.id,
  user.name,
  createdGroup.id,                                    // real Supabase UUID
  selectedRenters.map(r => r.id),
  selectedListing,
  agentMessage,
  selectedRenters.map(r => ({ id: r.id, name: r.name, photo: r.photos?.[0] }))
);
```

### J5 + J6 — Fix getAgentGroups — Empty Members and Missing Listing Data

**File:** `services/agentMatchmakerService.ts`

**Bug:** The `getAgentGroups` Supabase query (around lines 126-160) returns `members: []` and `targetListing: undefined` because:
1. It queries `group_members(user_id, role)` but never joins user profile data — so member names/photos are never loaded
2. It gets `target_listing_id` but never joins listing data — so listing title/price are never available

Both are visible in the `AgentGroupsScreen` render where member chips and listing info are always blank.

**Fix:** Extend the Supabase query to join user and listing data:
```ts
const { data: groups, error } = await supabase
  .from('groups')
  .select(`
    id,
    name,
    created_by_agent,
    target_listing_id,
    group_status,
    created_at,
    group_members(
      user_id,
      role,
      user:users!user_id(id, full_name, avatar_url)
    ),
    listing:listings!target_listing_id(id, title, price, bedrooms, neighborhood)
  `)
  .eq('created_by_agent', agentId)
  .eq('agent_assembled', true)
  .order('created_at', { ascending: false });
```

Then update the mapping to populate `members` and `targetListing`:
```ts
return groups.map(g => {
  const rawMembers = g.group_members || [];
  const memberProfiles = rawMembers.map((m: any) => ({
    id: m.user?.id ?? m.user_id,
    name: m.user?.full_name ?? 'Unknown',
    photos: m.user?.avatar_url ? [m.user.avatar_url] : [],
  }));

  const listing = Array.isArray(g.listing) ? g.listing[0] : g.listing;

  return {
    id: g.id,
    name: g.name || 'Untitled Group',
    agentId: g.created_by_agent,
    targetListingId: g.target_listing_id,
    targetListing: listing ? {
      id: listing.id,
      title: listing.title,
      price: listing.price,
      bedrooms: listing.bedrooms,
      neighborhood: listing.neighborhood,
    } as any : undefined,
    members: memberProfiles,
    memberIds: rawMembers.map((m: any) => m.user_id),
    groupStatus: g.group_status || 'assembling',
    avgCompatibility: 0,
    combinedBudgetMin: 0,
    combinedBudgetMax: 0,
    coversRent: false,
    invites: [],
    createdAt: g.created_at,
  };
});
```

### J9 — Fix Agent Subscription Plan ID Mismatch and Missing Stripe Price IDs

**Files:** `constants/planLimits.ts`, `constants/stripePrices.ts`, `utils/hostPricing.ts`

**Background:** There are three places that define agent plans and they don't agree on ID naming:
- `hostPlansByType.ts` (the subscription UI) uses IDs: `'agent_starter'`, `'agent_pro'`, `'agent_business'`
- `planLimits.ts` (feature gating) uses IDs: `'starter'`, `'pro'`, `'business'` (no `agent_` prefix)
- `stripePrices.ts` has **no agent plan entries at all**

The result: a paying agent gets their `agentPlan` stored as `'agent_starter'`, but `getAgentPlanLimits('agent_starter')` finds no matching key in `AGENT_PLAN_LIMITS` and falls back to `pay_per_use`. The agent paid $49/mo and still has a shortlist cap of 5. Additionally, any call to look up a Stripe price ID for agent plans returns `null`, meaning the purchase flow can't complete.

---
